### 12.2 菜单结构

> 菜单由后端动态管理，前端根据用户角色加载。菜单存储在 `menus` 表，通过 `role_menus` 关联表绑定到角色。

#### 12.2.1 后端数据模型

**菜单表（`menus`）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uint | 主键 |
| parent_id | uint? | 父菜单 ID（NULL=顶级） |
| name | string(50) | 路由名称（英文，如 `UserList`） |
| title | string(50) | 菜单显示名称（中文，如"用户列表"） |
| type | int8 | 类型：1=目录，2=菜单（页面），3=按钮（权限） |
| path | string(200) | 路由路径（如 `/system/user`） |
| component | string(200) | 前端组件路径（如 `views/system/user/index.vue`） |
| redirect | string(200) | 重定向路径（目录类型用） |
| icon | string(50) | 图标（Element Plus 图标名或 SVG） |
| hidden | bool | 是否隐藏（侧边栏不显示） |
| permission | string(100) | 权限标识（如 `user:view`，按钮类型必填） |
| sort | int | 排序权重（越小越靠前） |
| status | int8 | 状态：0=禁用，1=启用 |
| created_at | time | 创建时间 |
| updated_at | time | 更新时间 |
| deleted_at | time? | 软删除时间 |

**关联表**：
- `role_menus`（role_id, menu_id）— 角色与菜单的多对多关系
- `button_permissions`（id, menu_id, permission_code, permission_name, sort, status, remark）— 按钮权限扩展表

**角色表（`roles`）关键字段**：

| 字段 | 说明 |
|------|------|
| id | 主键 |
| name | 角色名称 |
| code | 角色编码（如 `admin`, `teacher`, `student`） |
| is_system | 系统角色不可删除 |
| status | 状态 |

#### 12.2.2 API 接口

**用户端接口**（已认证）：

| 接口 | 说明 | 返回格式 |
|------|------|---------|
| `GET /api/v1/menus` | 获取当前用户的菜单树 | `Menu[]`（排除 type=3 按钮） |
| `GET /api/v1/menus/routes` | 获取当前用户的路由树 | `RouteItem[]`（前端路由格式） |
| `GET /api/v1/menus/buttons` | 获取当前用户的按钮权限 | `string[]`（权限标识数组） |

**管理端接口**（需 admin 角色）：

| 接口 | 说明 |
|------|------|
| `GET /api/v1/admin/menus` | 菜单列表（平铺） |
| `GET /api/v1/admin/menus/tree` | 菜单树（完整） |
| `GET /api/v1/admin/menus/:id` | 菜单详情 |
| `POST /api/v1/admin/menus` | 创建菜单 |
| `PUT /api/v1/admin/menus/:id` | 更新菜单 |
| `DELETE /api/v1/admin/menus/:id` | 删除菜单（有子菜单时禁止） |

#### 12.2.4 动态菜单加载流程

```
用户登录
  │
  ├─ 1. POST /api/v1/auth/login → 获取 access_token + refresh_token
  │
  ├─ 2. GET /api/v1/menus/routes → 获取路由树
  │     └─ 后端：根据 user → roles → role_menus → menus 构建树
  │        └─ 只返回 type=1（目录）和 type=2（菜单），排除 type=3（按钮）
  │
  ├─ 3. GET /api/v1/menus/buttons → 获取按钮权限列表
  │     └─ 后端：收集所有角色关联的 type=3 菜单的 permission 字段
  │
  ├─ 4. 前端动态生成路由（router.addRoute）
  │     └─ 将 RouteItem[] 转换为 Vue Router 路由配置
  │
  └─ 5. 渲染侧边栏菜单 + 按钮级权限控制
        ├─ 侧边栏：遍历菜单树渲染
        └─ 按钮：v-permission="'user:create'" 指令判断
```

#### 12.2.6 权限标识规范

**命名规则**：`模块:操作`

| 模块 | 权限标识示例 |
|------|------------|
| 用户 | `user:view`, `user:create`, `user:edit`, `user:delete`, `user:reset-password`, `user:assign-role` |
| 角色 | `role:view`, `role:create`, `role:edit`, `role:delete`, `role:assign-menu`, `role:assign-perm` |
| 题目 | `question:view`, `question:create`, `question:edit`, `question:delete`, `question:batch-publish`, `question:import`, `question:export` |
| 试卷 | `paper:view`, `paper:create`, `paper:edit`, `paper:delete`, `paper:preview`, `paper:export` |
| 考试 | `exam:view`, `exam:create`, `exam:edit`, `exam:delete`, `exam:publish`, `exam:close`, `exam:monitor` |
| 成绩 | `score:view`, `score:export`, `score:review` |
| 班级 | `class:view`, `class:create`, `class:edit`, `class:delete`, `class:member:manage` |
| 评论 | `comment:view`, `comment:audit`, `comment:delete`, `comment:pin`, `comment:featured` |
| 系统 | `system:settings`, `system:security`, `system:feature`, `system:storage`, `system:cache` |
| 日志 | `log:operation`, `log:login`, `log:system`, `log:error` |
| 备份 | `backup:view`, `backup:create`, `backup:restore`, `backup:delete`, `backup:config` |

**前端使用方式**：

```vue
<!-- 按钮权限指令 -->
<el-button v-permission="'user:create'">新增用户</el-button>
<el-button v-permission="'user:edit'">编辑</el-button>

<!-- 多权限（满足任一即可） -->
<el-button v-permission="['user:edit', 'user:delete']">操作</el-button>

<!-- 编程式判断 -->
<script setup>
import { usePermission } from '@/composables/usePermission'
const { hasPermission } = usePermission()
const canCreate = hasPermission('user:create')
</script>
```

#### 12.4.13 菜单与权限管理

**菜单管理**（`/system/menu`）：

> 菜单通过后端接口动态管理，前端根据用户角色动态加载菜单

| 功能点 | 功能描述 | API |
|-------|---------|-----|
| 菜单列表 | 树形展示所有菜单 | `GET /admin/menus/tree` |
| 新增菜单 | 创建菜单（目录/菜单/按钮） | `POST /admin/menus` |
| 编辑菜单 | 修改菜单信息 | `PUT /admin/menus/:id` |
| 删除菜单 | 删除菜单（有子菜单时禁止删除） | `DELETE /admin/menus/:id` |

**菜单数据结构**：

| 字段 | 说明 |
|------|------|
| id | 菜单 ID |
| parent_id | 父菜单 ID（0=顶级） |
| name | 菜单名称 |
| type | 菜单类型：directory（目录）/ menu（菜单）/ button（按钮） |
| path | 路由路径（如 /system/user） |
| component | 前端组件路径（如 views/system/user/index.vue） |
| icon | 菜单图标 |
| permission | 权限标识（如 user:view） |
| sort_order | 排序权重 |
| visible | 是否显示（0=隐藏 1=显示） |
| status | 状态（0=禁用 1=启用） |

**动态菜单加载流程**：
1. 用户登录成功 → 后端返回用户角色
2. 前端调用 `/api/v1/menus` 接口获取当前用户的菜单列表
3. 后端根据用户角色查询关联的菜单权限，返回树形菜单数据
4. 前端动态生成路由表，渲染侧边栏菜单
5. 按钮级别权限通过权限标识控制（`v-permission` 指令）

**按钮权限管理**（`/system/button-permission`）：

| 功能点 | 功能描述 | API |
|-------|---------|-----|
| 按钮列表 | 查看菜单下的按钮权限列表 | `GET /admin/button-permissions/menu/:menu_id` |
| 新增/编辑/删除 | 按钮权限 CRUD | `CRUD /admin/button-permissions` |

**权限标识命名规范**：`模块:操作`
- 用户模块：`user:view`, `user:create`, `user:edit`, `user:delete`
- 题目模块：`question:view`, `question:create`, `question:edit`, `question:delete`
- 考试模块：`exam:view`, `exam:create`, `exam:publish`, `exam:monitor`
- 班级模块：`class:view`, `class:create`, `class:member:manage`
- 系统模块：`system:settings`, `system:log:view`, `system:backup`

**按钮权限使用示例**：
```vue
<el-button v-permission="'user:create'">新增用户</el-button>
<el-button v-permission="'user:edit'">编辑</el-button>
<el-button v-permission="'user:delete'">删除</el-button>
```
