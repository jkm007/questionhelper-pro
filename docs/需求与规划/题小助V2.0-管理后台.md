| 题库管理 | SetupAdminQuestionRoutes | `/api/v1/admin/questions`, `/api/v1/admin/categories`, `/api/v1/admin/knowledge-points`, `/api/v1/admin/reviews`, `/api/v1/admin/sensitive-words`, `/api/v1/admin/shares` |

| 班级管理 | SetupAdminClassRoutes | `/api/v1/admin/class` |
| 内容管理 | SetupAdminCommentRoutes | `/api/v1/admin/comments`, `/api/v1/admin/comments/reports`, `/api/v1/admin/comments/blacklist`, `/api/v1/admin/comments/audit-rules` |

| 文件管理 | SetupAdminFileRoutes | `/api/v1/admin/file` |
| 系统设置 | SetupAdminSystemRoutes | `/api/v1/admin/settings`, `/api/v1/admin/features`, `/api/v1/admin/security`, `/api/v1/admin/storage`, `/api/v1/admin/email`, `/api/v1/admin/sms`, `/api/v1/admin/cache`, `/api/v1/admin/theme` |

| 备份管理 | SetupAdminSystemRoutes (backup) | `/api/v1/admin/backup` |
| 菜单/权限 | SetupAdminMenuRoutes, SetupAdminButtonPermissionRoutes | `/api/v1/admin/menus`, `/api/v1/admin/button-permissions` |

### 12.4 页面功能

#### 12.4.1 仪表盘

| 功能点 | 功能描述 | API |
|-------|---------|-----|
| 核心指标 | 总用户数、总题目数、总考试数、今日活跃等核心指标卡片 | `GET /admin/statistics` |
| 趋势图表 | 用户增长趋势、答题量趋势、正确率趋势 | `GET /admin/statistics` |
| 数据对比 | 不同时间段数据对比（本周/本月/自定义） | `GET /admin/statistics/compare` |
| 快速入口 | 新增题目、创建考试、用户管理等快捷入口 | — |
| 数据预警 | 数据异常预警（如突然大量注册、错误率飙升） | `GET /admin/alerts/records` |

#### 12.4.2 用户管理

**用户列表**（`/system/user`）：

| 功能点 | 功能描述 | API |
|-------|---------|-----|
| 用户列表 | 分页查询用户，支持关键词/状态/角色筛选 | `GET /admin/users` |
| 用户详情 | 查看用户基本信息、角色、登录记录 | `GET /admin/users/:id` |

#### 12.4.13 菜单与权限管理

**菜单管理**（`/system/menu`）：

> 菜单通过后端接口动态管理，前端根据用户角色动态加载菜单

| 功能点 | 功能描述 | API |
|-------|---------|-----|
| 菜单列表 | 树形展示所有菜单 | `GET /admin/menus/tree` |
| 新增菜单 | 创建菜单（目录/菜单/按钮） | `POST /admin/menus` |
| 编辑菜单 | 修改菜单信息 | `PUT /admin/menus/:id` |
| 删除菜单 | 删除菜单（有子菜单时禁止删除） | `DELETE /admin/menus/:id` |

**菜单数据结构**：

| 字段 | 说明 |
|------|------|
| id | 菜单 ID |
| parent_id | 父菜单 ID（0=顶级） |
| name | 菜单名称 |
| type | 菜单类型：directory（目录）/ menu（菜单）/ button（按钮） |
| path | 路由路径（如 /system/user） |
| component | 前端组件路径（如 views/system/user/index.vue） |
| icon | 菜单图标 |
| permission | 权限标识（如 user:view） |
| sort_order | 排序权重 |
| visible | 是否显示（0=隐藏 1=显示） |
| status | 状态（0=禁用 1=启用） |

**动态菜单加载流程**：
1. 用户登录成功 → 后端返回用户角色
2. 前端调用 `/api/v1/menus` 接口获取当前用户的菜单列表
3. 后端根据用户角色查询关联的菜单权限，返回树形菜单数据
4. 前端动态生成路由表，渲染侧边栏菜单
5. 按钮级别权限通过权限标识控制（`v-permission` 指令）

**按钮权限管理**（`/system/button-permission`）：

| 功能点 | 功能描述 | API |
|-------|---------|-----|
| 按钮列表 | 查看菜单下的按钮权限列表 | `GET /admin/button-permissions/menu/:menu_id` |
| 新增/编辑/删除 | 按钮权限 CRUD | `CRUD /admin/button-permissions` |

**权限标识命名规范**：`模块:操作`
- 用户模块：`user:view`, `user:create`, `user:edit`, `user:delete`
- 题目模块：`question:view`, `question:create`, `question:edit`, `question:delete`
- 考试模块：`exam:view`, `exam:create`, `exam:publish`, `exam:monitor`
- 班级模块：`class:view`, `class:create`, `class:member:manage`
- 系统模块：`system:settings`, `system:log:view`, `system:backup`

**按钮权限使用示例**：
```vue
<el-button v-permission="'user:create'">新增用户</el-button>
<el-button v-permission="'user:edit'">编辑</el-button>
<el-button v-permission="'user:delete'">删除</el-button>
```
