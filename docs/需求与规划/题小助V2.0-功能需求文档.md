# 题小助 V2.0 功能需求文档

**项目名称**：题小助 (QuestionHelper)
**版本**：V2.0
**日期**：2026-05-30

---

## 文档概述

### 项目背景

题小助（QuestionHelper）是一款面向教育领域的题库管理与在线考试平台。V2.0 版本在 V1.0 基础上，新增了多角色权限体系、班级管理系统、内容创作系统、刷题练习系统等核心功能，支持公共/私有/班级三种资源管理模式，满足教育机构、教师、学生和内容创作者的多样化需求。

### 文档目的

本文档为题小助 V2.0 的功能需求文档，用于：
- 明确系统各模块的功能需求和业务规则
- 作为产品设计、开发实现和测试验收的依据
- 作为项目各方（产品、设计、开发、测试）的沟通基准

### 目标读者

| 角色 | 说明 |
|------|------|
| 产品经理 | 理解功能需求，指导产品设计 |
| 设计师 | 了解功能逻辑，指导 UI/UX 设计 |
| 开发工程师 | 理解业务规则，指导技术实现 |
| 测试工程师 | 编写测试用例，验证功能实现 |
| 项目管理者 | 评估工作量，制定开发计划 |

### 适用范围

本文档适用于题小助 V2.0 的 Web 端（管理后台 + 用户端）和移动端（H5/小程序）的功能需求定义。

## 术语定义

| 术语 | 说明 |
|------|------|
| 公共资源 | 由内容管理员/管理员创建的题目、分类、知识点、试卷等，所有用户可见可用 |
| 私有资源 | 由创作者/教师创建的资源，仅创建者可见，不影响公共题库 |
| 班级资源 | 由教师/班级创作者创建的资源，仅班级成员可见 |
| 班级创作者 | 班级成员通过申请审批获得的特殊角色，可在班级内创建题目、分类、知识点 |
| 内容管理员 | 管理公共题库、公共分类、公共知识点的角色 |
| 创作者 | 可创建私有资源并可提交公共审核的角色 |
| 教师 | 可创建班级、管理班级资源、发布考试的角色 |
| 分类树 | 无限层级的知识分类体系，支持父子关系 |
| 知识点 | 细粒度的知识标签，与分类树配合使用 |
| 试卷 | 由多道题目组成的考核集合，包含分值、时长等配置 |
| 考试 | 试卷的发布实例，包含开始/结束时间、防作弊配置等 |
| 练习 | 非考核模式的答题活动，即时评判对错 |
| 错题本 | 自动收集用户答错题目的学习工具 |
| Access Token | 访问令牌，有效期 2 小时，用于接口认证 |
| Refresh Token | 刷新令牌，有效期 7 天，用于静默续期 Access Token |
| session_id | 练习会话标识，同一次连续练习共享 |

## 一、用户认证系统

### 1.1 用户注册

| 功能点 | 功能描述 |
|-------|---------|
| 用户名注册 | 用户名 4-20 位，字母/数字/下划线，必填 |
| 昵称设置 | 昵称 2-20 个字符，支持中英文、数字、下划线，敏感词过滤 |
| 密码设置 | 密码 6-20 位需包含字母和数字，确认密码二次校验，前端显示密码强度指示（弱/中/强） |
| 邮箱验证 | 邮箱为必填项，需发送验证码并验证通过后方可注册 |
| 手机号绑定 | 手机号为选填项，用于找回密码和通知 |
| 图形验证码 | 注册时需填写图形验证码防机器人 |
| 用户协议 | 勾选同意《用户协议》和《隐私政策》方可注册 |
| 注册来源 | 记录注册渠道（H5/小程序/App/Web），用于统计分析 |

**功能逻辑**：
- 用户名唯一约束，重复时返回对应错误码
- 邮箱唯一约束，重复时返回对应错误码
- 邮箱为必填项，需先发送验证码并验证通过后才能提交注册
- 手机号为选填项，填写时校验唯一约束
- 密码使用 bcrypt（cost=12）加密存储
- 注册成功后返回 Access Token + Refresh Token，自动登录
- 注册来源通过请求头或参数自动识别

### 1.2 用户登录

| 功能点 | 功能描述 |
|-------|---------|
| 账号密码登录 | 支持用户名/邮箱/手机号 + 密码登录 |
| 邮箱验证码登录 | 邮箱 + 邮箱验证码登录 |
| 第三方 OAuth 登录 | 支持微信（小程序/公众号/扫码）、GitHub、Google 等第三方授权登录 |
| 图形验证码 | 密码错误 3 次后需填写图形验证码 |
| 记住我 | 勾选后延长 Refresh Token 有效期至 30 天 |
| 登录限制 | 同一账号连续登录失败5次，锁定15分钟 |
| 异常登录检测 | 检测异地登录、新设备登录，触发安全验证 |
| 新设备登录通知 | 新设备首次登录时，发送站内消息+邮箱安全通知 |

**第三方 OAuth 登录逻辑**：
- 小程序端：wx.login() 获取 code → 后端换 OpenID → 自动登录或绑定已有账号
- 公众号/扫码：授权获取 openid → 自动登录或绑定已有账号
- 首次 OAuth 登录需绑定已有账号或创建新账号

**微信小程序静默登录**：
- 小程序启动时调用 wx.checkSession() 检查登录态是否过期
- 未过期：直接使用本地缓存的 Token
- 已过期：调用 wx.login() 获取新 code，静默刷新 Token
- 静默登录失败时，跳转登录页引导用户手动登录

**登录限制规则**：
- 同一账号连续密码错误5次，账号锁定15分钟
- 锁定期间可通过邮箱验证码解锁
- 同一IP短时间内大量登录失败，IP临时封禁

**功能逻辑**：
- 登录成功返回 Access Token（2小时有效期）+ Refresh Token（7天有效期）
- Token 通过 `Authorization: Bearer <token>` HTTP Header 传递
- 禁用用户（status=disabled）不能登录
- 401 响应自动跳转登录页
- 登录后返回用户角色列表，前端根据角色展示对应功能菜单
- 拥有多角色的用户可在不同角色间切换（如教师角色和创作者角色）

### 1.2.1 登录设备管理

| 功能点 | 功能描述 |
|-------|---------|
| 设备列表 | 查看当前登录设备列表（设备类型、登录时间、IP地址、地理位置） |
| 当前设备标识 | 标识当前正在使用的设备 |
| 踢出设备 | 强制退出指定设备的登录状态 |
| 踢出所有设备 | 退出除当前设备外的所有设备 |

**功能逻辑**：
- 登录时记录设备信息（设备类型、IP、User-Agent）
- 踢出设备时，该设备的 Token 加入黑名单
- 踢出所有设备时，清空该用户所有 Refresh Token

### 1.3 Token 管理

| 功能点 | 功能描述 |
|-------|---------|
| Token 刷新 | Access Token 过期前通过 Refresh Token 静默续期，返回新 Token 对 |
| Token 注销 | 退出登录时 Access Token 加入 Redis 黑名单，TTL = Token 剩余有效期 |
| 前端守卫 | 管理后台路由基于 Token 做登录态拦截，401 自动退出并跳转登录页 |

**功能逻辑**：
- Refresh Token 过期需重新登录
- Redis 黑名单 Key：`questionhelper:token:bl:{jti}`，值为 `"1"`
- Access Token 存储在内存或 localStorage（Web端），Refresh Token 存储在 httpOnly Cookie
- Token 刷新失败时（Refresh Token 过期或失效），清除本地 Token，跳转登录页

**Token 自动刷新时机**：
- Access Token 有效期 2 小时，过期前 5 分钟自动刷新
- 前端每次请求前检查 Token 剩余有效期，小于 5 分钟时触发刷新
- 刷新期间新请求排队等待，刷新完成后使用新 Token 重发
- 并发请求只触发一次刷新，避免多次刷新冲突

### 1.4 密码找回

| 功能点 | 功能描述 |
|-------|---------|
| 邮箱找回 | 输入邮箱 + 图形验证码 → 发送邮箱验证码 → 设置新密码 |

**功能逻辑**：
- 账号不存在时返回统一提示"验证码已发送"（防止账号枚举）
- 重置令牌有效期 15 分钟
- 验证码有效期 5 分钟
- 新密码不能与最近3次历史密码相同
- 密码重置成功后，所有设备强制退出，需重新登录

### 1.5 退出登录

| 功能点 | 功能描述 |
|-------|---------|
| 退出当前设备 | 退出当前设备登录状态，清除本地 Token |
| 退出所有设备 | 退出所有设备登录状态（跳转到1.2.1设备管理操作） |

**功能逻辑**：
- 退出时 Access Token 加入 Redis 黑名单
- 退出所有设备时，所有设备的 Token 加入黑名单，Refresh Token 同时失效

### 1.6 账号注销

| 功能点 | 功能描述 |
|-------|---------|
| 注销条件检查 | 检查是否有未完成的考试、进行中的班级、未处理的角色申请等 |
| 申请注销 | 用户申请注销账号，需验证身份（密码/邮箱验证码） |
| 数据导出 | 注销前可导出个人数据（答题记录、错题本、收藏等） |
| 注销确认 | 注销前提示数据将被清除，需二次确认 |
| 注销处理 | 账号标记为已注销，30天内可撤销，30天后永久删除 |

**注销条件**：
- 无进行中的考试（答题中/待阅卷）
- 无未处理的角色申请
- 作为教师的班级已转让或解散
- 无未完成的订单（如有付费功能）

**功能逻辑**：
- 注销后账号不可登录
- 30天冷静期内可联系客服撤销
- 30天后数据永久删除，不可恢复

### 1.7 图形验证码

| 功能点 | 功能描述 |
|-------|---------|
| 获取验证码 | 返回 Base64 编码的验证码图片 + captchaKey |
| 验证校验 | 后续请求携带 captchaKey + 用户输入的验证码 |
| 验证码类型 | 支持数字验证码、字母验证码、算术验证码（如 3+5=?） |
| 安全级别 | 随机干扰线、噪点、字符扭曲，防止OCR识别 |

**功能逻辑**：
- 验证码有效期 5 分钟
- 验证失败后自动刷新验证码
- 同一 captchaKey 只能验证一次
- 验证码字符忽略大小写

### 1.8 邮箱验证码限制

| 功能点 | 功能描述 |
|-------|---------|
| 发送频率限制 | 同一邮箱：60秒内只能发送1次 |
| 每日上限 | 同一邮箱：每天最多发送10次 |
| 验证次数限制 | 同一验证码最多验证5次，超过后需重新发送 |
| IP限制 | 同一IP每小时最多发送50次验证码 |

**功能逻辑**：
- 超过频率限制返回对应错误码和剩余等待时间
- 验证码有效期 5 分钟
- 超过验证次数后验证码失效

### 1.9 第三方账号绑定

| 功能点 | 功能描述 |
|-------|---------|
| 绑定微信 | 绑定微信账号，用于微信一键登录 |
| 绑定GitHub | 绑定GitHub账号，用于GitHub登录 |
| 绑定Google | 绑定Google账号，用于Google登录 |
| 解绑账号 | 解绑已绑定的第三方账号（需验证身份） |
| 绑定状态 | 查看已绑定的第三方账号列表 |

**绑定规则**：
- 一个第三方账号只能绑定一个本地账号
- 绑定时如该第三方账号已绑定其他账号，提示冲突
- 解绑时需验证密码或邮箱验证码
- 解绑后无法使用该第三方账号登录

## 二、用户管理系统

### 2.1 用户信息管理

| 个人信息修改 | 修改昵称、头像、性别、生日、简介等 |
| 头像上传 | 上传头像，支持jpg/png格式，大小不超过2MB，自动裁剪为正方形 |
| 修改密码 | 输入旧密码验证后设置新密码 |
| 绑定邮箱 | 绑定或更换邮箱，需验证新邮箱 |
| 绑定手机号 | 绑定或更换手机号，需验证新手机号（已实名认证的需验证原手机号） |
| 第三方账号管理 | 查看已绑定的第三方账号，支持绑定/解绑 |
| 角色信息 | 查看当前用户拥有的角色列表 |
| 登录日志 | 查看最近登录记录（时间、IP、设备），支持查看异常登录 |
| 班级创作者状态 | 查看在各班级的班级创作者状态（已申请/已通过/已驳回） |
| 隐私设置 | 设置个人信息对外可见范围 |

**隐私设置项**：

| 设置项 | 可选值 | 说明 |
|-------|-------|------|
| 个人主页可见性 | 所有人/仅班级成员/仅自己 | 控制其他用户是否能访问个人主页 |
| 真实姓名可见性 | 所有人/仅班级成员/仅自己 | 控制真实姓名是否对外显示 |

#### 3.2.3 导入流程

```
上传文件 → 格式校验 → 解析内容 → 分组校验 → 查重 → 批量入库 → 返回报告
```

1. 校验文件格式（仅 .xlsx）和大小（< 10MB）
2. 从第 2 行开始解析（第 1 行为表头）
3. 按"题目序号"分组，同一序号的多行 = 一道题的多个选项
4. 逐组校验：题型合法、题目内容非空、分类 ID 存在、选择题至少一个正确选项
5. 内容 MD5 查重（跳过重复题目，记录到失败列表）
6. 构建 content JSON，批量 INSERT（事务）
7. 返回导入报告：`{total, success, failed, errors[{row, reason}]}`

|------|--------|---------|------|

| 私有分类 | 创作者/教师 | 仅创建者 | 个人空间的分类 |
| 班级分类 | 教师/班级创作者 | 班级成员 | 班级内的分类 |

#### 3.3.2 数据结构与逻辑

**分类树数据结构**：
- `parent_id`：父级 ID（0=根节点）
- `level`：层级，根节点为 1，由后端自动计算
- `path`：节点路径，如 `/1/3/5/`，用于子树查询
- `sort_order`：同级排序权重
- `type`：分类类型（public/private/class）

**分类树维护逻辑**：
- 创建分类：level = parent.level + 1；path = 父 path + 当前 ID + "/"
- 移动分类：校验目标父节点存在 → 禁止 parent_id = 自身 ID → 禁止移动到自身后代节点下 → 更新 parent_id、level、path → 递归刷新所有后代节点 level、path
- 筛选题目：通过 path 查询该分类全部后代 ID，使用 `category_id IN (当前分类 + 后代分类)` 查询题目

#### 4.1.3 智能组卷

**组卷规则**：

| 规则项 | 说明 |
|-------|------|
| 题型 | 指定题型（单选/多选/判断/填空/简答） |
| 分类 | 指定分类树节点，父分类自动包含子分类 |
| 难度 | 指定难度（1-3），或不限制 |
| 数量 | 指定抽取数量 |
| 分值 | 指定每题分值 |

**组卷逻辑**：
1. 创建 exams 记录
2. 维护已抽题 ID 集合 excludes = {}
3. 遍历 rules 数组，每条规则执行：
   - 如果传入 category_id，先计算分类子树 IDs（当前分类 + 全部后代分类）
   - `SELECT * FROM questions WHERE type=? AND deleted_at IS NULL AND status='published' AND (category_id IN categoryIDs) AND (difficulty=? OR 不限制) AND id NOT IN (excludes) ORDER BY RAND() LIMIT ?`
   - 将抽到的题 ID 加入 excludes
   - 如果实际抽到数量 < count：删除已创建试卷并返回错误
   - 写入 exam_rules 和 exam_questions（sort_order 递增，score 赋值，保存 question_snapshot）
4. 返回生成的试卷 ID 和题目数量

**question_snapshot**：试卷创建或考试开始时写入，记录题目完整内容（题干+选项顺序+正确答案），保证历史试卷一致性。即使原题目被修改，已考完的试卷展示仍保持一致。

#### 4.3.2 考试流程

```
查看考试 → 开始考试 → 答题过程 → 交卷 → 查看成绩
```

**详细流程**：
1. **查看试卷**：GET /api/exams/:id → 返回试卷基本信息 + 题型分布汇总（不含具体题目）
2. **开始考试**：POST /api/exams/:id/start
   - 校验：考试是否在开放时间内、用户是否超过 max_attempts、试卷是否存在题目
   - 执行：读取 exam_questions 及题目快照 → 创建 exam_records（status=in_progress） → 写入 Redis（exam:session + exam:scores） → 写入延迟队列（ZADD tixuan:exam:delay） → 返回 record_id、时长、总分、题量
3. **答题过程**：每答一题 POST /api/exams/:id/save-answer → 答案幂等写入 Redis Hash
4. **断线续考**：GET /api/exams/:id/resume → 从 Redis 读取 answers + session TTL 计算 remaining_sec
5. **交卷**：POST /api/exams/:id/submit 或延迟队列到期
   - 从 Redis 读取所有答案
   - 客观题自动评分，主观题标记待阅卷（is_correct=2）
   - 写入 exam_answers 表
   - 更新 exam_records（score, obj_score, status, duration_used）
   - 答错题写入/更新 wrong_questions 表
   - DEL 所有 Redis 考试相关 Key，ZREM 延迟队列成员
6. **查看成绩**：GET /api/exams/:id/record → 客观题即时展示，主观题待阅卷后展示

#### 4.3.3 Redis 会话设计

| Key | 说明 | TTL |
|-----|------|-----|
| `tixuan:exam:session:{uid}:{eid}` | 考试题目顺序/会话状态 | 考试时长 |
| `tixuan:exam:answers:{uid}:{eid}` | 用户答案缓存（Hash，幂等覆盖写入） | 考试时长 |
| `tixuan:exam:scores:{uid}:{eid}` | 本次考试每题分值 | 考试时长 |
| `tixuan:exam:delay`（ZSet） | 延迟队列，Score = 交卷时间戳，Member = uid:eid:attempt_no | - |

### 4.4 评分规则

#### 4.4.1 评分方式

| 评分方式 | 适用题型 | 实现逻辑 |
|----------|---------|---------|
| 精确匹配 | 单选、多选 | 用户答案与 answer 完全一致即得分 |
| 关键词子串匹配 | 填空题（客观） | 用户答案包含 keywords 中所有关键词即得分；忽略大小写和首尾空格 |
| 人工阅卷 | 填空题（主观）、简答题 | 管理员在后台查看答案并打分 |
| 部分得分 | 多选题 | 可配置全部选对才得分或多选漏选部分得分 |

**分数存储**：所有分数字段统一使用 DECIMAL(5,2)（最大 999.99 分），支持 0.5 分的小题分值。

#### 4.4.2 阅卷流程

**阅卷权限**：
- 公共考试：内容管理员/管理员阅卷
- 班级考试：教师阅卷
- 私有考试：创建者自阅

**阅卷功能**：

| 功能点 | 功能描述 |
|-------|---------|

| 评语模板 | 预设常用评语，提高效率 |

| 标准答案展示 | 考试结束后展示标准答案和解析（可配置是否显示） |

**成绩复核流程**：
```
学生申请 → 教师审核 → 结果反馈 → 学生确认
```

**成绩复核规则**：

- 教师需在 3 天内审核
- 复核结果通过站内消息通知学生

#### 5.3.3 权限与审批

**班级创作者权限**：

| 权限项 | 班级创作者 | 普通班级成员 |
|-------|-----------|-------------|
| 查看班级题目 | ✓ | ✓ |
| 创建班级题目 | ✓ | - |
| 创建班级分类 | ✓ | - |
| 创建班级知识点 | ✓ | - |
| 使用公共题目 | ✓ | ✓ |

- 班级创作者不影响公共题库
- 教师可随时撤销班级创作者权限

**审批规则**：
- 教师只能审批自己班级的班级创作者申请
- 管理员/超级管理员可审批所有班级的班级创作者申请
- 审批通过后立即生效，成员获得班级创作者权限
- 审批驳回需填写驳回原因，申请人可查看

**权限粒度**：

- 申请流程：班级成员申请 → 教师审批 → 通过后获得班级创作者权限

## 十四、文件管理

### 14.1 文件上传

| 功能点 | 功能描述 |
|-------|---------|
| 图片上传 | 支持 jpg/png/gif/webp 格式，单文件最大 5MB |
| 文档上传 | 支持 xlsx/docx/pdf 格式，单文件最大 20MB |
| 音频上传 | 支持 mp3/wav/m4a 格式，单文件最大 50MB |
| 视频上传 | 支持 mp4/webm 格式，单文件最大 200MB |
| 头像上传 | 自动裁剪为正方形，最大 2MB |
| 封面上传 | 班级封面、试卷封面，最大 5MB |
| 批量上传 | 支持多文件同时上传，最多 20 个文件 |
| 拖拽上传 | 支持拖拽文件到上传区域 |
| 上传进度 | 显示上传进度条和上传速度 |

**上传限制规则**：
- 文件名长度不超过 200 字符
- 文件名自动重命名（UUID + 原始扩展名），防止冲突和路径注入
- 图片自动压缩（超过 1920px 宽度时等比缩放）
- 上传前校验文件 MIME 类型，拒绝不匹配的文件
- 同一用户每日上传文件数限制 500 个

### 14.2 文件存储

| 功能点 | 功能描述 |
|-------|---------|
| 存储配置 | 支持本地存储、OSS（阿里云/腾讯云/MinIO）等存储后端 |
| 目录结构 | 按业务类型分目录：avatar/、question/、exam/、class/、temp/ |
| 文件访问 | 公开文件直接访问，私有文件签名 URL（有效期 1 小时） |
| CDN 加速 | 支持配置 CDN 域名加速静态资源访问 |
| 缩略图 | 图片自动生成缩略图（100x100、300x300、800x800） |
| 文件去重 | 基于文件 MD5 去重，相同文件只存储一份 |

### 14.3 文件管理

| 功能点 | 功能描述 |
|-------|---------|
| 文件列表 | 管理员查看所有上传文件，支持按类型、大小、时间筛选 |
| 文件详情 | 查看文件信息：名称、大小、类型、上传者、上传时间、引用次数 |
| 文件删除 | 删除文件（检查引用计数，有引用时提示确认） |
| 孤立文件清理 | 定时清理未被引用的临时文件（temp 目录下超过 24 小时的文件） |
| 存储统计 | 统计存储使用量：总量、各类型占比、增长趋势 |
| 文件搜索 | 按文件名、上传者、类型搜索文件 |
| 批量删除 | 批量删除选中的文件 |

### 14.4 文件引用与安全

| 功能点 | 功能描述 |
|-------|---------|
| 引用计数 | 记录每个文件被引用的次数，防止误删正在使用的文件 |
| 防盗链 | 配置 Referer 白名单，防止外部站点盗链 |
| 病毒扫描 | 上传文件自动进行病毒扫描（可选） |
| 敏感内容检测 | 图片上传自动进行敏感内容检测（可选） |
| 水印功能 | 图片可配置自动添加水印（文字/图片水印） |
| 访问日志 | 记录文件访问日志（访问时间、IP、文件路径） |

**文件生命周期**：
```
上传 → 临时存储（temp/）→ 业务关联 → 正式存储
                                    ↓
                              解除引用 → 标记为孤立
                                    ↓
                              定时清理（24小时后）
```
