# 题小助 V2.0 — 用户与系统

> 本文档为需求文档主文件，包含文档概述、术语定义、用户认证、用户管理、系统设置。其他模块见对应子文件。

## 文档概述

### 项目背景

题小助（QuestionHelper）是一款面向教育领域的题库管理与在线考试平台。V2.0 版本在 V1.0 基础上，新增了多角色权限体系、班级管理系统、内容创作系统、刷题练习系统等核心功能，支持公共/私有/班级三种资源管理模式，满足教育机构、教师、学生和内容创作者的多样化需求。

### 文档目的

本文档为题小助 V2.0 的功能需求文档，用于：
- 明确系统各模块的功能需求和业务规则
- 作为产品设计、开发实现和测试验收的依据
- 作为项目各方（产品、设计、开发、测试）的沟通基准

### 目标读者

| 角色 | 说明 |
|------|------|
| 产品经理 | 理解功能需求，指导产品设计 |
| 设计师 | 了解功能逻辑，指导 UI/UX 设计 |
| 开发工程师 | 理解业务规则，指导技术实现 |
| 测试工程师 | 编写测试用例，验证功能实现 |
| 项目管理者 | 评估工作量，制定开发计划 |

### 适用范围

本文档适用于题小助 V2.0 的 Web 端（管理后台 + 用户端）和移动端（H5/小程序）的功能需求定义。

## 术语定义

| 术语 | 说明 |
|------|------|
| 公共资源 | 由内容管理员/管理员创建的题目、分类、知识点、试卷等，所有用户可见可用 |
| 私有资源 | 由创作者/教师创建的资源，仅创建者可见，不影响公共题库 |
| 班级资源 | 由教师/班级创作者创建的资源，仅班级成员可见 |
| 班级创作者 | 班级成员通过申请审批获得的特殊角色，可在班级内创建题目、分类、知识点 |
| 内容管理员 | 管理公共题库、公共分类、公共知识点的角色 |
| 创作者 | 可创建私有资源并可提交公共审核的角色 |
| 教师 | 可创建班级、管理班级资源、发布考试的角色 |
| 分类树 | 无限层级的知识分类体系，支持父子关系 |
| 知识点 | 细粒度的知识标签，与分类树配合使用 |
| 试卷 | 由多道题目组成的考核集合，包含分值、时长等配置 |
| 考试 | 试卷的发布实例，包含开始/结束时间、防作弊配置等 |
| 练习 | 非考核模式的答题活动，即时评判对错 |
| 错题本 | 自动收集用户答错题目的学习工具 |
| Access Token | 访问令牌，有效期 2 小时，用于接口认证 |
| Refresh Token | 刷新令牌，有效期 7 天，用于静默续期 Access Token |
| session_id | 练习会话标识，同一次连续练习共享 |

## 一、用户认证系统

### 1.1 用户注册

| 功能点 | 功能描述 |
|-------|---------|
| 用户名注册 | 用户名 4-20 位，字母/数字/下划线，必填 |
| 昵称设置 | 昵称 2-20 个字符，支持中英文、数字、下划线，敏感词过滤 |
| 密码设置 | 密码 6-20 位需包含字母和数字，确认密码二次校验，前端显示密码强度指示（弱/中/强） |
| 邮箱验证 | 邮箱为必填项，需发送验证码并验证通过后方可注册 |
| 手机号绑定 | 手机号为选填项，用于找回密码和通知 |
| 图形验证码 | 注册时需填写图形验证码防机器人 |
| 用户协议 | 勾选同意《用户协议》和《隐私政策》方可注册 |
| 注册来源 | 记录注册渠道（H5/小程序/App/Web），用于统计分析 |

**功能逻辑**：
- 用户名唯一约束，重复时返回对应错误码
- 邮箱唯一约束，重复时返回对应错误码
- 邮箱为必填项，需先发送验证码并验证通过后才能提交注册
- 手机号为选填项，填写时校验唯一约束
- 密码使用 bcrypt（cost=12）加密存储
- 注册成功后返回 Access Token + Refresh Token，自动登录
- 注册来源通过请求头或参数自动识别

### 1.2 用户登录

#### 1.2.1 登录方式

| 登录方式 | 说明 | 适用场景 |
|---------|------|---------|
| 账号密码登录 | 用户名/邮箱/手机号 + 密码 | Web端、移动端通用 |
| 邮箱验证码登录 | 邮箱 + 邮箱验证码 | 忘记密码时使用 |
| 微信小程序登录 | wx.login() 获取 code | 微信小程序端 |
| 微信公众号登录 | 授权获取 openid | 微信公众号端 |
| 微信扫码登录 | 扫描二维码授权 | Web端 |
| GitHub 登录 | OAuth 2.0 授权 | Web端 |
| Google 登录 | OAuth 2.0 授权 | Web端 |

**账号密码登录**：
- 支持用户名/邮箱/手机号 + 密码登录
- 密码错误 3 次后需填写图形验证码
- 勾选"记住我"后延长 Refresh Token 有效期至 30 天

**邮箱验证码登录**：
- 输入邮箱 + 图形验证码 → 发送邮箱验证码 → 输入验证码登录
- 验证码有效期 5 分钟

**第三方 OAuth 登录**：
- 小程序端：wx.login() 获取 code → 后端换 OpenID → 自动登录或绑定已有账号
- 公众号/扫码：授权获取 openid → 自动登录或绑定已有账号
- 首次 OAuth 登录需绑定已有账号或创建新账号

**微信小程序静默登录**：
- 小程序启动时调用 wx.checkSession() 检查登录态是否过期
- 未过期：直接使用本地缓存的 Token
- 已过期：调用 wx.login() 获取新 code，静默刷新 Token
- 静默登录失败时，跳转登录页引导用户手动登录

#### 1.2.2 登录限制与安全

**登录限制规则**：

| 限制项 | 规则 | 处理方式 |
|-------|------|---------|
| 密码错误 | 同一账号连续错误 5 次 | 锁定 15 分钟 |
| IP 限制 | 同一 IP 短时间内大量登录失败 | IP 临时封禁 |
| 图形验证码 | 密码错误 3 次后 | 需填写图形验证码 |

**解锁方式**：
- 锁定期间可通过邮箱验证码解锁
- 锁定时间到期自动解锁
- 管理员可手动解锁

**异常登录检测**：
- 检测异地登录（IP 地理位置变化）
- 检测新设备登录（User-Agent 变化）
- 触发安全验证（邮箱验证码）
- 新设备首次登录时，发送站内消息 + 邮箱安全通知

#### 1.2.3 登录后处理

**Token 返回**：
- 登录成功返回 Access Token（2小时有效期）+ Refresh Token（7天有效期）
- Token 通过 `Authorization: Bearer <token>` HTTP Header 传递
- 禁用用户（status=disabled）不能登录
- 401 响应自动跳转登录页

**用户信息返回**：
- 登录后返回用户角色列表
- 前端根据角色展示对应功能菜单
- 拥有多角色的用户可在不同角色间切换（如教师角色和创作者角色）

### 1.2.1 登录设备管理

| 功能点 | 功能描述 |
|-------|---------|
| 设备列表 | 查看当前登录设备列表（设备类型、登录时间、IP地址、地理位置） |
| 当前设备标识 | 标识当前正在使用的设备 |
| 踢出设备 | 强制退出指定设备的登录状态 |
| 踢出所有设备 | 退出除当前设备外的所有设备 |

**功能逻辑**：
- 登录时记录设备信息（设备类型、IP、User-Agent）
- 踢出设备时，该设备的 Token 加入黑名单
- 踢出所有设备时，清空该用户所有 Refresh Token

### 1.3 Token 管理

| 功能点 | 功能描述 |
|-------|---------|
| Token 刷新 | Access Token 过期前通过 Refresh Token 静默续期，返回新 Token 对 |
| Token 注销 | 退出登录时 Access Token 加入 Redis 黑名单，TTL = Token 剩余有效期 |
| 前端守卫 | 管理后台路由基于 Token 做登录态拦截，401 自动退出并跳转登录页 |

**功能逻辑**：
- Refresh Token 过期需重新登录
- Redis 黑名单 Key：`questionhelper:token:bl:{jti}`，值为 `"1"`
- Access Token 存储在内存或 localStorage（Web端），Refresh Token 存储在 httpOnly Cookie
- Token 刷新失败时（Refresh Token 过期或失效），清除本地 Token，跳转登录页

**Token 自动刷新时机**：
- Access Token 有效期 2 小时，过期前 5 分钟自动刷新
- 前端每次请求前检查 Token 剩余有效期，小于 5 分钟时触发刷新
- 刷新期间新请求排队等待，刷新完成后使用新 Token 重发
- 并发请求只触发一次刷新，避免多次刷新冲突

### 1.4 密码找回

| 功能点 | 功能描述 |
|-------|---------|
| 邮箱找回 | 输入邮箱 + 图形验证码 → 发送邮箱验证码 → 设置新密码 |

**功能逻辑**：
- 账号不存在时返回统一提示"验证码已发送"（防止账号枚举）
- 重置令牌有效期 15 分钟
- 验证码有效期 5 分钟
- 新密码不能与最近3次历史密码相同
- 密码重置成功后，所有设备强制退出，需重新登录

### 1.5 退出登录

| 功能点 | 功能描述 |
|-------|---------|
| 退出当前设备 | 退出当前设备登录状态，清除本地 Token |
| 退出所有设备 | 退出所有设备登录状态（跳转到1.2.1设备管理操作） |

**功能逻辑**：
- 退出时 Access Token 加入 Redis 黑名单
- 退出所有设备时，所有设备的 Token 加入黑名单，Refresh Token 同时失效

### 1.6 账号注销

| 功能点 | 功能描述 |
|-------|---------|
| 注销条件检查 | 检查是否有未完成的考试、进行中的班级、未处理的角色申请等 |
| 申请注销 | 用户申请注销账号，需验证身份（密码/邮箱验证码） |
| 数据导出 | 注销前可导出个人数据（答题记录、错题本、收藏等） |
| 注销确认 | 注销前提示数据将被清除，需二次确认 |
| 注销处理 | 账号标记为已注销，30天内可撤销，30天后永久删除 |

**注销条件**：
- 无进行中的考试（答题中/待阅卷）
- 无未处理的角色申请
- 作为教师的班级已转让或解散
- 无未完成的订单（如有付费功能）

**功能逻辑**：
- 注销后账号不可登录
- 30天冷静期内可联系客服撤销
- 30天后数据永久删除，不可恢复

### 1.7 图形验证码

| 功能点 | 功能描述 |
|-------|---------|
| 获取验证码 | 返回 Base64 编码的验证码图片 + captchaKey |
| 验证校验 | 后续请求携带 captchaKey + 用户输入的验证码 |
| 验证码类型 | 支持数字验证码、字母验证码、算术验证码（如 3+5=?） |
| 安全级别 | 随机干扰线、噪点、字符扭曲，防止OCR识别 |

**功能逻辑**：
- 验证码有效期 5 分钟
- 验证失败后自动刷新验证码
- 同一 captchaKey 只能验证一次
- 验证码字符忽略大小写

### 1.8 邮箱验证码限制

| 功能点 | 功能描述 |
|-------|---------|
| 发送频率限制 | 同一邮箱：60秒内只能发送1次 |
| 每日上限 | 同一邮箱：每天最多发送10次 |
| 验证次数限制 | 同一验证码最多验证5次，超过后需重新发送 |
| IP限制 | 同一IP每小时最多发送50次验证码 |

**功能逻辑**：
- 超过频率限制返回对应错误码和剩余等待时间
- 验证码有效期 5 分钟
- 超过验证次数后验证码失效

### 1.9 第三方账号绑定

| 功能点 | 功能描述 |
|-------|---------|
| 绑定微信 | 绑定微信账号，用于微信一键登录 |
| 绑定GitHub | 绑定GitHub账号，用于GitHub登录 |
| 绑定Google | 绑定Google账号，用于Google登录 |
| 解绑账号 | 解绑已绑定的第三方账号（需验证身份） |
| 绑定状态 | 查看已绑定的第三方账号列表 |

**绑定规则**：
- 一个第三方账号只能绑定一个本地账号
- 绑定时如该第三方账号已绑定其他账号，提示冲突
- 解绑时需验证密码或邮箱验证码
- 解绑后无法使用该第三方账号登录

## 二、用户管理系统

### 2.1 用户信息管理

#### 2.1.1 基本信息

| 功能点 | 功能描述 |
|-------|---------|
| 个人信息查看 | 查看头像、昵称、真实姓名、性别、邮箱、手机号、简介 |
| 个人信息修改 | 修改昵称、头像、性别、生日、简介等 |
| 头像上传 | 上传头像，支持jpg/png格式，大小不超过2MB，自动裁剪为正方形 |
| 角色信息 | 查看当前用户拥有的角色列表 |
| 班级创作者状态 | 查看在各班级的班级创作者状态（已申请/已通过/已驳回） |

**信息修改规则**：
- 昵称：2-20个字符，支持中英文、数字、下划线，敏感词过滤
- 简介：最多200个字符
- 用户名：注册后不可修改
- 真实姓名：实名认证后不可修改（需联系管理员）

#### 2.1.2 账号安全

| 功能点 | 功能描述 |
|-------|---------|
| 修改密码 | 输入旧密码验证后设置新密码 |
| 绑定邮箱 | 绑定或更换邮箱，需验证新邮箱 |
| 绑定手机号 | 绑定或更换手机号，需验证新手机号（已实名认证的需验证原手机号） |
| 第三方账号管理 | 查看已绑定的第三方账号，支持绑定/解绑 |
| 登录日志 | 查看最近登录记录（时间、IP、设备），支持查看异常登录 |

**密码修改规则**：
- 需输入旧密码验证
- 新密码不能与最近 3 次历史密码相同
- 密码修改成功后，所有设备强制退出，需重新登录
- 修改成功后发送站内消息 + 邮箱通知

**绑定规则**：
- 邮箱：绑定或更换需验证新邮箱验证码
- 手机号：绑定或更换需验证新手机号（已实名认证的需验证原手机号）
- 第三方账号：详见 **1.9 第三方账号绑定**

#### 2.1.3 隐私设置

| 设置项 | 可选值 | 说明 |
|-------|-------|------|
| 个人主页可见性 | 所有人/仅班级成员/仅自己 | 控制其他用户是否能访问个人主页 |
| 真实姓名可见性 | 所有人/仅班级成员/仅自己 | 控制真实姓名是否对外显示 |
| 手机号可见性 | 仅自己 | 手机号默认不对外显示 |
| 邮箱可见性 | 所有人/仅班级成员/仅自己 | 控制邮箱是否对外显示 |
| 学习统计可见性 | 所有人/仅班级成员/仅自己 | 控制学习数据是否对外显示 |
| 班级信息可见性 | 所有人/仅班级成员/仅自己 | 控制加入的班级是否对外显示 |

**隐私设置规则**：
- 默认值：手机号仅自己，其他为所有人
- 班级成员可查看：加入同一班级的成员可查看
- 仅自己：只有本人可查看

### 2.2 实名认证

| 功能点 | 功能描述 |
|-------|---------|
| 绑定手机号 | 绑定手机号，接收短信验证码验证（如未绑定） |
| 填写实名信息 | 填写真实姓名、身份证号 |
| 认证提交 | 提交认证，系统自动验证身份信息格式 |
| 认证状态查看 | 查看认证状态（未认证/已认证） |

**认证规则**：
- 申请创作者/教师角色前必须完成实名认证
- 实名认证通过手机号验证码+身份信息格式验证，无需人工审核
- 认证通过后姓名和身份证号不可修改，如需修改需联系管理员
- 一个身份证号只能认证一个账号
- 一个手机号只能绑定一个账号
- 绑定的手机号即为账号安全手机，用于找回密码和登录

**身份证校验规则**：
- 必须为18位身份证号码
- 校验前17位为数字，第18位为数字或X
- 校验位算法验证（ISO 7064:1983.MOD 11-2）
- 校验出生日期合法性
- 校验地址码合法性

### 2.3 角色申请

#### 2.3.1 申请成为创作者

| 功能点 | 功能描述 |
|-------|---------|
| 申请入口 | 用户可在个人中心申请成为创作者 |
| 申请条件 | 必须完成实名认证 |
| 申请信息 | 填写申请理由、擅长领域、创作经验说明 |
| 申请状态 | 查看申请状态（见下方状态机） |
| 审核结果 | 通过/驳回，驳回需填写原因 |

**申请流程**：
```
用户申请 → 填写信息 → 提交审核 → 管理员审核 → 通过 → 获得创作者角色
                              ↓
                            驳回 → 修改信息重新提交
```

#### 2.3.2 申请成为教师

| 功能点 | 功能描述 |
|-------|---------|
| 申请入口 | 用户可在个人中心申请成为教师 |
| 申请条件 | 必须完成实名认证 |
| 申请信息 | 填写申请理由、教学经验、所属机构/学校 |
| 申请状态 | 查看申请状态（见下方状态机） |
| 审核结果 | 通过/驳回，驳回需填写原因 |

**申请流程**：
```
用户申请 → 填写信息 → 提交审核 → 管理员审核 → 通过 → 获得教师角色
                              ↓
                            驳回 → 修改信息重新提交
```

#### 2.3.3 申请状态机

**状态定义**：

| 状态 | 说明 |
|------|------|
| pending（待审核） | 已提交申请，等待管理员审核 |
| approved（已通过） | 审核通过，已获得角色 |
| rejected（已驳回） | 审核驳回，可修改后重新提交 |
| revoked（已撤销） | 管理员撤销角色，需重新申请 |

**状态流转**：
```
未申请 → pending（提交申请）
pending → approved（审核通过）
pending → rejected（审核驳回）
approved → revoked（管理员撤销角色）
rejected → pending（重新提交申请）
revoked → pending（重新申请）
```

**状态流转规则**：
- 同一角色同时只能有一个 pending 状态的申请
- 已通过的申请不可重复申请（如需重新申请需先被撤销）
- 驳回后可立即重新提交（无冷却期）
- 撤销后 30 天内不可重新申请（防止滥用）

#### 2.3.4 审核规则与资源影响

**审核规则**：
- 创作者/教师申请由管理员/超级管理员审核
- 审核通过后立即生效，用户获得对应角色
- 审核驳回需填写驳回原因，用户可查看
- 用户可同时申请创作者和教师角色
- 申请期间可正常使用学生功能

**角色撤销后资源归属**：

| 角色 | 撤销后资源处理 |
|------|--------------|
| 创作者 | 私有题目保留，仅创建者可见；已发布的公共题目不受影响 |
| 教师 | 班级保留（可指定新教师）；私有题目保留；班级考试/作业保留 |
| 班级创作者 | 班级资源保留，但不可再创建新资源；已有资源不可编辑 |

**用户禁用/注销后资源影响**：

| 场景 | 资源处理 |
|------|---------|
| 用户被禁用 | 用户不可登录；创建的私有题目不可见；班级资源保留可见；公共题目不受影响 |
| 用户账号注销 | 私有题目 30 天后物理删除；班级资源保留（归属班级）；公共题目保留（归属系统）；错题本数据删除 |
