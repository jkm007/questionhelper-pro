### 4.3 索引设计原则

- 主键：统一使用 `id BIGINT UNSIGNED AUTO_INCREMENT`
- 外键：应用层维护，不使用数据库外键约束
- 唯一索引：用户名、邮箱、手机号等业务唯一字段
- 普通索引：高频查询字段（status、category_id、created_at 等）
- 复合索引：多条件组合查询（如 user_id + status）
- 软删除：使用 `deleted_at DATETIME NULL` 字段

## 五、缓存设计

### 5.1 Redis 使用场景

| 场景 | Key 格式 | 数据结构 | TTL | 说明 |
|------|---------|---------|-----|------|
| Token 黑名单 | `qh:token:bl:{jti}` | String | Token 剩余有效期 | 退出登录后 Token 失效 |
| 考试会话 | `qh:exam:session:{uid}:{eid}` | Hash | 考试时长 | 考试状态、题目顺序 |
| 用户答案 | `qh:exam:answers:{uid}:{eid}` | Hash | 考试时长 | 用户答题缓存 |
| 考试分值 | `qh:exam:scores:{uid}:{eid}` | Hash | 考试时长 | 每题分值 |
| 考试延迟队列 | `qh:exam:delay` | ZSet | - | Score=交卷时间戳 |
| 图形验证码 | `qh:captcha:{key}` | String | 5 分钟 | 验证码存储 |
| 短信验证码 | `qh:sms:{phone}` | String | 5 分钟 | 验证码存储 |
| 短信频率限制 | `qh:sms:limit:{phone}` | String | 60 秒 | 防止频繁发送 |
| 每日短信限制 | `qh:sms:daily:{phone}` | String | 24 小时 | 每日上限 |
| IP 登录限制 | `qh:login:limit:{ip}` | String | 1 小时 | IP 级别限流 |
| 用户登录失败 | `qh:login:fail:{uid}` | String | 15 分钟 | 账号锁定 |
| 热门题目 | `qh:hot:questions` | ZSet | 1 小时 | 热度排行 |
| 用户会话 | `qh:user:session:{uid}` | Hash | 7 天 | 用户在线状态 |

## 六、消息队列

### 6.1 使用场景

| 场景 | 说明 | 优先级 |
|------|------|-------|
| 考试超时交卷 | 延迟队列，考试结束自动交卷 | 高 |
| 通知推送 | 异步发送站内消息、短信、邮件 | 中 |
| 数据统计 | 异步更新统计数据 | 低 |
| 文件处理 | 异步处理图片压缩、缩略图生成 | 低 |

### 6.2 方案选型

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| Redis ZSet | 延迟队列（考试超时） | 轻量级，适合简单延迟任务 |
| Redis Stream | 消息队列（通知推送） | Redis 5.0+ 原生支持 |
| Go Channel | 进程内异步 | 轻量级，无需外部依赖 |

**建议**：
- 考试超时交卷：使用 Redis ZSet 延迟队列
- 通知推送：使用 Redis Stream 或 Go Channel
- 如需更复杂的消息路由，可引入 RabbitMQ

## 十一、安全方案

### 11.1 接口安全

| 措施 | 技术方案 |
|------|---------|
| 认证 | JWT + Redis Token 黑名单 |
| 授权 | 自定义 RBAC 权限中间件 |
| 限流 | golang.org/x/time/rate（令牌桶） |
| 防重放 | 请求签名 + 时间戳 + Nonce |
| CORS | gin-contrib/cors 白名单配置 |

**接口权限实现**：

| 环节 | 方案 |
|------|------|
| 权限标识 | 路由定义时绑定 permission 标识（如 `user:view`） |
| 中间件校验 | 请求到达时，中间件从 Token 解析用户角色，查询角色权限列表 |
| 权限匹配 | 判断用户权限列表是否包含当前接口的 permission 标识 |
| 缓存优化 | 用户权限列表缓存到 Redis，避免每次请求查库 |

**接口权限中间件流程**：
```
请求到达 → JWT 中间件解析 Token → 获取用户 ID
    → 从 Redis 缓存读取用户权限列表
    → 匹配当前接口的 permission 标识
    → 有权限 → 继续处理
    → 无权限 → 返回 403 Forbidden
```

### 11.2 数据权限

| 场景 | 方案 |
|------|------|
| 公共资源 | 所有用户可见 |
| 私有资源 | 仅创建者可见（WHERE created_by = ?） |
| 班级资源 | 仅班级成员可见（WHERE class_id IN 用户加入的班级） |
| 管理员 | 可查看所有数据 |

**数据权限实现**：
- Repository 层根据用户角色和资源类型自动拼接查询条件
- 公共资源：无条件限制
- 私有资源：`AND created_by = ?`
- 班级资源：`AND class_id IN (?)`

### 11.3 资源权限

| 资源类型 | 可见范围 | 创建权限 |
|---------|---------|---------|
| 公共题目 | 所有用户 | 内容管理员/管理员 |
| 私有题目 | 仅创建者 | 创作者/教师 |
| 班级题目 | 班级成员 | 教师/班级创作者 |

### 11.4 数据安全

| 措施 | 技术方案 |
|------|---------|
| 密码加密 | bcrypt（cost=12） |
| SQL 注入 | GORM 参数化查询 |
| XSS 防护 | bluemonday HTML 消毒 |
| CSRF 防护 | SameSite Cookie + Token 验证 |
| 敏感数据 | 手机号、身份证号脱敏存储/展示 |

### 11.5 内容安全

| 措施 | 技术方案 |
|------|---------|
| 敏感词过滤 | go-sensitive / 自定义敏感词库（DFA 算法） |
| 图片审核 | 阿里云内容安全 / 腾讯云天御 |
| 文本审核 | 阿里云内容安全 / 自建关键词库 |

### 11.6 防作弊技术

| 措施 | 实现方式 |
|------|---------|
| 题目乱序 | 服务端生成随机顺序，存入 Redis |
| 选项乱序 | 服务端生成随机顺序，存入 Redis |
| 服务端倒计时 | Redis TTL + 延迟队列，不依赖客户端时间 |
| 切屏检测 | Page Visibility API + 前端埋点上报 |
| 答题行为分析 | 记录答题时间、切换次数等异常指标 |

## 十二、API 版本管理

### 12.1 版本策略

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| URL 路径版本 | `/api/v1/users` | 推荐，清晰直观 |

### 12.2 版本兼容

- 版本变更通过 API 文档和变更日志通知
- 重大变更（Breaking Change）必须升级主版本号

```json
  "message": "success",
  "data": {}
}
```

**错误码规范**：
- `"00000"`：成功
- `"Axxx"`：参数/认证错误
- `"Bxxx"`：业务逻辑错误
- `"Sxxx"`：服务器内部错误

## 十六、运维监控

### 16.1 监控方案

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 指标采集 | Prometheus | 系统指标、业务指标采集 |
| 可视化 | Grafana | 监控仪表盘、图表展示 |
| 告警 | Alertmanager | 告警规则、通知渠道 |
| 日志 | Loki + Promtail | 轻量级日志收集 |
| 链路追踪 | Jaeger / OpenTelemetry | 分布式链路追踪（可选） |

### 16.2 监控指标

| 类型 | 指标 |
|------|------|
| 系统指标 | CPU、内存、磁盘、网络 |
| 应用指标 | QPS、响应时间、错误率、并发数 |
| 数据库指标 | 连接数、慢查询、QPS |
| Redis 指标 | 内存使用、命中率、连接数 |
| 业务指标 | 在线用户数、考试进行数、注册数 |

### 16.3 告警规则

| 场景 | 告警条件 |
|------|---------|
| 服务宕机 | 实例不可用 > 1 分钟 |
| 高延迟 | P99 响应时间 > 3 秒 |
| 高错误率 | 5xx 错误率 > 1% |
| 磁盘空间 | 磁盘使用率 > 85% |
| 内存使用 | 内存使用率 > 90% |
| 慢查询 | 慢查询数量 > 10/分钟 |
