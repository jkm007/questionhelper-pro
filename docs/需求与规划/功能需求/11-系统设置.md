### 11.2 安全设置

| 功能点 | 功能描述 |
|-------|---------|
| 登录保护 | 开启/关闭登录时需邮箱验证码验证 |
| 新设备登录保护 | 开启/关闭新设备首次登录需验证 |
| 登录通知 | 开启/关闭登录时发送站内消息通知（默认开启） |
| 安全日志 | 查看账号安全相关操作日志（登录、修改密码、绑定/解绑等） |

**安全通知方式**：
- 站内消息：所有安全事件默认发送站内消息
- 邮箱通知：新设备登录、密码修改、账号注销等高危操作发送邮件

**通知触发场景**：
- 新设备首次登录 → 站内消息 + 邮箱
- 密码修改成功 → 站内消息 + 邮箱
- 第三方账号绑定/解绑 → 站内消息
- 账号注销申请 → 站内消息 + 邮箱
- 角色申请审核结果 → 站内消息

**安全响应头**：

| 响应头 | 值 | 说明 |
|--------|------|------|
| X-Content-Type-Options | nosniff | 禁止 MIME 类型嗅探 |
| X-Frame-Options | DENY | 禁止 iframe 嵌入 |
| Strict-Transport-Security | max-age=31536000 | 强制 HTTPS |
| Content-Security-Policy | default-src 'self' | 限制资源加载来源 |

**错误信息规范**：
- 生产环境不返回堆栈信息、SQL 语句、文件路径等敏感信息
- 统一返回用户友好的错误提示
- 详细错误信息仅记录到日志，不返回给客户端

### 11.4 权限管理

| 功能点 | 功能描述 |
|-------|---------|
| 权限树查看 | 查看权限树结构，权限类型包括 menu（菜单）、api（接口）、button（按钮） |
| 创建权限 | 创建权限节点，设置编码、名称、类型、父权限、路由/接口路径、HTTP 方法 |
| 编辑权限 | 修改权限信息 |
| 删除权限 | 删除权限节点 |
| 角色授权 | 为角色分配权限，支持菜单权限、接口权限、按钮权限 |

**权限粒度**：
- 菜单权限：控制用户可见的菜单
- 按钮权限：控制用户可操作的按钮
- 接口权限：控制用户可访问的 API 接口
- 数据权限：控制用户可访问的数据范围（本部门/本部门及以下/全部）
- 资源权限：控制用户可访问的资源类型（公共资源/私有资源/班级资源）

**权限编码规范**：`模块:子模块:操作`，如 `question:public:view`、`exam:paper:create`、`user:list:delete`。三级编码规范确保权限粒度足够细，子模块用于区分同一模块下的不同资源类型。

**班级创作者权限**：
- 班级创作者是一种特殊权限，通过申请审批获得
- 申请流程：班级成员申请 → 教师审批 → 通过后获得班级创作者权限
- 权限范围：在班级内创建题目、分类、知识点，仅班级成员可见
- 权限撤销：教师可撤销成员的班级创作者权限
- 详细管理功能见 **5.3 班级创作者管理**

**权限分配约束**：
- 分配者的权限必须包含被分配的权限（不能分配自己没有的权限）
- 超级管理员拥有所有权限，不受此约束
- 角色分配时校验分配者权限 >= 被分配角色的所有权限之和

#### 11.5.5 创作者/教师资源使用规则

**使用公共资源**：
- 创作者/教师可浏览所有公共分类、公共知识点、公共题目
- 创作者/教师可使用公共题目创建私有试卷
- 教师可使用公共题目创建班级考试

**使用私有资源**：
- 创作者/教师只能查看和使用自己创建的私有资源
- 私有资源不影响公共题库

**使用班级资源**：
- 教师可查看和使用自己班级的所有资源
- 班级创作者可查看和使用所在班级的所有资源
- 普通班级成员只能查看班级资源，不能创建

### 11.6 日志与审计

#### 11.6.1 操作审计

| 功能点 | 功能描述 |
|-------|---------|
| 审计日志查看 | 分页查看操作日志，支持按模块、操作类型、操作人、时间范围筛选 |
| 记录内容 | 记录操作人、操作类型、资源类型、资源ID、IP 地址、客户端信息、操作详情 |
| 覆盖范围 | 登录、创建题目、修改分类、组卷、阅卷、删除等关键操作。管理员查看他人私有资源、管理员重置用户密码、管理员分配/撤销角色等敏感操作必须记录审计日志。 |
| 班级创作者申请日志 | 查看班级创作者申请和审批记录 |

**管理员资源访问审计**：管理员查看他人私有资源、班级资源时，必须记录审计日志，包含操作人、资源类型、资源 ID、访问时间。

**审计日志安全要求**：
- 审计日志写入后不可修改和删除
- 日志保留期限不少于 180 天
- 覆盖所有敏感操作（登录、权限变更、数据删除、用户管理）
