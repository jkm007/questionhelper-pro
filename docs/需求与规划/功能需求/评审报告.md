## 设计亮点

1. **统一的 API 响应格式**：`{"code": "00000", "msg": "success", "data": {...}}` 全局一致
2. **JWT + Redis 黑名单方案**：Access/Refresh Token 分离设计合理，黑名单 TTL 自动过期
3. **三级防作弊设计**：无/基础/严格三级可配置，覆盖常见作弊场景
4. **question_snapshot 快照机制**：考试开始时冻结题目内容，确保历史试卷一致性
5. **三种资源隔离模式**：公共/私有/班级三级可见范围设计清晰
6. **后台任务体系**：Redis Stream MQ + Job Worker + 定时 Task 三层异步架构
7. **完整的数据统计设计**：涵盖 DAU/MAU、留存、转化漏斗等多维度指标
8. **文件生命周期管理**：引用计数 + 孤立文件清理 + 软删除联动
9. **敏感词过滤中间件**：通用中间件设计，可应用于多种内容场景
10. **WebSocket Hub 设计**：按用户 ID 管理多设备连接，心跳检测机制完整

## 下一步建议

### 第一阶段：阻塞性问题修复（1-2 周）

1. **统一数据模型定义**：召开跨模块协调会，明确角色枚举、is_correct 语义、时间字段归属、权限编码规范。输出统一的《数据字典》文档。
2. **补齐核心 API 设计**：重点补充练习 session API（H3）、作业管理 API（H5）、考试逐题详情 API（H4）、错题来源导航 API（H2）。
3. **修复移动端 API 路径不匹配**（M-H1）和 **Token 刷新机制**（M-H2），这两项导致移动端核心功能不可用。
4. **补充安全基线要求**：HTML 净化、文件类型校验、全局限流、CORS 策略、Cookie 安全标志。输出《安全开发规范》。

### 第二阶段：流程闭环与安全加固（2-4 周）

5. **完善业务流程闭环**：考试暂停/恢复状态机、成绩复核流程、作业与考试边界定义、成员审核流程统一。
6. **安全加固**：防作弊增强（复制粘贴禁止、全屏模式）、考试答案延迟公布、敏感数据加密存储、审计日志防篡改。
7. **移动端离线/弱网处理**：答案本地缓存、请求重试机制、考试倒计时服务端校准。
8. **第三方集成第一阶段**：实现邮件发送（SMTP），支撑注册验证码和安全通知。

### 第三阶段：架构优化与体验提升（4-8 周）

9. **技术架构优化**：MQ 增加重试/死信、WebSocket 跨实例分发（Redis Pub/Sub）、数据库高频表分区、热数据缓存层。
10. **移动端平台适配**：微信小程序登录、推送通知对接、分享功能、文件上传平台差异处理。
11. **文件存储增强**：音视频格式支持、大文件分片上传、缩略图生成、文件去重。
12. **补充低优先级 API**：收藏/分享、隐私设置、学习计划、班级二维码等。
