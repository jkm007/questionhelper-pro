## 六、刷题练习系统

### 6.1 资源访问规则

**公共资源访问**：
- 所有登录用户可浏览公共分类树和公共题目
- 所有用户可使用公共题目进行练习和考试

**私有资源访问**：
- 创作者/教师可浏览自己创建的私有分类和题目
- 创作者/教师可使用私有题目进行练习

**班级资源访问**：
- 班级成员可浏览班级内的分类和题目
- 班级成员可使用班级题目进行练习和考试

**练习与错题本关联**：
- 练习答错的题目自动收录到错题本（来源标记为 practice）
- 错题重练模式从错题本中选题练习
- 详见 **七、错题本系统 7.1 错题收集**

> 关联章节：**三、题库管理系统**（题目来源）、**四、考试管理系统 4.3**（考试答题）、**七、错题本系统**（错题管理）

### 6.3 练习系统

#### 6.3.1 练习模式

| 模式 | 说明 | 出题方式 |
|------|------|---------|
| 随机练习 | 按条件随机出题 | 按分类/知识点/难度/标签/数量随机抽取 |
| 顺序练习 | 按顺序逐题练习 | 按题目编号顺序 |
| 错题重练 | 从错题本中选题练习 | 从错题本中选取 |
| 收藏练习 | 从收藏中选题练习 | 从收藏夹中选取 |
| 模拟考试 | 模拟真实考试环境 | 限时、计分，模拟考试流程 |
| 闯关模式 | 逐关挑战 | 按难度递进出题 |
| 计时模式 | 限时答题 | 限定时间内尽可能多答题 |

**出题规则**：
- 父分类包含子分类：选择父分类时自动包含子分类题目
- 资源来源：练习时可选择公共资源、私有资源或班级资源
- 题目数量自定义：自定义每次练习的题目数量
- 题目难度自适应：根据用户水平自动调整题目难度

#### 6.3.2 练习功能

| 功能点 | 功能描述 |
|-------|---------|
| 即时评判 | 选择题点击选项即时判断对错，显示解析 |
| 自动保存 | 每答一题自动保存进度 |
| 练习暂停/继续 | 暂停练习，下次继续 |
| 练习记录 | 记录每次练习的答题结果，通过 session_id 聚合同一批次 |
| 错题自动加入 | 答错的题目自动加入错题本 |

**错题收录规则**：
- 客观题答错：自动加入错题本，记录来源为 practice
- 主观题：用户自判后标记为错的，自动加入错题本
- 详见 **七、错题本系统 7.1 错题收集**

#### 6.3.3.1 练习 Session API

| API | 方法 | 说明 |
|-----|------|------|
| /api/v1/practices/start | POST | 开始练习（返回 session_id 和题目列表） |
| /api/v1/practices/:sessionId/save-answer | POST | 保存单题答案（幂等） |
| /api/v1/practices/:sessionId/pause | POST | 暂停练习（保存进度） |
| /api/v1/practices/:sessionId/resume | POST | 恢复练习（返回当前进度） |
| /api/v1/practices/:sessionId/submit | POST | 提交练习（批量保存并结算） |
| /api/v1/practices/:sessionId/status | GET | 获取练习状态和进度 |

**Session 生命周期**：
1. 调用 start → 创建 session，返回 session_id 和随机/顺序出题结果
2. 每答一题调用 save-answer → 答案写入 DB，即时评判
3. 可随时调用 pause 暂停，调用 resume 恢复
4. 调用 submit 结束练习 → 更新 session 统计，触发错题收录
5. session 超过 24 小时未操作自动结束

**练习记录查询与导出**：

| API | 方法 | 说明 |
|-----|------|------|
| /api/v1/practices/records | GET | 练习记录列表（支持按时间、分类、正确率筛选） |
| /api/v1/practices/records/export | GET | 导出练习记录为 Excel |
| /api/v1/practices/records/search | GET | 搜索练习记录（按题目内容关键词） |

#### 6.3.4 练习数据结构

**session_id 设计**：
- UUID，同一次连续练习共享，便于按批次查询和统计
- 练习暂停时保存 session_id，继续时恢复
- 建议对超过 90 天的练习记录定期归档

**练习记录字段**：
- session_id：练习会话 ID
- user_id：用户 ID
- question_id：题目 ID
- user_answer：用户答案
- is_correct：是否正确（0=错误、1=正确；练习模式中用户自判的主观题使用 0 或 1，不使用 2（待阅卷））
- answer_time：答题时间
- source：来源（practice/exam）

**练习记录持久化策略**：
- 每答一题立即写入 practice_records 表（幂等覆盖，同一 session + question_id 仅保留最新答案）
- session 结束时更新 session 统计信息（总题数、正确率、用时）
- 建议对超过 90 天的练习记录定期归档到 practice_records_archive 表
- 数据统计模块从 practice_records 表聚合数据，详见 **十、数据统计系统 10.3**
