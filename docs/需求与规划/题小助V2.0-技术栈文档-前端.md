## 四、动态菜单方案

> 菜单通过后端接口动态管理，前端根据用户角色动态加载

### 4.1 技术实现

| 环节 | 方案 |
|------|------|
| 后端菜单管理 | GORM 模型 + 无限层级树（parent_id + path） |
| 菜单缓存 | Redis 缓存菜单数据，修改后自动刷新 |
| 前端菜单加载 | 登录后调用 `/api/user/menus` 获取菜单树 |
| 动态路由 | Vue Router 动态添加路由（router.addRoute） |
| 按钮权限 | v-permission 指令 + 权限标识控制 |

### 4.2 前端动态菜单流程

```
用户登录 → 获取用户信息和角色 → 调用菜单接口 → 后端返回菜单树（含按钮权限）
    → 前端动态生成路由 → 渲染侧边栏菜单 → 存储按钮权限列表 → v-permission 控制按钮
```

### 4.3 按钮权限实现

| 环节 | 方案 |
|------|------|
| 后端存储 | 菜单类型为 button，permission 字段存储权限标识 |
| 接口返回 | 菜单接口同时返回按钮权限列表（permission 标识数组） |
| 前端存储 | Pinia store 存储按钮权限列表 |
| 前端控制 | v-permission 指令判断按钮是否显示 |

### 4.4 v-permission 指令实现

```typescript
// directive/permission.ts
export const vPermission = {
  mounted(el: HTMLElement, binding: DirectiveBinding) {
    const permissions = usePermissionStore().permissions
    const value = binding.value
    if (!permissions.includes(value)) {
      el.parentNode?.removeChild(el)
    }
  }
}
```
