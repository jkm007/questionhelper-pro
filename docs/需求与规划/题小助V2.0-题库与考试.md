## 三、题库管理系统

### 3.1 题目管理

#### 3.1.1 资源类型与权限

**资源类型**：

| 类型 | 创建者 | 可见范围 | 存储位置 |
|------|--------|---------|---------|
| 公共题目 | 内容管理员/管理员 | 所有用户 | 公共题库 |
| 私有题目 | 创作者/教师 | 仅创建者 | 个人空间 |
| 班级题目 | 教师/班级创作者 | 班级成员 | 班级空间 |

**角色权限矩阵**：

| 操作 | 超级管理员 | 管理员 | 内容管理员 | 教师 | 创作者 | 班级创作者 | 学生 |
|------|-----------|-------|-----------|------|-------|-----------|------|
| 查看公共题目 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 创建公共题目 | ✓ | ✓ | ✓ | - | - | - | - |
| 编辑公共题目 | ✓ | ✓ | ✓ | - | - | - | - |
| 删除公共题目 | ✓ | ✓ | ✓ | - | - | - | - |
| 审核公共题目 | ✓ | ✓ | ✓ | - | - | - | - |
| 创建私有题目 | ✓ | ✓ | - | ✓ | ✓ | - | - |
| 编辑私有题目 | ✓ | ✓ | - | ✓（自己的） | ✓（自己的） | - | - |
| 创建班级题目 | ✓ | ✓ | - | ✓（自己的班级） | - | ✓（自己的班级） | - |
| 提交公共审核 | ✓ | ✓ | - | ✓ | ✓ | - | - |
| 查看题目统计 | ✓ | ✓ | ✓ | ✓（自己的） | ✓（自己的） | - | - |

**资源互转规则**：
- 私有题目 → 公共题目：需提交审核，通过后变为公共题目
- 公共题目 → 私有题目：不支持（公共题目不可转为私有）
- 私有题目 → 班级题目：不支持（需在班级内重新创建）
- 班级题目 → 公共题目：不支持（班级资源仅限班级内使用）

#### 3.1.2 题目功能

| 功能点 | 功能描述 |
|-------|---------|
| 题目列表 | 分页展示，支持按分类树节点、难度（1-3：1=简单、2=中等、3=困难）、题型、状态、标签、关键词筛选；选择父分类时自动包含子分类题目 |
| 题目排序 | 支持按创建时间、难度、使用次数、正确率排序 |
| 题目搜索 | 支持题干内容全文搜索、题目编号精确搜索 |

| 编辑题目 | 修改已有题目的题型、主客观标记、分类、难度、内容、解析、标签、知识点关联 |

| 题目复制 | 快速复制已有题目创建副本，可修改部分内容后保存为新题目 |
| 题目收藏 | 用户端可收藏重要题目，支持按收藏夹分类管理，便于复习 |

**题目附件管理**：
- 题目内容支持插入图片、音频、视频等附件
- 附件通过文件管理模块上传，存储到对应业务目录（question/）
- 图片自动压缩并生成缩略图
- 附件随题目软删除，定时清理孤立文件
- 详见 [十四、文件管理](题小助V2.0-学习与互动.md#十四)

**标签管理**：
- 标签为自由标签，用户输入时自动匹配已有标签
- 支持标签的创建、编辑、删除（删除时解除关联）
- 预置标签：易错题、高频考点、重点、难点
- 标签支持按颜色区分

#### 3.1.3 题目引用与保护

**引用检查**：
- 删除/下架题目前，检查是否被试卷引用
- 被引用的题目删除时提示："该题目已被 N 份试卷引用，确认删除？"
- 已用于考试快照的题目，快照内容不受影响

**题目使用统计**：

| 统计项 | 说明 |
|-------|------|
| 引用试卷数 | 被多少份试卷引用 |
| 练习次数 | 被多少人练习过 |
| 正确率 | 练习/考试中的整体正确率 |
| 收藏数 | 被多少人收藏 |
| 评论数 | 题目下的评论数量 |

**题目删除/下架影响**：

| 场景 | 影响 |
|------|------|
| 被试卷引用（未考试） | 删除后试卷中该题自动移除，总分重新计算 |
| 被试卷引用（已有考试记录） | 题目软删除，试卷中保留该题引用（只读），已有答题记录不受影响 |
| 已考试有快照 | 快照保留，历史记录不受影响 |
| 错题本中有该题 | 错题本保留题目快照，显示"题目已删除" |
| 评论区有评论 | 评论保留，显示"题目已删除" |
| 收藏夹中有该题 | 自动从收藏夹移除 |

**删除保护规则**：
- 题目被未完成的考试引用时，禁止删除（需先从试卷移除或结束考试）
- 题目被已有答题记录的考试引用时，仅允许软删除，不物理删除
- 公共题目删除需内容管理员/管理员权限
- 私有题目删除仅需创建者权限
- 班级题目删除需教师权限（班级创作者不可删除他人题目）

> 关联章节：**四、考试管理系统 4.1.2**（试卷与题目的关联）、[七、错题本系统 7.1](题小助V2.0-学习与互动.md#七)（错题收集）

#### 3.1.4 题型说明

| 题型 | 标识 | content 结构 | 评分方式 |
|------|------|-------------|---------|
| 单选题 | single_choice | title + options + answer="A" | 精确匹配 |
| 多选题 | multiple_choice | title + options + answer="A,B,D" | 精确匹配（全部选对才得分） |
| 判断题 | true_false | title + answer="true"/"false" | 精确匹配 |
| 填空题 | fill_blank | title（含___占位）+ answer + keywords | 客观：关键词子串匹配；主观：人工阅卷 |
| 简答题 | short_answer | title + answer（参考答案） | 人工阅卷 / 用户自判 |

**content JSON 结构**：
```json
{
  "title": "题干内容（支持 HTML）",
  "options": ["A. 选项一", "B. 选项二", "C. 选项三", "D. 选项四"],
  "answer": "A",
  "keywords": ["关键词1", "关键词2"]
}
```

**主客观标记**：`is_subjective` 字段，0=客观（自动评分）、1=主观（人工阅卷）

#### 3.1.5 题目状态

**状态流转**：
```
draft（草稿）→ published（已发布）→ archived（已下架）
                 ↑                        │
                 └──────── 恢复 ───────────┘
```

| 状态 | 用户端可见 | 管理端可见 | 可用于考试 | 可用于练习 |
|------|-----------|-----------|-----------|-----------|
| draft（草稿） | ✗ | ✓ | ✗ | ✗ |
| published（已发布） | ✓ | ✓ | ✓ | ✓ |
| archived（已下架） | ✗ | ✓ | ✗ | ✗ |

**状态变更规则**：
- 草稿 → 已发布：管理员手动发布或审核通过后自动发布
- 已发布 → 已下架：管理员手动下架
- 已下架 → 已发布：管理员手动恢复
- Excel 导入的题目默认 draft 状态，需管理员审核后发布
- 创作者/教师创建的私有题目默认 draft 状态，可自行发布

#### 3.2.1 导入功能

| 功能点 | 功能描述 |

| 导入进度显示 | 大文件导入时显示进度条（已处理行数/总行数） |
| 导入失败重试 | 部分失败后可修正数据，重新上传失败部分 |
| Word格式导入 | 支持从 Word 文档（.docx）导入题目，自动解析题目结构 |

**模板列**：题目序号 | 题目类型 | 题目内容 | 所属分类ID | 难度 | 是否主观 | 是否正确项 | 选项内容 | 选项解析 | 答案 | 关键词 | 解析

#### 3.2.2 导出功能

| 功能点 | 功能描述 |
|-------|---------|
| Excel 导出 | 按筛选条件导出题目为 Excel 文件 |
| 导出范围 | 支持导出当前筛选结果、选中题目、全部题目 |
| 导出内容 | 题目基本信息、选项、答案、解析、分类、知识点、标签 |
| 导出格式 | 支持 .xlsx 格式 |

#### 3.2.3 导入流程

```
上传文件 → 格式校验 → 解析内容 → 分组校验 → 查重 → 批量入库 → 返回报告
```

1. 校验文件格式（仅 .xlsx）和大小（< 10MB）
2. 从第 2 行开始解析（第 1 行为表头）
3. 按"题目序号"分组，同一序号的多行 = 一道题的多个选项
4. 逐组校验：题型合法、题目内容非空、分类 ID 存在、选择题至少一个正确选项
5. 内容 MD5 查重（跳过重复题目，记录到失败列表）
6. 构建 content JSON，批量 INSERT（事务）
7. 返回导入报告：`{total, success, failed, errors[{row, reason}]}`

#### 3.3.1 分类功能

| 功能点 | 功能描述 |
|-------|---------|
| 树形展示 | 分类以无限层级树表格展示，支持任意深度 |

| 编辑分类 | 修改分类名称、父级、排序；禁止移动到自身或自身子树下 |
| 删除保护 | 分类存在子分类、知识点或关联题目时禁止删除 |
| 树形选择器 | 题目管理、知识点管理、手动组卷、智能组卷统一使用树形分类选择器 |
| 分类合并 | 将两个分类合并为一个，关联题目自动归属到目标分类 |
| 分类图标/颜色 | 为分类设置图标和颜色，前端展示更直观 |
| 分类统计 | 显示各分类下的题目数量、难度分布、题型分布 |

**分类类型**：

| 类型 | 创建者 | 可见范围 | 说明 |
|------|--------|---------|------|
| 公共分类 | 内容管理员/管理员 | 所有用户 | 公共题库的分类体系 |
| 私有分类 | 创作者/教师 | 仅创建者 | 个人空间的分类 |
| 班级分类 | 教师/班级创作者 | 班级成员 | 班级内的分类 |

#### 3.3.2 数据结构与逻辑

**分类树数据结构**：
- `parent_id`：父级 ID（0=根节点）
- `level`：层级，根节点为 1，由后端自动计算
- `path`：节点路径，如 `/1/3/5/`，用于子树查询
- `sort_order`：同级排序权重
- `type`：分类类型（public/private/class）

**分类树维护逻辑**：
- 创建分类：level = parent.level + 1；path = 父 path + 当前 ID + "/"
- 移动分类：校验目标父节点存在 → 禁止 parent_id = 自身 ID → 禁止移动到自身后代节点下 → 更新 parent_id、level、path → 递归刷新所有后代节点 level、path
- 筛选题目：通过 path 查询该分类全部后代 ID，使用 `category_id IN (当前分类 + 后代分类)` 查询题目

### 3.5 内容审核

#### 3.5.1 审核功能

| 功能点 | 功能描述 |
|-------|---------|
| 审核流程 | 题目提交 → 审核 → 发布（通过/驳回） |
| 审核状态 | draft（草稿）→ pending（待审核）→ published（已发布）/ rejected（已拒绝） |
| 单题审核 | 审核通过或驳回，驳回需填写原因 |
| 批量审核 | 批量通过/批量驳回，返回成功/失败数量 |
| 审核记录 | 记录审核操作日志 |
| 审核超时提醒 | 待审核题目超过 24 小时未处理，自动提醒审核人员 |
| 审核意见回复 | 被驳回后创作者可与审核人沟通，支持多次回复直到达成一致 |
| 审核统计 | 统计审核通过率、驳回率、平均审核时长、各审核人员工作量 |

#### 3.5.2 审核流程

**公共题目审核流程**（内容管理员/管理员审核）：
```
创建题目(draft) → 发布(published)          # 管理员直接创建
创建题目(draft) → 提交审核(pending) → 审核 → 通过(published) / 驳回(rejected)
```

**创作者提交公共题目流程**：
1. 创作者/教师创建私有题目（默认 draft 状态）
2. 创作者/教师选择"提交为公共题目"
3. 题目状态变为 pending（待审核）
4. 内容管理员/管理员审核
5. 审核通过 → 题目变为公共题目，所有用户可见
6. 审核驳回 → 题目退回为私有状态，创作者可修改后重新提交

**班级题目审核流程**（教师审核）：
- 教师创建的班级题目：直接生效，无需审核
- 班级创作者创建的班级题目：可配置是否需要教师审核（默认不需要）
- 需要审核时：班级创作者提交 → 教师审核 → 通过/驳回

#### 3.5.3 审核权限

| 审核场景 | 审核人 | 说明 |
|---------|--------|------|
| 公共题目审核 | 内容管理员/管理员 | 创作者提交的公共题目 |
| 班级题目审核 | 教师（自己班级） | 班级创作者创建的班级题目 |
| 班级创作者申请审核 | 教师（自己班级）/ 管理员 | 申请成为班级创作者 |

> 创作者/教师的审核功能详见 [九、内容创作系统 9.5 内容审核](题小助V2.0-学习与互动.md#九)

#### 4.1.1 资源类型与权限

#### 4.1.3 智能组卷

**组卷规则**：

| 规则项 | 说明 |
|-------|------|
| 题型 | 指定题型（单选/多选/判断/填空/简答） |
| 分类 | 指定分类树节点，父分类自动包含子分类 |
| 难度 | 指定难度（1-3），或不限制 |
| 数量 | 指定抽取数量 |
| 分值 | 指定每题分值 |

**组卷逻辑**：
1. 创建 exams 记录
2. 维护已抽题 ID 集合 excludes = {}
3. 遍历 rules 数组，每条规则执行：
   - 如果传入 category_id，先计算分类子树 IDs（当前分类 + 全部后代分类）
   - `SELECT * FROM questions WHERE type=? AND deleted_at IS NULL AND status='published' AND (category_id IN categoryIDs) AND (difficulty=? OR 不限制) AND id NOT IN (excludes) ORDER BY RAND() LIMIT ?`
   - 将抽到的题 ID 加入 excludes
   - 如果实际抽到数量 < count：删除已创建试卷并返回错误
   - 写入 exam_rules 和 exam_questions（sort_order 递增，score 赋值，保存 question_snapshot）
4. 返回生成的试卷 ID 和题目数量

**question_snapshot**：试卷创建或考试开始时写入，记录题目完整内容（题干+选项顺序+正确答案），保证历史试卷一致性。即使原题目被修改，已考完的试卷展示仍保持一致。

#### 4.3.2 考试流程

```
查看考试 → 开始考试 → 答题过程 → 交卷 → 查看成绩
```

**详细流程**：
1. **查看试卷**：GET /api/exams/:id → 返回试卷基本信息 + 题型分布汇总（不含具体题目）
2. **开始考试**：POST /api/exams/:id/start
   - 校验：考试是否在开放时间内、用户是否超过 max_attempts、试卷是否存在题目
   - 执行：读取 exam_questions 及题目快照 → 创建 exam_records（status=in_progress） → 写入 Redis（exam:session + exam:scores） → 写入延迟队列（ZADD tixuan:exam:delay） → 返回 record_id、时长、总分、题量
3. **答题过程**：每答一题 POST /api/exams/:id/save-answer → 答案幂等写入 Redis Hash
4. **断线续考**：GET /api/exams/:id/resume → 从 Redis 读取 answers + session TTL 计算 remaining_sec
5. **交卷**：POST /api/exams/:id/submit 或延迟队列到期
   - 从 Redis 读取所有答案
   - 客观题自动评分，主观题标记待阅卷（is_correct=2）
   - 写入 exam_answers 表
   - 更新 exam_records（score, obj_score, status, duration_used）
   - 答错题写入/更新 wrong_questions 表
   - DEL 所有 Redis 考试相关 Key，ZREM 延迟队列成员
6. **查看成绩**：GET /api/exams/:id/record → 客观题即时展示，主观题待阅卷后展示

#### 4.3.3 Redis 会话设计

| Key | 说明 | TTL |
|-----|------|-----|
| `tixuan:exam:session:{uid}:{eid}` | 考试题目顺序/会话状态 | 考试时长 |
| `tixuan:exam:answers:{uid}:{eid}` | 用户答案缓存（Hash，幂等覆盖写入） | 考试时长 |
| `tixuan:exam:scores:{uid}:{eid}` | 本次考试每题分值 | 考试时长 |
| `tixuan:exam:delay`（ZSet） | 延迟队列，Score = 交卷时间戳，Member = uid:eid:attempt_no | - |

### 4.4 评分规则

#### 4.4.1 评分方式

| 评分方式 | 适用题型 | 实现逻辑 |
|----------|---------|---------|
| 精确匹配 | 单选、多选 | 用户答案与 answer 完全一致即得分 |
| 关键词子串匹配 | 填空题（客观） | 用户答案包含 keywords 中所有关键词即得分；忽略大小写和首尾空格 |
| 人工阅卷 | 填空题（主观）、简答题 | 管理员在后台查看答案并打分 |
| 部分得分 | 多选题 | 可配置全部选对才得分或多选漏选部分得分 |

**分数存储**：所有分数字段统一使用 DECIMAL(5,2)（最大 999.99 分），支持 0.5 分的小题分值。

| 成员审核 | 加入班级需要教师审核 |
| 成员统计 | 统计成员活跃度、参与度等 |

**加入逻辑**：
- 校验邀请码有效
- 校验班级成员未满
- 校验用户不在该班级中

**成员退出/移除后资源影响**：

| 场景 | 班级题目 | 班级考试记录 | 班级创作者权限 |
|------|---------|-------------|--------------|
| 成员主动退出 | 保留（归属班级） | 保留（个人记录不可见） | 自动撤销 |
| 教师移除成员 | 保留（归属班级） | 保留（个人记录不可见） | 自动撤销 |
| 成员重新加入 | 恢复可见 | 恢复可见 | 需重新申请 |

> 关联章节：**五、班级管理系统 5.3**（班级创作者管理）、**五、班级管理系统 5.4**（班级资源管理）

### 5.3 班级创作者管理

#### 5.3.1 班级创作者功能

| 功能点 | 功能描述 |
|-------|---------|
| 申请成为班级创作者 | 班级成员可申请成为班级创作者，填写申请理由 |
| 查看申请列表 | 教师可查看班级创作者申请列表，含申请人信息、申请理由、申请时间 |
| 教师审批 | 教师可审批班级创作者申请（通过/驳回），驳回需填写原因 |
| 管理员审批 | 管理员/超级管理员可审批所有班级的创作者申请 |
| 撤销班级创作者 | 教师可撤销成员的班级创作者权限 |
| 查看班级创作者列表 | 教师可查看班级内所有班级创作者 |
| 申请历史 | 查看历史申请记录（含已通过/已驳回） |
| 创作者权限配置 | 自定义班级创作者的权限范围（如：是否可创建考试） |
| 创作者统计 | 统计班级创作者的创建题目数、活跃度等 |
| 创作者等级 | 根据贡献度设置创作者等级 |

#### 5.3.2 申请流程

```
成员申请 → 教师审批 → 通过 → 成为班级创作者
                    ↓
                  驳回 → 退回申请（可重新申请）
```

**申请规则**：
- 班级成员均可申请成为班级创作者
- 一个用户可在多个班级申请成为班级创作者
- 申请时需填写申请理由
- 审批通过后立即生效

#### 5.3.3 权限与审批

**班级创作者权限**：

| 权限项 | 班级创作者 | 普通班级成员 |
|-------|-----------|-------------|
| 查看班级题目 | ✓ | ✓ |
| 创建班级题目 | ✓ | - |
| 创建班级分类 | ✓ | - |
| 创建班级知识点 | ✓ | - |
| 使用公共题目 | ✓ | ✓ |
| 使用班级题目 | ✓ | ✓ |
| 参加班级考试 | ✓ | ✓ |
| 查看班级成绩 | ✓ | ✓ |

**权限说明**：
- 班级创作者权限仅在申请的班级内有效
- 班级创作者创建的资源仅班级成员可见
- 班级创作者不影响公共题库
- 教师可随时撤销班级创作者权限

**审批规则**：
- 教师只能审批自己班级的班级创作者申请
- 管理员/超级管理员可审批所有班级的班级创作者申请
- 审批通过后立即生效，成员获得班级创作者权限
- 审批驳回需填写驳回原因，申请人可查看

### 5.4 班级资源管理

#### 5.4.1 资源功能

| 功能点 | 功能描述 |
|-------|---------|
| 班级题库 | 教师/班级创作者可为班级创建题目，仅班级成员可见 |
| 班级分类 | 教师/班级创作者可为班级创建分类，仅班级内使用 |
| 班级知识点 | 教师/班级创作者可为班级创建知识点，关联班级题目 |
| 使用公共资源 | 教师/班级创作者可使用公共题库的题目创建班级考试 |
| 使用班级资源 | 教师/班级创作者可使用班级内其他创作者的题目 |
| 资源查看 | 班级成员可查看班级内的所有题目、分类、知识点 |
| 资源统计 | 教师可查看班级资源统计（题目数、分类数、知识点数） |
| 资源导入 | 从公共题库导入题目到班级 |
| 资源导出 | 导出班级资源（题目、分类、知识点） |
| 资源审核 | 教师创建的资源直接可见；班级创作者创建的资源可配置是否需要教师审核（默认不需要） |
| 资源搜索 | 在班级资源库中搜索题目 |
| 资源标签 | 给资源打标签便于分类 |

#### 5.4.2 资源创建规则

**创建权限**：

| 角色 | 可创建资源 | 资源可见范围 |
|------|-----------|-------------|
| 教师 | 班级题目/分类/知识点 | 班级成员 |
| 班级创作者 | 班级题目/分类/知识点 | 班级成员 |
| 普通班级成员 | - | - |

**审核规则**：
- 教师创建的资源：直接生效，无需审核
- 班级创作者创建的资源：可配置是否需要教师审核（默认不需要）
- 需要审核时：班级创作者提交 → 教师审核 → 通过/驳回

**资源使用规则**：
- 班级资源仅班级成员可见，不影响公共题库
- 班级资源不能提交为公共资源
- 教师/班级创作者可使用公共题目创建班级考试

### 5.5 班级功能

#### 5.5.1 作业管理

| 功能点 | 功能描述 |
|-------|---------|
| 布置作业 | 教师为班级布置作业（关联试卷），设置标题、截止时间、说明 |

| 作业提醒 | 作业截止前提醒未提交的学生 |
| 作业模板 | 创建作业模板便于复用 |
| 学生互评 | 支持学生之间互相评价作业 |

**作业规则**：
- 作业关联试卷，学生需在截止时间前完成
- 教师可查看作业完成情况和统计
- 作业成绩计入班级成绩统计

#### 5.5.2 考试管理

| 功能点 | 功能描述 |

| 班级考试 | 教师发布班级考试，可选择公共题目或班级题目 |
| 查看成绩 | 查看班级整体成绩 |
| 成绩分析 | 班级成绩统计分析，支持班级间对比 |
| 班级排名 | 班级内成绩排名（可配置是否显示） |
| 成绩预警 | 成绩低于阈值时自动提醒学生 |
| 成绩导出 | 导出班级考试成绩为 Excel |

**班级考试规则**：
- 教师可选择公共题目或班级题目创建班级试卷
- 班级考试仅班级成员可见可考
- 班级考试成绩仅班级成员和教师可见
- 教师可查看班级考试成绩统计和分析
- 详见 **四、考试管理系统 4.2**

#### 5.5.3 班级互动

| 班级通知 | 班级相关通知推送（站内信） |
| 班级文件 | 存储和管理班级共享文件 |
| 班级日志 | 记录班级操作日志（成员变动、考试发布等） |

#### 5.5.4 学习管理

| 学习计划 | 制定学习计划并追踪完成情况 |
| 考勤管理 | 记录学生出勤情况（签到/签退） |
