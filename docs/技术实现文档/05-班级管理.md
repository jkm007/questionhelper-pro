## 一、模块概述

班级管理模块是题小助平台的核心教学组织单元，涵盖以下功能领域：

- **班级 CRUD**：创建、编辑、删除、归档、置顶、搜索、二维码、有效期管理
- **成员管理**：邀请码加入、成员审核、角色设置（学生/教师/管理员）、移除、转让、批量导入导出
- **班级创作者管理**：申请流程（成员申请 -> 教师审批 -> 通过/驳回）、权限矩阵、撤销创作者
- **班级资源管理**：班级题目/分类/知识点的创建与隔离、从公共题库导入、资源版本、标签、审核
- **作业管理**：布置作业、截止时间、提交、批改、迟交机制、作业模板、学生互评
- **班级考试**：发布考试到班级、班级内访问控制、成绩分析与排名
- **班级互动**：公告发布与通知推送、讨论帖与回复、共享文件管理
- **学习管理**：学习计划、任务跟踪、考勤管理、学习报告、成绩预警

**请求处理流程**：

```
客户端请求
    │
    ▼
Router (路由匹配, router/class.go)
    │
    ▼
Middleware 链
    ├── CORS 中间件
    ├── RateLimit 中间件 (令牌桶限流)
    ├── Auth 中间件 (JWT 解析 + 黑名单检查)
    ├── ClassMember 中间件 (检查是否为班级成员)
    └── OperationLog 中间件 (操作日志)
    │
    ▼
Controller (参数绑定 + 校验, controller/class/*.go)
    │
    ▼
Service (业务逻辑 + 权限校验 + 事务管理, service/class/*.go)
    │
    ▼
Repository (GORM 数据库操作, repository/class/*.go)
    │
    ▼
Model (数据库实体, model/class.go + model/class_ext.go)
```

## 五、数据库表

班级管理模块涉及以下核心数据表：

| 表名 | 说明 | 关键字段 | 关联关系 |
|------|------|----------|----------|
| `classes` | 班级主表 | id, name, description, cover, code (unique), creator_id (FK->users), member_count, status | 主表，被所有班级表引用 |
| `class_members` | 班级成员 | class_id (FK->classes), user_id (FK->users), role (1=学生/2=教师/3=管理员), joined_at | N:M 关联 classes 和 users |
| `class_notices` | 班级公告 | class_id (FK->classes), title, content, creator_id (FK->users) | N:1 关联 classes |
| `homeworks` | 作业 | class_id (FK->classes), title, description, deadline, creator_id (FK->users) | N:1 关联 classes |
| `homework_submissions` | 作业提交 | homework_id (FK->homeworks), user_id (FK->users), content, score, status, submitted_at | N:1 关联 homeworks，唯一索引 (homework_id, user_id) |
| `class_exams` | 班级考试 | class_id (FK->classes), exam_id (FK->exams), published_by, published_at | N:M 关联 classes 和 exams |
| `class_resources` | 班级资源 | class_id, resource_type, resource_id, creator_id | 多态关联，class_id 隔离资源 |
| `class_discussions` | 班级讨论 | class_id (FK->classes), user_id (FK->users), title, content, reply_count, is_pinned, last_reply_at | N:1 关联 classes |
| `class_discussion_replies` | 讨论回复 | discussion_id (FK->class_discussions), parent_id, user_id, content, reply_to_user_id | 树形结构，支持楼中楼 |
| `class_join_applications` | 加入申请 | class_id (FK->classes), user_id (FK->users), reason, status (0=待审核/1=已通过/2=已驳回), reviewed_by | N:1 关联 classes |
| `class_creator_applications` | 创作者申请 | class_id, user_id, reason, status, reviewed_by, reviewed_at, remark | N:1 关联 classes，申请流程驱动 |
| `class_creator_permissions` | 创作者权限 | user_id (FK->users), max_classes, can_create, expires_at, granted_by, reason | 用户级权限，审批通过后创建 |
| `class_attendances` | 考勤活动 | class_id (FK->classes), title, type, start_time, end_time, location, latitude, longitude, radius, creator_id | N:1 关联 classes |

| `class_tags` | 班级标签 | name (unique), color, creator_id | 全局标签 |
| `class_tag_relations` | 标签关联 | class_id (FK->classes), tag_id (FK->class_tags) | 唯一索引 (class_id, tag_id) |

| `class_study_plans` | 学习计划 | class_id (FK->classes), title, start_date, end_date, status, creator_id | N:1 关联 classes |

| `class_resource_versions` | 资源版本 | class_id, resource_type, resource_id, version, content_snapshot, creator_id | 唯一索引 (resource_type, resource_id, version) |
| `class_resource_reviews` | 资源审核 | class_id, resource_type, resource_id, creator_id, status, reviewer_id, remark | 审核流程 |
| `class_resource_tags` | 资源标签 | class_id, name, color, creator_id | 唯一索引 (class_id, name) |

| `score_alerts` | 成绩预警 | class_id, student_id, exam_id, score, threshold, status | 唯一索引 (class_id, student_id, exam_id) |
| `class_peer_reviews` | 学生互评 | homework_id, class_id, reviewer_id, reviewee_id, score, content, status | 唯一索引 (homework_id, reviewer_id, reviewee_id) |
| `class_import_records` | 导入记录 | class_id, file_name, total, success, failed, errors (JSON), operator_id | N:1 关联 classes |
| `class_templates` | 班级模板 | name, description, config_json, creator_id, is_system, usage_count | 模板库 |

**索引设计原则**：

- 所有外键字段建立索引（class_id、user_id、creator_id 等）
- 查询频繁的字段组合建立联合唯一索引（防止重复数据）
- 软删除字段 deleted_at 建立索引（GORM 自动过滤）
- 状态字段配合索引加速筛选查询

## 七、核心实现

### 7.1 班级创建与邀请码

**请求入口**：`POST /api/v1/class`

**邀请码生成算法**：

```
generateClassCode()
    │
    ├── 字符集: "ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789" (36 字符)
    ├── 长度: 6 位
    ├── 随机源: math/rand（非密码学安全，邀请码无需密码学强度）
    │
    ├── 生成流程:
    │   1. 生成 6 位随机字符串
    │   2. 查询 classes 表 WHERE code = ?
    │   3. 若已存在 → 重新生成（冲突概率极低：36^6 ≈ 21.8 亿种组合）
    │   4. 不存在 → 返回邀请码
    │
    └── 生成后:
        1. INSERT classes (name, description, cover, code, creator_id, member_count=1, status=1)
        2. INSERT class_members (class_id, user_id, role=3, joined_at=now)
           — 创建者自动成为管理员 (role=3)
        3. 记录操作日志 class_logs
```

**二维码生成**：

```
GET /api/v1/class/:id/qrcode
    │
    ├── 1. 查询班级信息，获取 code
    ├── 2. 构造邀请 URL: /class/join?code={code}
    ├── 3. 使用 qrcode 库生成 PNG 图片
    ├── 4. Base64 编码返回
    │
    └── 响应: { code, url, image (base64) }
```

**班级删除保护规则**：

```
DELETE /api/v1/class/:id
    │
    ├── 前置检查:
    │   ├── 系统管理员 → 跳过权限检查
    │   ├── 班级创建者 (creator_id == operatorID) → 允许
    │   └── 其他 → 拒绝
    │
    ├── 删除保护:
    │   ├── 成员数 > 0 → "请先移除所有成员"
    │   ├── 有进行中考试 → "考试进行中，无法删除班级"
    │   ├── 有考试记录 → 允许软删除，记录保留
    │   └── 有班级资源 → 允许软删除，资源归档
    │
    └── 执行: GORM 软删除 (UPDATE deleted_at)
```

**班级转让规则**：

- 教师可将班级转让给其他教师
- 转让后原教师变为普通成员 (role=1)，目标教师变为管理员 (role=3)
- 更新 classes.creator_id
- 转让需确认，不可撤销
- 记录操作日志

**权限检查逻辑**：

```go
// Service 层检查创作者权限
func checkCreatorPermission(classID, userID uint) bool {
    // 1. 检查是否是班级教师/管理员
    member, err := classRepo.FindMember(classID, userID)
    if err == nil && member.Role >= 2 {
        return true // 教师/管理员自动拥有创作者权限
    }
    // 2. 检查是否是班级创作者
    perm, err := classRepo.FindCreatorPermission(userID)
    if err != nil {
        return false
    }
    return perm.CanCreate
}
```

```
    DELETE /api/v1/class/:id/discussions/:discussionId/replies/:replyId
        ├── 只有回复者或班级管理员才能删除
        ├── 递减 reply_count
        └── 软删除回复记录
```

## 八、权限控制

### 8.1 权限模型

班级管理采用**基于角色的权限控制**，角色定义在 `class_members.role` 字段：

| 角色值 | 标识 | 说明 |
|--------|------|------|
| 1 | student | 普通学生，只读权限 |
| 2 | teacher | 教师，管理权限 |
| 3 | admin | 管理员，完全控制 |

### 8.2 权限校验实现

```go
// checkClassPermission 检查班级内权限
// role: 1=学生, 2=教师, 3=管理员
func checkClassPermission(classID, userID uint, requiredRole int8) error {
    member, err := classRepo.FindMember(classID, userID)
    if err != nil {
        return errors.New("不是班级成员")
    }
    if member.Role < requiredRole {
        return errors.New("权限不足")
    }
    return nil
}
```

## 九、安全考虑

### 9.1 邀请码安全

| 措施 | 说明 |
|------|------|
| 唯一性保证 | 数据库 unique 约束 + 生成后查重 |
| 复杂度 | 36 字符集 6 位，约 21.8 亿种组合 |
| 不可预测性 | 随机生成，不包含可识别信息 |
| 重新生成支持 | 教师可重新生成邀请码，旧码立即失效 |

### 9.2 班级资源隔离

| 措施 | 说明 |
|------|------|
| class_id 强制过滤 | Repository 层所有资源查询强制加入 class_id 条件 |
| 成员身份校验 | 每个班级 API 中间件检查用户是否为班级成员 |
| 资源类型隔离 | 班级资源 resource_type="class"，与公共资源完全独立 |
| 导入时复制 | 从公共题库导入时数据完全复制，修改互不影响 |

### 9.3 成员数量限制

| 措施 | 说明 |
|------|------|
| 默认上限 500 | 班级最大成员数默认 500，可配置 |
| 加入前检查 | JoinClass 中检查 member_count >= max_members |
| 计数器一致性 | 使用数据库原子操作 member_count + 1/-1 |

### 9.4 数据安全

| 措施 | 说明 |
|------|------|
| 软删除 | 所有实体使用 GORM 软删除 (deleted_at) |
| 操作日志 | 关键操作记录到 class_logs 表 |
| 敏感词过滤 | 讨论内容、公告内容经过敏感词中间件过滤 |
| XSS 防护 | 输出转义，Content-Type: application/json |
