# 03-题库管理 技术实现文档

## 1. 模块概述

题库管理模块是题小助平台的核心业务模块，负责题目的全生命周期管理，涵盖题目创建、编辑、审核、发布、归档、删除等操作，以及分类树、知识点、标签、附件、分享、收藏等关联功能。该模块为考试管理、练习系统、内容创作等上层业务提供基础数据支撑。

### 1.1 功能边界

| 子模块 | 职责范围 |
|--------|---------|
| 题目管理 | CRUD、批量操作、状态流转、版本快照、查重、预览、复制 |
| 分类管理 | 无限层级树、物化路径、拖拽排序、合并、统计 |
| 知识点管理 | CRUD、与题目多对多关联、统计、批量移动 |
| 标签管理 | 自由标签、系统预置标签、颜色标识 |
| 内容审核 | 审核流程、批量审核、超时提醒、审核统计 |
| 导入导出 | Excel 导入导出、模板下载、进度跟踪、报告生成 |
| 附件管理 | 文件上传、引用计数、孤立文件清理 |
| 分享管理 | 分享链接生成、密码保护、过期控制、查看统计 |
| 收藏管理 | 收藏夹分组、收藏/取消收藏、收藏列表 |
| 题目互动 | 笔记、难度评价、质量评分、纠错提交 |

### 1.2 资源可见性模型

系统支持三种资源可见性：

| 可见性 | 值 | 创建者角色 | 可见范围 |
|--------|---|-----------|---------|
| 公共 (public) | 1 | 内容管理员/管理员 | 所有用户 |
| 私有 (private) | 2 | 创作者/教师 | 仅创建者 |
| 班级 (class) | 3 | 教师/班级创作者 | 班级成员 |

资源互转规则：
- 私有 -> 公共：需提交审核，通过后变为公共
- 公共 -> 私有：不支持
- 私有 -> 班级：不支持（需在班级内重新创建）
- 班级 -> 公共：不支持

### 3.2 依赖组装方式

题库管理模块采用**包级函数**模式（非结构体方法），无依赖注入框架：

- Controller 层：结构体方法，通过 `NewXxxController()` 返回空结构体实例
- Service 层：包级函数，直接通过导入包互相调用
- Repository 层：包级函数，通过全局 `database.DB` 访问数据库

```go
// Controller 调用 Service 示例
func (ctrl *QuestionController) CreateQuestion(c *gin.Context) {
    // ...
    if err := question.CreateQuestion(creatorID, &req); err != nil { ... }
}

// Service 调用 Repository 示例
func CreateQuestion(creatorID uint, req *dto.CreateQuestionRequest) error {
    // ...
    if err := questionRepo.Create(q); err != nil { ... }
}
```

```go
func SetupQuestionRoutes(r *gin.RouterGroup, ...) {
    q := r.Group("/questions")
    {
        q.GET("", ctrl.ListQuestions)           // 题目列表
        q.GET("/search", ctrl.SearchQuestions)  // 搜索

        q.POST("/:id/quality-rating", ...)      // 质量评分
        q.POST("/:id/corrections", ...)         // 纠错
    }
}

// 管理端路由（/api/v1/admin/...）
func SetupAdminQuestionRoutes(r *gin.RouterGroup, ...) {
    q := r.Group("/questions")
    {
        q.GET("", ctrl.AdminListQuestions)      // 管理视图列表
        q.POST("", ctrl.CreateQuestion)         // 创建
        q.PUT("/:id", ctrl.UpdateQuestion)      // 更新
        q.DELETE("/:id", ctrl.DeleteQuestion)   // 删除
        q.PUT("/:id/status", ...)               // 状态变更
        q.POST("/import", ...)                  // 导入
        q.GET("/export", ...)                   // 导出
    }
    batch := r.Group("/questions/batch") { ... }
    categories := r.Group("/categories") { ... }
    knowledge := r.Group("/knowledge-points") { ... }
    review := r.Group("/reviews") { ... }
    sensitive := r.Group("/sensitive-words") { ... }
}
```

## 7. 核心实现

### 7.1 题目内容结构

#### 7.1.1 五种题型的 Content JSON Schema

题目内容存储在 `questions.content` 字段（TEXT 类型），同时选项存储在 `options` 表中。不同题型的结构如下：

**单选题 (type=1)**:
```json
{
  "title": "<p>以下哪个是 Go 语言的特点？</p>",
  "options": [
    {"label": "A", "content": "静态类型", "is_correct": true},
    {"label": "B", "content": "动态类型", "is_correct": false},
    {"label": "C", "content": "弱类型", "is_correct": false},
    {"label": "D", "content": "无类型", "is_correct": false}
  ],
  "answer": "A",
  "analysis": "<p>Go 是静态类型语言...</p>"
}
```

**多选题 (type=2)**:
```json
{
  "title": "<p>以下哪些是 Go 语言的并发原语？</p>",
  "options": [
    {"label": "A", "content": "goroutine", "is_correct": true},
    {"label": "B", "content": "channel", "is_correct": true},
    {"label": "C", "content": "thread", "is_correct": false},
    {"label": "D", "content": "select", "is_correct": true}
  ],
  "answer": "A,B,D",
  "analysis": "<p>Go 的并发原语包括 goroutine、channel 和 select...</p>"
}
```

**判断题 (type=3)**:
```json
{
  "title": "<p>Go 语言支持继承。</p>",
  "answer": "false",
  "analysis": "<p>Go 使用组合而非继承...</p>"
}
```

**填空题 (type=4)**:
```json
{
  "title": "<p>Go 语言中，用 ___ 关键字声明变量。</p>",
  "answer": "var",
  "keywords": ["var"],
  "analysis": "<p>Go 使用 var 关键字声明变量...</p>"
}
```

**简答题 (type=5)**:
```json
{
  "title": "<p>请解释 Go 语言中 goroutine 和线程的区别。</p>",
  "answer": "<p>goroutine 是 Go 运行时管理的轻量级线程...</p>",
  "analysis": "<p>参考答案要点：...</p>"
}
```

#### 7.1.2 XSS 防护实现

后端使用双重防护策略：

**第一层：敏感词中间件过滤**

```go
// internal/middleware/sensitive.go
func SensitiveFilterMiddleware() gin.HandlerFunc {
    filter := sensitive.NewFilter()
    return func(c *gin.Context) {
        if c.Request.Method == "POST" || c.Request.Method == "PUT" {
            body, _ := io.ReadAll(c.Request.Body)
            c.Request.Body = io.NopCloser(bytes.NewBuffer(body))
            if filter.HasSensitive(string(body)) {
                response.Error(c, 400, "内容包含敏感词，请修改后重试")
                c.Abort()
                return
            }
        }
        c.Next()
    }
}
```

**第二层：内容净化（Service 层）**

在 `CreateQuestion` 和 `UpdateQuestion` 中，对 HTML 内容进行净化：

```go
// 允许的安全标签：p, br, strong, em, img, table, tr, td, th, ul, ol, li
// 禁止的标签：script, iframe, object, embed, form, style, link
// 禁止的属性：onerror, onclick, onload, onmouseover 等事件属性
// 图片 src 协议白名单：http, https, data
```

**前端防护**：使用 DOMPurify 对富文本内容进行二次净化。

#### 7.1.3 Content JSON 校验规则

| 规则 | 限制 |
|------|------|
| 题干最大长度 | 5000 字符 |
| 选项最大长度 | 1000 字符/每个选项 |
| 解析最大长度 | 5000 字符 |
| JSON 嵌套深度 | 不超过 5 层 |
| 未知字段 | 不允许 (additionalProperties: false) |

### 7.5 批量导入流程

导入采用 10 步流程，确保数据完整性和一致性：


#### 7.5.1 流程详细实现

**Step 1: 上传文件**

**Step 4: 解析内容**

```go
// service/question/import_service.go
func ImportQuestions(creatorID uint, categoryID uint, visibility int8, data []byte) (int, error) {
    // 使用 excelize 解析 Excel
    f, err := excelize.OpenReader(bytes.NewReader(data))
    if err != nil {
        return 0, fmt.Errorf("解析 Excel 失败: %w", err)
    }
    defer f.Close()

    // 从第 2 行开始解析（第 1 行为表头）
    rows := f.GetRows(f.GetSheetName(0))
    if len(rows) < 2 {
        return 0, errors.New("Excel 文件为空")
    }

    // Step 5: 按"题目序号"分组
    groups := groupByQuestionNumber(rows[1:]) // 跳过表头

    // Step 6: 逐组校验
    var validQuestions []model.Question
    var importErrors []ImportError

    for questionNum, groupRows := range groups {
        // 校验题型合法
        // 校验题目内容非空
        // 校验分类 ID 存在
        // 校验选择题至少一个正确选项

        q, err := validateAndBuildQuestion(groupRows, categoryID, visibility, creatorID)
        if err != nil {
            importErrors = append(importErrors, ImportError{
                Row:    questionNum,
                Reason: err.Error(),
            })
            continue
        }

        // Step 7: 内容 MD5 查重
        contentHash := md5Hash(q.Title + q.Content)
        if isDuplicate(contentHash) {
            importErrors = append(importErrors, ImportError{
                Row:    questionNum,
                Reason: "题目内容重复",
            })
            continue
        }
        q.ContentHash = contentHash

        validQuestions = append(validQuestions, *q)
    }

    // Step 8: 构建 content JSON
    // 在 validateAndBuildQuestion 中已完成

    // Step 9: 事务批量入库
    if err := questionRepo.BatchCreate(validQuestions); err != nil {
        return 0, fmt.Errorf("批量创建题目失败: %w", err)
    }

    // Step 10: 返回导入报告
    return len(validQuestions), nil
}
```

### 7.6 题目去重

#### 7.6.1 Content Hash 计算

题目创建或导入时，对题干内容计算 MD5 哈希值：

```go
func computeContentHash(title, content string) string {
    // 对题干内容进行标准化后计算 MD5
    normalized := strings.TrimSpace(title) + strings.TrimSpace(content)
    hash := md5.Sum([]byte(normalized))
    return hex.EncodeToString(hash[:])
}
```

#### 7.6.2 查重检测

```go
// API: POST /api/v1/admin/questions/check-duplicate
func CheckDuplicate(req *dto.CheckDuplicateRequest) (*dto.DuplicateResult, error) {
    contentHash := computeContentHash(req.Title, req.Content)

    // 查询是否存在相同 hash 的题目
    var count int64
    database.DB.Model(&model.Question{}).
        Where("content_hash = ? AND deleted_at IS NULL", contentHash).
        Count(&count)

    if count > 0 {
        // 查询重复题目列表
        var questions []model.Question
        database.DB.Where("content_hash = ? AND deleted_at IS NULL", contentHash).
            Find(&questions)

        list := make([]dto.QuestionInfo, 0, len(questions))
        for _, q := range questions {
            list = append(list, toQuestionInfo(&q))
        }

        return &dto.DuplicateResult{
            IsDuplicate: true,
            Count:       int(count),
            Questions:   list,
        }, nil
    }

    return &dto.DuplicateResult{IsDuplicate: false}, nil
}
```

#### 7.8.2 完整状态码

|--------|--------|-----------|-----------|

#### 7.8.3 状态流转校验

```go
// service/question/question_service.go
var validStatusTransitions = map[int8][]int8{
    0: {1, 3}, // 草稿 -> 待审核, 已发布
    1: {2},    // 已发布 -> 已归档
    2: {1},    // 已归档 -> 已发布
    3: {1, 4}, // 待审核 -> 已发布, 已拒绝
    4: {0},    // 已拒绝 -> 草稿
}

func isValidStatusTransition(from, to int8) bool {
    allowed, ok := validStatusTransitions[from]
    if !ok {
        return false
    }
    for _, s := range allowed {
        if s == to {
            return true
        }
    }
    return false
}
```

#### 7.8.4 审核流程实现

```go
// 创作者提交审核
func SubmitForReview(questionID, userID uint) error {
    q, _ := questionRepo.FindByID(questionID)

    // 权限校验：只有创建者可以提交
    if q.CreatorID != userID {
        return errors.New("无权提交此题目审核")
    }

    // 状态校验：只有草稿状态可以提交
    if q.Status != 0 {
        return errors.New("只有草稿状态的题目才能提交审核")
    }

    // 更新题目状态为待审核
    q.Status = 3
    questionRepo.Update(q)

    // 创建审核记录
    review := &model.QuestionReview{
        QuestionID:  questionID,
        Status:      0, // 待审核
        SubmittedBy: userID,
        SubmittedAt: time.Now(),
    }
    questionRepo.CreateReview(review)

    return nil
}

// 审核通过
func ApproveReview(reviewID, reviewerID uint) error {
    review, _ := questionRepo.FindReviewByID(reviewID)

    review.Status = 1 // 通过
    review.ReviewerID = &reviewerID
    now := time.Now()
    review.ReviewedAt = &now
    questionRepo.UpdateReview(review)

    // 更新题目状态为已发布
    q, _ := questionRepo.FindByID(review.QuestionID)
    q.Status = 1
    questionRepo.Update(q)

    return nil
}

// 审核驳回
func RejectReview(reviewID, reviewerID uint, reason string) error {
    review, _ := questionRepo.FindReviewByID(reviewID)

    review.Status = 2 // 驳回
    review.ReviewerID = &reviewerID
    review.Opinion = reason
    now := time.Now()
    review.ReviewedAt = &now
    questionRepo.UpdateReview(review)

    // 更新题目状态为草稿（允许修改后重新提交）
    q, _ := questionRepo.FindByID(review.QuestionID)
    q.Status = 0
    questionRepo.Update(q)

    return nil
}
```

### 8.2 敏感词过滤

```go
// 内部/middleware/sensitive.go
func SensitiveFilterMiddleware() gin.HandlerFunc {
    filter := sensitive.NewFilter()
    return func(c *gin.Context) {
        if c.Request.Method == "POST" || c.Request.Method == "PUT" {
            body, _ := io.ReadAll(c.Request.Body)
            c.Request.Body = io.NopCloser(bytes.NewBuffer(body))
            if filter.HasSensitive(string(body)) {
                response.Error(c, 400, "内容包含敏感词，请修改后重试")
                c.Abort()
                return
            }
        }
        c.Next()
    }
}
```

敏感词过滤器使用 AC 自动机算法，支持高性能多模式匹配。

### 8.3 数据权限

```go
// 内部/middleware/data_permission.go
func DataPermissionMiddleware(tableName string) gin.HandlerFunc {
    return func(c *gin.Context) {
        userID, _ := c.Get("user_id")
        roleIDs, _ := c.Get("role_ids")

        // 管理员拥有全部权限
        if isAdmin(roleIDs.([]uint)) {
            c.Next()
            return
        }

        // 普通用户只能看到自己创建的数据
        condition := fmt.Sprintf("%s.created_by = ?", tableName)
        c.Set("data_permission_condition", condition)
        c.Set("data_permission_args", []interface{}{userID})
        c.Next()
    }
}
```

## 10. 安全考虑

### 10.1 输入验证

| 验证点 | 实现方式 |
|--------|---------|
| DTO 验证 | 使用 `binding` 标签进行结构化验证 |
| 题型校验 | `oneof=1 2 3 4 5` |
| 难度校验 | `oneof=1 2 3` |
| 可见性校验 | `oneof=1 2 3` |
| 内容长度 | `max=5000` (题干/解析), `max=1000` (选项) |
| 分页参数 | PageSize 最大 100 |

### 10.2 XSS 防护

- 后端：敏感词中间件 + 内容净化（白名单标签）
- 前端：DOMPurify 二次净化
- 禁止：script, iframe, object, embed, form, style, link
- 禁止：onerror, onclick, onload 等事件属性
- 图片 src 仅允许 http/https/data 协议

### 10.3 SQL 注入防护

- 使用 GORM 参数化查询，禁止字符串拼接
- 所有用户输入通过 `ShouldBindJSON` / `ShouldBindQuery` 绑定
- 分页参数限制最大值

### 10.4 权限控制

| 操作 | 权限要求 |
|------|---------|
| 查看公共题目 | 已登录用户 |
| 创建公共题目 | admin, content_admin 角色 |
| 编辑公共题目 | admin, content_admin 角色 |
| 删除公共题目 | admin, content_admin 角色 |
| 创建私有题目 | teacher, creator 角色 |
| 编辑私有题目 | 仅创建者本人 |
| 审核题目 | admin, content_admin 角色 |
| 导入导出 | admin, content_admin 角色 |

### 10.5 删除保护

- 题目被未完成的考试引用时，禁止删除
- 题目被已有答题记录的考试引用时，仅允许软删除
- 分类存在子分类或关联题目时，禁止删除
- 知识点有关联题目时，禁止删除

## 11. 性能考虑

### 11.1 数据库优化

| 优化点 | 实现方式 |
|--------|---------|
| 索引优化 | content_hash, category_id, creator_id, status, visibility 建立索引 |
| 分页查询 | 使用 OFFSET/LIMIT，避免全表扫描 |
| 预加载 | 使用 GORM Preload 加载关联数据（Options, Category, Creator） |
| 计数缓存 | question_count, like_count, favorite_count 使用原子更新 |
| 软删除索引 | deleted_at 字段建立索引 |

### 11.2 缓存策略

| 缓存项 | 缓存时间 | 更新策略 |
|--------|---------|---------|
| 分类树 | 30 分钟 | 创建/更新/删除分类时主动失效 |
| 知识点列表 | 30 分钟 | 创建/更新/删除知识点时主动失效 |
| 题目详情 | 5 分钟 | 更新题目时主动失效 |
| 统计数据 | 1 小时 | 定时任务更新 |
| 敏感词库 | 永久 | 添加/删除敏感词时重新加载 |

### 11.3 批量操作优化

- 批量导入使用事务 + `CreateInBatches`（每批 100 条）
- 批量删除使用软删除，避免物理删除开销
- 批量更新使用 `Updates` 批量操作，避免逐条更新

### 11.4 搜索优化

- 题目搜索使用 `LIKE` 模糊匹配（当前实现）
- 后续可升级为全文索引或 Elasticsearch
- 搜索建议使用 Redis 缓存热门关键词

```go
func BatchPublish(req *dto.BatchQuestionRequest) (*dto.BatchResult, error) {
    result := &dto.BatchResult{Total: len(req.IDs)}

    for _, id := range req.IDs {
        err := UpdateQuestionStatus(id, 1)

                Reason: err.Error(),

        }
```
