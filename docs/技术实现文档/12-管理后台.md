## 1. 模块概述

管理后台是题小助 V2.0 的 Web 端管理界面，基于 vue3-element-admin 模板构建的单页应用 (SPA)。面向管理员、教师和创作者角色，提供题库管理、考试管理、班级管理、数据统计、系统设置等全功能管理能力。

核心职责：

- **仪表盘**：KPI 卡片、趋势图表、快捷操作、数据导出
- **动态菜单与权限**：后端驱动的菜单树 + 按钮级权限控制
- **通用 CURD 框架**：配置驱动的表格 + 表单 + 分页复用组件
- **主题与国际化**：亮色/暗色主题切换、多语言支持
- **数据导出**：ExcelJS 前端导出能力

技术栈：Vue 3.4+ / TypeScript 5.x / Vite 5.x / Element Plus 2.5+ / Pinia 2.x / Vue Router 4.x / Axios 1.x / ECharts 5.x / WangEditor 5.x / ExcelJS 4.x / UnoCSS / vue-i18n 9.x。

### 3.2 分层职责

| API | `src/api/` | 后端接口封装（Axios） | → Utils (request) |
| Utils | `src/utils/` | 通用工具（request、auth） | - |
| Layouts | `src/layouts/` | 布局容器（BaseLayout） | → Store, Components |
| Styles | `src/styles/` | 全局样式、CSS 变量 | - |

## 5. 核心实现

### 5.1 仪表盘

仪表盘是管理员登录后的默认首页，展示平台核心运营指标、趋势图表和快捷操作入口。

#### 5.1.5 数据导出

仪表盘支持将数据导出为 Excel，调用 `exportStatisticsApi()` 后端接口获取 Blob 数据，前端通过 `Blob` + `<a>` 标签触发下载：

```typescript
async function handleExport(params: ExportParams) {
  const blob = await exportStatisticsApi(params);
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `统计数据_${formatDate(new Date())}.xlsx`;
  a.click();
  URL.revokeObjectURL(url);
}
```

#### 5.1.6 数据订阅

数据订阅功能通过 `/api/v1/admin/subscriptions` 系列接口管理，支持创建日报/周报/月报订阅，配置发送时间和包含内容（核心指标、趋势图、异常预警），由后端定时任务触发邮件发送。

### 5.2 动态菜单加载

管理后台的菜单和路由由后端动态管理，根据用户角色返回不同的菜单树和按钮权限列表。

#### 5.2.1 加载流程

```
用户登录成功
    │
    ▼
路由守卫触发 (router/guards/permission.ts)
    │
    ▼
检查 userStore.userInfo 是否存在
    │
    ├── 已存在 → 直接放行
    │
    └── 不存在 → 执行初始化流程
         │
         ▼
    userStore.getUserInfo()
    │  调用 GET /api/v1/user/info
    │  获取用户信息、角色、权限列表
    │
    ▼
    permissionStore.generateRoutes()
    │  ├── 调用 GET /api/v1/menus/routes → 获取路由菜单树
    │  └── 调用 GET /api/v1/menus/buttons → 获取按钮权限列表
    │
    ▼
    transformRoutes(routeData)
    │  将后端返回的菜单数据转换为 Vue Router 路由配置
    │  ├── component: "Layout" → import("@/layouts/BaseLayout.vue")
    │  └── component: "views/xxx" → import(`@/views/${component}.vue`)
    │
    ▼
    routes.forEach(route => router.addRoute(route))
    │  动态注入路由到 Router 实例
    │
    ▼
    next({ ...to, replace: true })
    │  重新导航到目标路由（确保动态路由已生效）
    │
    ▼
    BaseLayout.vue 读取 permissionStore.routes
    │  渲染侧边栏菜单（过滤 hidden 路由）
    │
    ▼
    页面渲染完成
```

#### 5.2.2 路由转换

后端返回的菜单数据结构与 Vue Router `RouteRecordRaw` 之间的映射关系：

| 后端字段 | RouteRecordRaw 字段 | 说明 |
|----------|---------------------|------|
| `path` | `path` | 路由路径 |
| `name` | `name` | 路由名称 |
| `component` | `component` | 组件路径，通过 `resolveComponent()` 动态导入 |
| `redirect` | `redirect` | 重定向路径 |
| `icon` | `meta.icon` | 菜单图标 |
| `title` | `meta.title` | 菜单标题 |
| `hidden` | `meta.hidden` | 是否在菜单中隐藏 |
| `children` | `children` | 子路由递归转换 |

`resolveComponent` 函数处理两种情况：
- `"Layout"` → 返回 `BaseLayout.vue`（一级菜单作为布局容器）
- `"views/xxx/index"` → 返回 `@/views/xxx/index.vue`（具体页面组件）

#### 5.2.3 侧边栏渲染

`BaseLayout.vue` 从 `permissionStore.routes` 读取路由列表，过滤掉 `meta.hidden` 为 true 的路由，递归渲染 `el-menu` / `el-sub-menu` / `el-menu-item`。菜单图标通过 `<component :is="route.meta.icon" />` 动态渲染 Element Plus 图标组件。

#### 5.2.4 菜单管理页面

`src/views/system/menu/index.vue` 提供菜单的 CRUD 管理界面：
- 树形表格展示所有菜单（无限层级）
- 新增/编辑菜单：设置名称、路由路径、组件路径、图标、父菜单、排序、类型（目录/菜单/按钮）、权限标识
- 删除菜单：有子菜单时禁止删除
- 启用/禁用菜单状态

对应 API：`src/api/menu.ts` 中的 `listMenusApi()`、`getMenuTreeApi()`、`createMenuApi()`、`updateMenuApi()`、`deleteMenuApi()`。

### 5.3 按钮权限控制

#### 5.3.1 实现机制

按钮权限通过自定义 Vue 指令 `v-permission` 实现，定义在 `src/directives/permission/index.ts`。

```typescript
// src/directives/permission/index.ts
import type { Directive, DirectiveBinding } from "vue";
import { useUserStore } from "@/stores/user";

export const permission: Directive = {
  mounted(el: HTMLElement, binding: DirectiveBinding) {
    const userStore = useUserStore();
    const { value } = binding;

    if (value) {
      const hasPerm = Array.isArray(value)
        ? value.some((p: string) => userStore.hasPermission(p))
        : userStore.hasPermission(value);

      if (!hasPerm) {
        el.parentNode?.removeChild(el); // 无权限则移除 DOM 元素
      }
    }
  },
};
```

#### 5.3.2 权限检查逻辑

`userStore.hasPermission(perm)` 的检查逻辑：

1. `super_admin` 角色拥有所有权限，直接返回 `true`
2. 其他角色检查 `userInfo.permissions` 数组是否包含该权限标识
3. 权限标识来源于登录后 `GET /api/v1/menus/buttons` 接口返回的按钮权限列表

#### 5.3.3 使用方式

```vue
<!-- 单个权限 -->
<el-button v-permission="'user:list:create'">新增用户</el-button>

<!-- 多个权限（满足任一即可） -->
<el-button v-permission="['user:list:edit', 'user:list:delete']">操作</el-button>
```

#### 5.3.4 权限标识命名规范

采用三级编码：`模块:子模块:操作`

| 模块 | 示例 |
|------|------|
| 用户 | `user:list:view`, `user:list:create`, `user:list:edit`, `user:list:delete` |
| 题目 | `question:public:view`, `question:private:view`, `question:public:create` |
| 考试 | `exam:paper:view`, `exam:paper:create`, `exam:publish:view`, `exam:monitor:view` |
| 班级 | `class:list:view`, `class:list:create`, `class:member:manage`, `class:creator:approve` |
| 系统 | `system:settings:view`, `system:log:view`, `system:backup:view` |

### 5.5 主题切换

#### 5.5.1 实现方案

主题切换基于 CSS 变量 + Pinia 状态管理，通过 `document.documentElement.setAttribute("data-theme", theme)` 切换根元素的 `data-theme` 属性，CSS 变量根据属性值自动切换。

#### 5.5.2 状态管理

`src/stores/settings.ts` 管理主题状态：

```typescript
export const useSettingsStore = defineStore("settings", () => {
  const theme = ref<string>("light");         // "light" | "dark"
  const primaryColor = ref<string>("#409eff"); // 主题色

  function setTheme(val: string) {
    theme.value = val;
    document.documentElement.setAttribute("data-theme", val);
  }

  function setPrimaryColor(val: string) {
    primaryColor.value = val;
    document.documentElement.style.setProperty("--el-color-primary", val);
  }

  return { theme, primaryColor, setTheme, setPrimaryColor };
});
```

#### 5.5.3 CSS 变量定义

在 `src/styles/index.scss` 中定义亮色/暗色两套 CSS 变量：

```scss
:root,
[data-theme="light"] {
  --bg-color: #ffffff;
  --text-color: #303133;
  --border-color: #dcdfe6;
  --sidebar-bg: #304156;
  --navbar-bg: #ffffff;
  // ...
}

[data-theme="dark"] {
  --bg-color: #1d1e1f;
  --text-color: #e5eaf3;
  --border-color: #414243;
  --sidebar-bg: #1d1e1f;
  --navbar-bg: #1d1e1f;
  // ...

```

### 5.7 数据导出

#### 5.7.1 ExcelJS 集成

使用 ExcelJS 4.x 在前端生成 Excel 文件，支持格式化、样式、多 Sheet 等高级功能。

```typescript
// src/utils/export.ts
import ExcelJS from "exceljs";

interface ExportColumn {
  header: string;
  key: string;
  width?: number;
}

export async function exportToExcel(
  data: any[],
  columns: ExportColumn[],
  filename: string
) {
  const workbook = new ExcelJS.Workbook();
  const sheet = workbook.addWorksheet("Sheet1");

  // 设置列定义
  sheet.columns = columns.map((col) => ({
    header: col.header,
    key: col.key,
    width: col.width || 20,
  }));

  // 设置表头样式
  sheet.getRow(1).font = { bold: true };
  sheet.getRow(1).fill = {
    type: "pattern",
    pattern: "solid",
    fgColor: { argb: "FFE8F0FE" },
  };

  // 填充数据
  data.forEach((row) => sheet.addRow(row));

  // 导出
  const buffer = await workbook.xlsx.writeBuffer();
  const blob = new Blob([buffer], {
    type: "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
  });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `${filename}.xlsx`;
  a.click();
  URL.revokeObjectURL(url);
}
```

#### 5.7.2 两种导出模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 前端导出 | ExcelJS 生成 Excel，数据已在前端 | 小数据量、自定义格式 |
| 后端导出 | 调用后端导出接口，返回 Blob | 大数据量、复杂查询 |

后端导出调用 `exportStatisticsApi()` 等接口，设置 `responseType: "blob"`，前端接收 Blob 后触发下载。

#### 5.7.3 CSV 导出

轻量级场景支持 CSV 导出：

```typescript
export function exportToCSV(data: any[], columns: ExportColumn[], filename: string) {
  const header = columns.map((c) => c.header).join(",");
  const rows = data.map((row) =>
    columns.map((c) => `"${String(row[c.key] ?? "").replace(/"/g, '""')}"`).join(",")
  );
  const csv = "﻿" + [header, ...rows].join("\n"); // BOM 头确保中文正确
  const blob = new Blob([csv], { type: "text/csv;charset=utf-8" });
  // 触发下载...
}
```

## 7. 路由与权限

### 7.1 静态路由

定义在 `src/router/index.ts`，不受权限控制：

| 路径 | 组件 | 说明 |
|------|------|------|
| `/login` | `views/login/index.vue` | 登录页 |
| `/401` | `views/error/401.vue` | 未授权页 |
| `/404` | `views/error/404.vue` | 页面未找到 |
| `/redirect/:path(.*)` | `views/redirect/index.vue` | 重定向中转 |

### 7.2 动态路由

动态路由由后端接口 `/api/v1/menus/routes` 返回，通过 `permissionStore.generateRoutes()` 转换后注入 Router。路由数据包含：

- 路由路径、名称、组件路径
- 菜单标题、图标、排序
- 子路由（递归）

### 7.3 路由守卫

`src/router/guards/permission.ts` 实现全局前置守卫，处理流程：

1. **有 Token**：
   - 访问 `/login` → 重定向到 `/`
   - 用户信息未加载 → 调用 `getUserInfo()` + `generateRoutes()`，动态注入路由后重新导航
   - 用户信息已加载 → 直接放行
2. **无 Token**：
   - 白名单路由（`/login`、`/401`、`/404`）→ 放行
   - 其他路由 → 重定向到 `/login?redirect={path}`

├── 题库管理

### 7.5 权限数据流

```
登录成功
  │
  ▼
GET /api/v1/user/info → userInfo.roles, userInfo.permissions
  │
  ▼
GET /api/v1/menus/routes → 路由菜单树（控制侧边栏显示）
  │
  ▼
GET /api/v1/menus/buttons → 按钮权限列表（控制页面内操作）
  │
  ▼
存储到 permissionStore.buttons
  │
  ▼
页面中 v-permission="'user:list:create'" → userStore.hasPermission()
  │
  ▼
检查 permissionStore.buttons 是否包含该标识 → 显示/移除按钮
```

## 8. 状态管理

### 8.1 Store 清单

| Store | 文件 | 职责 | 持久化 |
|-------|------|------|--------|
| `user` | `src/stores/user.ts` | 用户 token、userInfo、登录/登出、权限检查 | token → localStorage |
| `permission` | `src/stores/permission.ts` | 动态路由列表、按钮权限列表、路由生成状态 | 否（每次登录重新加载） |
| `app` | `src/stores/app.ts` | 侧边栏折叠状态、设备类型 | 否 |
| `settings` | `src/stores/settings.ts` | 主题、主色、标签页开关、面包屑开关 | theme/primaryColor → localStorage |
| `dict` | `src/stores/dict.ts` | 字典数据缓存（Map 结构） | 否（会话级缓存） |
| `tags-view` | `src/stores/tags-view.ts` | 已访问标签页列表 | 否 |

### 8.2 Store 间依赖关系

```
userStore
  ▲
  │ (hasPermission, userInfo)
  │
permissionStore ──→ userStore (无直接依赖，但 permission 指令依赖 userStore)
  │
  ▼
router (addRoute)

appStore (独立，无跨 Store 依赖)

settingsStore (独立，操作 DOM)

dictStore (独立，会话级缓存)
```

### 8.3 初始化顺序

```
main.ts
  │
  ▼
createPinia() → app.use(pinia)
  │
  ▼
app.use(router)
  │
  ▼
router.beforeEach (permission guard)
  │
  ▼

  │
  ▼

```

## 9. 性能考虑

### 9.1 路由懒加载

所有页面组件通过动态 `import()` 懒加载，按需下载：

```typescript
component: () => import("@/views/system/user/index.vue")
```

Vite 构建时自动进行代码分割，每个页面生成独立 chunk。

### 9.2 组件按需引入

Element Plus 通过 `unplugin-vue-components` + `unplugin-auto-import` 实现自动按需引入，避免全量打包。

### 9.3 图表按需引入

ECharts 通过按需引入减小包体积：

```typescript
import * as echarts from "echarts/core";
import { LineChart, BarChart, PieChart } from "echarts/charts";
import { GridComponent, TooltipComponent, LegendComponent } from "echarts/components";
import { CanvasRenderer } from "echarts/renderers";

echarts.use([LineChart, BarChart, PieChart, GridComponent, TooltipComponent, LegendComponent, CanvasRenderer]);
```

### 9.4 字典数据缓存

字典数据加载后缓存到 `dictStore`（Pinia），同一字典码只请求一次，后续从内存读取。

### 9.5 路由缓存

动态路由生成后存储在 `permissionStore.routes`，页面刷新时重新加载（守卫检测到 `userInfo` 为空时重新初始化）。

### 9.6 表格虚拟滚动

大数据量表格使用 vxe-table 4.x 的虚拟滚动能力，仅渲染可视区域内的行。

### 9.7 图片懒加载

富文本中的图片使用 `loading="lazy"` 属性延迟加载。

### 9.8 构建优化

- **Tree Shaking**：Vite 生产构建自动移除未使用代码
- **Gzip 压缩**：`vite-plugin-compression` 生成 `.gz` 文件
- **CDN 外置**：大型依赖（Vue、Element Plus、ECharts）可通过 `vite-plugin-externals` 外置到 CDN
- **分包策略**：`manualChunks` 配置将 node_modules 拆分为独立 vendor chunk
