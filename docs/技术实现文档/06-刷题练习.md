## 一、模块概述

刷题练习模块是题小助平台的核心学习功能，为用户提供多样化的练习方式和全面的练习数据追踪。涵盖以下核心功能：

- **练习会话管理**：基于 session_id 的练习生命周期管理，支持开始、答题、暂停、恢复、提交、自动超时结束
- **多种练习模式**：随机练习、顺序练习、错题重练、收藏练习、模拟考试、闯关模式、计时模式，共 7 种模式
- **即时评判**：答题后即时判断对错，选择题支持逐选项解析，支持主观题用户自判
- **练习统计**：总练习题数、正确率、分类统计、趋势分析、排行榜等多维度数据
- **每日练习与打卡**：每日推荐题目、练习打卡、连续打卡奖励、打卡日历
- **练习计划**：创建练习计划、按计划执行、进度追踪、暂停/恢复
- **闯关模式**：按难度递进的关卡挑战，前置关卡解锁机制
- **模拟考试**：模拟真实考试环境，限时计分，支持自定义配置
- **题目互动**：收藏（含收藏夹分组）、点赞、笔记、难度评价、质量评分、纠错

本模块与**题库管理**（题目来源）、**错题本**（错题收录与重练）、**数据统计**（聚合分析）三个模块深度关联。

**请求处理流程**：

```
客户端请求 (开始练习/答题/提交)
    │
    ▼
Router (路由匹配: /api/v1/practice/*)
    │
    ▼
Middleware 链
    ├── CORS 中间件 (跨域处理)
    ├── RateLimit 中间件 (IP 限流)
    ├── Auth 中间件 (JWT 解析 + 黑名单检查)
    └── Permission 中间件 (RBAC 权限校验, 仅管理员接口)
    │
    ▼
Controller (参数绑定 + 校验 + 提取 user_id)
    │
    ▼
Service (业务逻辑: 出题算法、答案验证、统计计算、错题收录)
    │
    ├── Repository (GORM 数据库操作)
    │
    ├── questionRepo (题目查询, 共享题库模块)
    │
    └── wrongRepo (错题收录, 共享错题本模块)
    │
    ▼
Model (PracticeSession, PracticeRecord, MockExamConfig, ...)
```

## 四、涉及文件

### 后端文件

**路由层 (Router)**：

- `internal/router/router.go` — 路由总入口，实例化 PracticeController，注册到 authorized 路由组
- `internal/router/practice.go` — 练习路由定义，基础练习、模拟考试、练习计划、每日练习、打卡、排行榜、闯关模式

**控制器层 (Controller)**：

- `internal/controller/practice/practice_controller.go` — 练习历史、统计、模拟考试 CRUD、练习计划 CRUD、每日练习、打卡、排行榜、闯关列表/详情/进度
- `internal/controller/practice/session.go` — 练习会话生命周期：StartPractice、SubmitPractice、GetPracticeResult、FinishPractice、StartMockExam、SubmitMockExam、StartChallenge、SubmitChallenge

**业务逻辑层 (Service)**：

- `internal/service/practice/practice_service.go` — 核心练习逻辑：StartPractice（出题+创建会话）、SubmitPractice（单题答题+即时评判+错题收录）、FinishPractice（结算统计）、GetPracticeResult、GetPracticeHistory、GetPracticeStats
- `internal/service/practice/advanced_service.go` — 高级功能：模拟考试（Start/Submit/History/Detail）、练习计划（CRUD/Execute）、每日练习、打卡（连续天数+奖励）、排行榜、闯关模式（关卡列表/开始/提交/进度+前置关卡解锁）

**数据访问层 (Repository)**：

- `internal/repository/practice/practice_repo.go` — 基础数据访问：PracticeSession CRUD、PracticeRecord 批量创建/查询、统计聚合查询
- `internal/repository/practice/advanced_repo.go` — 高级数据访问：MockExamConfig、PracticePlan、DailyPractice、PracticeCheckin、Leaderboard、UserAbilityProfile、ChallengeLevel、UserChallengeProgress

**数据模型 (Model)**：

- `internal/model/practice.go` — PracticeSession（练习会话）、PracticeRecord（练习记录）
- `internal/model/practice_advanced.go` — MockExamConfig、PracticePlan、PracticePlanLog、DailyPractice、PracticeCheckin、UserAbilityProfile、PracticeLeaderboard、ChallengeLevel、UserChallengeProgress
- `internal/model/question_favorite.go` — FavoriteFolder（收藏夹）、QuestionFavorite（题目收藏）、QuestionLike（题目点赞）
- `internal/model/question_interaction.go` — QuestionNote（题目笔记）、QuestionDifficultyRating（难度评价）、QuestionQualityRating（质量评分）、QuestionCorrection（题目纠错）
- `internal/model/question.go` — Question、Option、Category、Knowledge（题目相关模型，被练习模块引用）
- `internal/model/wrong.go` — WrongQuestion（错题模型，练习答错时自动收录）

**数据传输对象 (DTO)**：

- `internal/dto/practice_dto.go` — PracticeSessionInfo、PracticeRecordInfo、StartPracticeRequest、SubmitPracticeAnswerRequest、PracticeListRequest
- `internal/dto/practice_advanced_dto.go` — 模拟考试 DTO（StartMockExamRequest、MockExamSessionInfo、SubmitMockExamRequest、MockExamResultInfo 等）、练习计划 DTO（CreatePlanRequest、PlanInfo 等）、每日练习 DTO、打卡 DTO、排行榜 DTO、闯关 DTO

## 五、数据库表

刷题练习模块涉及以下数据表：

### 5.1 核心表

| 表名 | 说明 | 关键字段 | 关联关系 |
|------|------|----------|----------|
| `practice_sessions` | 练习会话表 | id, user_id (FK→users), category_id (FK→categories), total_count, correct_count, accuracy, duration, status (0=进行中,1=已完成) | N:1 关联 users 和 categories；被 practice_records 引用 |
| `practice_records` | 练习记录表 | id, session_id (FK→practice_sessions), question_id (FK→questions), answer, is_correct, duration | N:1 关联 practice_sessions 和 questions |

### 5.2 扩展表

| 表名 | 说明 | 关键字段 | 关联关系 |
|------|------|----------|----------|
| `mock_exam_configs` | 模拟考试配置表 | id, name, category_id, question_count, total_score, pass_score, duration, shuffle, show_answer, difficulty_mix, mode, max_attempts, status, creator_id | N:1 关联 categories 和 users |
| `practice_plans` | 练习计划表 | id, user_id, name, plan_type (1=每日,2=每周,3=自定义), category_id, question_type, difficulty, daily_count, daily_duration, start_date, end_date, total_target, total_done, progress, status (0=暂停,1=进行中,2=已完成,3=已过期) | N:1 关联 users 和 categories |
| `practice_plan_logs` | 练习计划执行记录表 | id, plan_id (FK→practice_plans), user_id, session_id, question_num, duration, accuracy, date | N:1 关联 practice_plans |
| `daily_practices` | 每日练习记录表 | id, user_id, date (YYYY-MM-DD), total_count, correct_count, accuracy, duration, session_count, wrong_count, new_question, category_stats (JSON) | 唯一索引 (user_id, date) |
| `practice_checkins` | 练习打卡表 | id, user_id, date (YYYY-MM-DD), is_checkin, question_count, duration, streak, reward | 唯一索引 (user_id, date) |
| `user_ability_profiles` | 用户能力模型表 | id, user_id, total_questions, total_correct, total_duration, overall_accuracy, consecutive_days, max_consecutive, total_checkins, level, experience, category_abilities (JSON), weak_points (JSON), strong_points (JSON), rank_score, last_practice_at | 唯一索引 (user_id) |
| `practice_leaderboards` | 练习排行榜快照表 | id, user_id, nickname, avatar, rank_type (1=日榜,2=周榜,3=月榜,4=总榜), rank_date, rank_pos, score, accuracy, total_count, duration | 唯一索引 (user_id, rank_type, rank_date) |
| `challenge_levels` | 闯关配置表 | id, name, description, level (关卡序号), category_id, question_count, pass_accuracy, pass_score, time_limit, difficulty, icon, badge, pre_level (前置关卡), status, sort | 唯一索引 (category_id, level) |
| `user_challenge_progress` | 用户闯关进度表 | id, user_id, challenge_level (FK→challenge_levels), status (0=未开始,1=进行中,2=已通关,3=失败), best_accuracy, best_duration, attempts, passed_at, score | 唯一索引 (user_id, challenge_level) |
| `question_favorites` | 题目收藏表 | id, user_id, question_id, folder_id (0=默认收藏夹), note | 唯一索引 (user_id, question_id) |
| `favorite_folders` | 收藏夹表 | id, user_id, name, description, icon, sort, count, is_default | N:1 关联 users |
| `question_likes` | 题目点赞表 | id, user_id, question_id | 唯一索引 (user_id, question_id) |
| `question_notes` | 题目笔记表 | id, user_id, question_id, content, is_public, like_count | 唯一索引 (user_id, question_id) |
| `question_difficulty_ratings` | 题目难度评价表 | id, user_id, question_id, rating (1-5), is_correct | 唯一索引 (user_id, question_id) |
| `question_quality_ratings` | 题目质量评分表 | id, user_id, question_id, score (1-5), clarity_score, relevance_score, comment | 唯一索引 (user_id, question_id) |
| `question_corrections` | 题目纠错表 | id, user_id, question_id, correction_type (1-6), description, screenshot, status (0=待处理,1=已采纳,2=已驳回,3=已修复), admin_reply, processed_by, processed_at, reward | N:1 关联 users 和 questions |

### 5.3 索引设计

- `practice_sessions` 表：user_id (index), category_id (index), deleted_at (index)
- `practice_records` 表：session_id (index), question_id (index)
- `practice_plans` 表：user_id (index), status (index), deleted_at (index)
- `practice_plan_logs` 表：plan_id (index), user_id (index), session_id (index)
- `practice_leaderboards` 表：user_id (index), rank_type + rank_date (联合索引)
- `challenge_levels` 表：category_id (index), level (index), status (index)
- `user_challenge_progress` 表：(user_id, challenge_level) 联合唯一索引
- `question_favorites` 表：(user_id, question_id) 联合唯一索引, folder_id (index)
- `question_corrections` 表：user_id (index), question_id (index), status (index)


**关键设计决策**：

- 每答一题立即写入 practice_records 表（幂等覆盖，同一 session + question_id 仅保留最新答案）
- session 结束时才更新 session 统计信息（total_count、correct_count、accuracy）
- session 超过 24 小时未操作由定时任务自动结束（status 设为 1）
- 同一用户同时只能有一个进行中的 session（防止重复创建）

### 7.2 练习模式

系统支持 7 种练习模式，通过不同的出题策略和会话配置实现：

#### 7.2.1 随机练习 (Random)

```
出题流程:
1. 根据 category_id 获取分类及其所有子分类的 ID 列表
2. 构建查询条件: category_id IN (ids) AND status = 1 (已发布)
3. 可选附加条件: difficulty, type, visibility
4. 使用 ORDER BY RAND() LIMIT count 随机抽取
5. 返回题目列表

实现位置: StartPractice() → questionRepo.List()
```

#### 7.2.2 顺序练习 (Sequential)

```
出题流程:
1. 根据 category_id 获取分类下所有题目
2. 按题目 ID 升序排列
3. 支持从指定题目 ID 开始（断点续练）
4. 分页获取，每次返回 count 道题目

实现位置: StartPractice() 中通过 sort 参数控制排序方式
```

#### 7.2.3 错题重练 (Wrong-Focused)

```
出题流程:
1. 从 wrong_questions 表查询用户未掌握的错题 (mastered = false)
2. 可选按 category_id 筛选
3. 优先选择错题次数多、最近复习时间远的题目（间隔重复算法）
4. 创建 PracticeSession，session 标记来源为 wrong_book
5. 答题后更新错题本: 连续正确 3 次标记为已掌握

实现位置: StartPractice() → wrongRepo.FindByUser()
关联模块: internal/service/wrong/
```

#### 7.2.4 收藏练习 (Favorite)

```
出题流程:
1. 从 question_favorites 表查询用户收藏的题目
2. 可选按 folder_id 筛选指定收藏夹
3. 随机或顺序出题
4. 创建 PracticeSession

实现位置: StartPractice() 中 mode=favorite 分支
关联表: question_favorites
```

#### 7.2.5 模拟考试 (Mock Exam)

```
出题流程:
1. 根据 config_id 获取 MockExamConfig 配置
2. 检查最大尝试次数限制
3. 根据配置的 mode 决定出题方式:
   - mode=1: 指定试卷 (PaperID)
   - mode=2: 随机抽题 (按 category_id + difficulty_mix)
   - mode=3: 错题重练
   - mode=4: 收藏题练习
4. 如需打乱题目顺序 (shuffle=true)，使用 Fisher-Yates 洗牌算法
5. 创建 PracticeSession (复用 session 模型，CategoryID 存储 ConfigID)
6. 答题结束后按 config.total_score 计算得分
7. 判断是否及格 (score >= config.pass_score)

实现位置: advanced_service.go → StartMockExam(), SubmitMockExam()
```

#### 7.2.6 闯关模式 (Challenge)

```
出题流程:
1. 根据 level_id 获取 ChallengeLevel 配置
2. 检查前置关卡是否已通关 (pre_level > 0 时)
3. 按 category_id + difficulty 出题，题目数量 = level.question_count
4. 创建 PracticeSession (CategoryID 存储 levelID)
5. 更新或创建 UserChallengeProgress (attempts++)
6. 答题结束后:
   - accuracy >= level.pass_accuracy → 通关
   - 更新 best_accuracy, best_duration
   - 通关时记录 passed_at, 奖励积分
   - 失败时标记 status=3

实现位置: advanced_service.go → StartChallenge(), SubmitChallenge()
```

#### 7.2.7 计时模式 (Timed)

```
出题流程:
1. 设置时间限制 (duration 秒)
2. 随机出题，题目数量不限
3. 前端维护倒计时
4. 时间到自动提交
5. 统计时间内答对的题目数

实现位置: 复用 PracticeSession，duration 字段存储时间限制
```

### 7.3 答题验证

答题验证是练习模块的核心逻辑，根据不同题型采用不同的验证策略：

```
SubmitPractice(sessionID, userID, request)
    │
    ├── 1. 会话校验
    │   ├── session 存在性检查
    │   ├── session.UserID == userID (权限校验)
    │   └── session.Status == 0 (必须是进行中)
    │
    ├── 2. 获取题目信息
    │   └── questionRepo.FindByID(questionID) → Question (含 Answer 字段)
    │
    ├── 3. 按题型验证答案
    │   │
    │   ├── 单选题 (type=1)
    │   │   ├── question.Answer = "A" (单个选项标签)
    │   │   └── isCorrect = (userAnswer == question.Answer)
    │   │
    │   ├── 多选题 (type=2)
    │   │   ├── question.Answer = "A,B,C" (逗号分隔，已排序)
    │   │   ├── 将 userAnswer 按逗号分隔并排序
    │   │   └── isCorrect = (sortedUserAnswer == sortedCorrectAnswer)
    │   │
    │   ├── 判断题 (type=3)
    │   │   ├── question.Answer = "true" 或 "false"
    │   │   └── isCorrect = (userAnswer == question.Answer)
    │   │
    │   ├── 填空题 (type=4)
    │   │   ├── question.Answer = "答案1|答案2" (多个可接受答案用 | 分隔)
    │   │   ├── 逐一比较，任一匹配即正确
    │   │   └── 比较时忽略前后空格，可选忽略大小写
    │   │
    │   └── 简答题 (type=5)
    │       ├── 练习模式: 用户自判 (userAnswer = "correct" 或 "wrong")
    │       │   └── isCorrect = (userAnswer == "correct")
    │       └── 显示标准答案和解析供用户参考
    │
    ├── 4. 保存答题记录
    │   └── INSERT INTO practice_records (session_id, question_id, answer, is_correct, duration)
    │       ON DUPLICATE KEY UPDATE answer=VALUES(answer), is_correct=VALUES(is_correct), duration=VALUES(duration)
    │       (幂等: 同一 session + question_id 仅保留最新答案)
    │
    └── 5. 错题收录
        ├── isCorrect == false 且 source = practice:
        │   ├── 查询 wrong_questions 是否已存在该 user_id + question_id
        │   ├── 已存在: wrong_count++, last_answer=新答案, mastered=false
        │   └── 不存在: INSERT wrong_questions (user_id, question_id, source=1, wrong_count=1)
        │
        └── 主观题用户自判为错:
            └── 同上逻辑，source=1 (practice)
```

**关键代码路径**：

- Controller: `internal/controller/practice/session.go` → `SubmitPractice()`
- Service: `internal/service/practice/practice_service.go` → `SubmitPractice()`
- 答案比较: `question.Answer == req.Answer` (简单字符串比较，多选题需排序后比较)
- 错题收录: `internal/service/practice/practice_service.go` → `updateWrongQuestion()`

### 7.4 正确率计算

正确率在两个层面计算：单次练习的实时正确率和用户维度的聚合正确率。

#### 7.4.1 单次练习正确率

```
FinishPractice(sessionID)
    │
    ├── 查询 practice_records WHERE session_id = ?
    ├── total = len(records)
    ├── correct = count(records where is_correct = true)
    └── accuracy = correct / total * 100  (保留两位小数)

更新 practice_sessions:
    ├── correct_count = correct
    ├── accuracy = accuracy
    └── status = 1 (已完成)
```

#### 7.4.2 用户全局正确率

```
GetUserPracticeStats(userID)
    │
    ├── SELECT COUNT(*) FROM practice_sessions WHERE user_id = ? → totalSessions
    ├── SELECT COUNT(*) FROM practice_records r
    │   JOIN practice_sessions s ON r.session_id = s.id
    │   WHERE s.user_id = ? → totalRecords
    ├── SELECT COUNT(*) FROM practice_records r
    │   JOIN practice_sessions s ON r.session_id = s.id
    │   WHERE s.user_id = ? AND r.is_correct = true → correctRecords
    └── accuracy = correctRecords / totalRecords * 100
```

#### 7.4.3 每日正确率

```
CompleteDailyPractice(userID, req)
    │
    ├── 查找或创建 daily_practices 记录 (user_id + date)
    ├── total_count += req.QuestionCount
    ├── correct_count += req.CorrectCount
    ├── accuracy = correct_count / total_count * 100
    └── 更新 daily_practices 表
```

### 7.5 练习统计

练习统计提供多维度的数据分析，支持实时查询和定期聚合两种模式。

#### 7.5.1 统计概览

```
GetPracticeStats(userID) → 返回:
    {
      "total_sessions": 150,        // 总练习次数
      "total_records": 3000,        // 总答题数
      "correct_records": 2400,      // 总正确数
      "accuracy": 80.0              // 总正确率 (%)
    }
```

**实现细节**：

- 通过 practice_sessions 和 practice_records 表 JOIN 查询聚合
- 使用数据库 COUNT 聚合函数，避免将大量数据加载到内存
- 可通过 Redis 缓存统计结果，TTL = 5 分钟，练习完成后主动失效

#### 7.5.2 分类统计

```
GetCategoryStats(userID) → 按分类聚合:
    [
      {
        "category_id": 1,
        "category_name": "数学",
        "total_count": 500,
        "correct_count": 400,
        "accuracy": 80.0
      },
      ...
    ]
```

**实现方式**：

- GROUP BY category_id 聚合 practice_sessions
- 关联 categories 表获取分类名称
- 支持按时间范围筛选

#### 7.5.3 趋势分析

```
GetPracticeTrend(userID, days) → 按日聚合最近 N 天数据:
    [
      {
        "date": "2026-05-01",
        "count": 30,
        "correct": 24,
        "accuracy": 80.0,
        "duration": 1800
      },
      ...
    ]
```

**实现方式**：

- 查询 daily_practices 表（已预聚合）
- 或从 practice_sessions 按 DATE(created_at) GROUP BY 聚合
- 支持 7 天、30 天、90 天时间范围

#### 7.5.4 排行榜统计

```
排行榜计算 (定时任务):
    │
    ├── 日榜: 汇总当日 practice_sessions 数据
    ├── 周榜: 汇总本周数据
    ├── 月榜: 汇总本月数据
    └── 总榜: 汇总历史所有数据

    排名维度:
    ├── 按题数排名 (rank_count): total_count DESC
    ├── 按正确率排名 (rank_accuracy): accuracy DESC (需满足最低题数门槛)
    └── 综合排名: score = total_count * 0.4 + accuracy * 0.6

    写入 practice_leaderboards 表
```

### 7.6 题目选择算法

题目选择是练习质量的关键，系统实现了多种选择算法：

#### 7.6.1 随机选择 (Random Selection)

```
算法: ORDER BY RAND() LIMIT count

适用场景: 随机练习、模拟考试随机抽题

优化策略:
1. 先统计符合条件的总题目数 (SELECT COUNT(*))
2. 如果总数 <= count，直接返回全部
3. 如果总数 > count，使用 RAND() 抽取
4. 大数据量场景下，使用 "抽样 ID + 批量查询" 两步法:
   - SELECT id FROM questions WHERE ... ORDER BY RAND() LIMIT count
   - SELECT * FROM questions WHERE id IN (ids)
```

#### 7.6.2 顺序选择 (Sequential Selection)

```
算法: ORDER BY id ASC, 从上次断点继续

适用场景: 顺序练习、按章节逐题练习

实现:
1. 查询用户在该分类下最后一次练习的最后题目 ID
2. 从该 ID 之后开始取题
3. 记录断点信息到 Redis: qh:practice:breakpoint:{user_id}:{category_id} = last_question_id
```

#### 7.6.3 自适应选择 (Adaptive Selection)

```
算法: 基于用户能力模型的自适应出题

实现流程:
1. 查询 user_ability_profiles 获取用户在该分类下的能力值
   - difficulty_score: 推荐难度分数 (1.0-5.0)
   - ability_level: 能力等级 (1=入门 ... 5=精通)

2. 题目选择策略:
   ├── 60% 题目: 难度 ≈ 用户能力等级
   ├── 20% 题目: 难度 = 用户能力等级 + 1 (挑战题)
   └── 20% 题目: 难度 = 用户能力等级 - 1 (巩固题)

3. 每次练习后更新能力模型:
   ├── 本次正确率 > 80%: difficulty_score += 0.1
   ├── 本次正确率 < 50%: difficulty_score -= 0.1
   ├── difficulty_score 限制在 [1.0, 5.0] 范围
   └── 更新 ability_level (基于 difficulty_score 映射)

4. 知识点薄弱补充:
   ├── 分析用户答错题目关联的知识点
   ├── weak_points 字段记录薄弱知识点
   └── 出题时优先从薄弱知识点中选题
```

#### 7.6.4 错题优先选择 (Wrong-Focused Selection)

```
算法: 间隔重复 (Spaced Repetition) 简化版

实现:
1. 查询 wrong_questions WHERE user_id = ? AND mastered = false
2. 优先级排序:
   ├── wrong_count DESC (错得越多越优先)
   ├── next_review_at ASC (到期复习的优先)
   └── last_review_at ASC (很久没复习的优先)
3. 选题后更新:
   ├── last_review_at = now
   ├── next_review_at = now + interval (基于 correct_streak 计算)
   │   ├── correct_streak = 0: 1 天后
   │   ├── correct_streak = 1: 3 天后
   │   ├── correct_streak = 2: 7 天后
   │   └── correct_streak >= 3: 标记为 mastered
   └── review_count++
```

### 7.7 收藏与点赞

收藏和点赞是题目互动的基础功能，支持收藏夹分组管理。

#### 7.7.1 收藏功能

```
收藏题目:
    POST /api/v1/questions/:id/favorite
    │
    ├── 1. 检查题目是否存在
    ├── 2. 检查是否已收藏 (user_id + question_id 唯一索引)
    │   ├── 已收藏 → 返回"已收藏"
    │   └── 未收藏 → 继续
    ├── 3. 获取或创建默认收藏夹 (folder_id=0 时)
    ├── 4. INSERT question_favorites (user_id, question_id, folder_id, note)
    ├── 5. UPDATE questions SET favorite_count = favorite_count + 1
    └── 6. 更新收藏夹 count 字段

取消收藏:
    DELETE /api/v1/questions/:id/favorite
    │
    ├── 1. DELETE question_favorites WHERE user_id = ? AND question_id = ?
    ├── 2. UPDATE questions SET favorite_count = favorite_count - 1
    └── 3. 更新收藏夹 count 字段

收藏夹管理:
    ├── 创建收藏夹: INSERT favorite_folders
    ├── 更新收藏夹: UPDATE favorite_folders
    ├── 删除收藏夹: 将夹内收藏移到默认收藏夹，再删除
    └── 移动收藏: UPDATE question_favorites SET folder_id = ?

查询收藏列表:
    GET /api/v1/questions/favorites?folder_id=1&page=1&page_size=10
    │
    └── SELECT * FROM question_favorites f
        JOIN questions q ON f.question_id = q.id
        WHERE f.user_id = ? AND f.folder_id = ?
        ORDER BY f.created_at DESC
```

#### 7.7.2 点赞功能

```
点赞/取消点赞 (Toggle):
    POST /api/v1/questions/:id/like
    │
    ├── 1. 检查是否已点赞 (user_id + question_id 唯一索引)
    │   ├── 已点赞:
    │   │   ├── DELETE question_likes WHERE user_id = ? AND question_id = ?
    │   │   ├── UPDATE questions SET like_count = like_count - 1
    │   │   └── 返回 { liked: false, count: like_count }
    │   └── 未点赞:
    │       ├── INSERT question_likes (user_id, question_id)
    │       ├── UPDATE questions SET like_count = like_count + 1
    │       └── 返回 { liked: true, count: like_count }
    │
    └── 使用数据库唯一索引防止并发重复点赞
```

## 八、Redis 缓存设计

### 8.1 练习相关 Key

| Key Pattern | 数据类型 | TTL | 说明 |
|-------------|---------|-----|------|
| `qh:practice:session:{session_id}` | Hash | 24h | 进行中的练习会话缓存（状态、进度、已答题目列表） |
| `qh:practice:breakpoint:{user_id}:{category_id}` | String | 7d | 顺序练习断点（最后答题的题目 ID） |
| `qh:practice:stats:{user_id}` | Hash | 5min | 用户练习统计缓存（总题数、正确率等），练习完成后主动失效 |
| `qh:practice:daily:{user_id}:{date}` | Hash | 48h | 每日练习数据缓存 |
| `qh:practice:checkin:{user_id}:{date}` | String | 48h | 今日打卡状态缓存 ("1" = 已打卡) |
| `qh:practice:leaderboard:{rank_type}` | Sorted Set | 1h | 排行榜缓存（member=user_id, score=排行分数） |
| `qh:practice:leaderboard:my:{user_id}` | Hash | 1h | 我的排名缓存 |
| `qh:practice:challenge:progress:{user_id}` | Hash | 30min | 闯关进度缓存 |
| `qh:practice:rate:{user_id}` | String (INCR) | 60s | 答题接口限流（每分钟最多 60 次提交） |
| `qh:practice:recent:{user_id}` | List | 30d | 最近浏览的分类（最多 20 条） |

### 8.2 题目互动相关 Key

| Key Pattern | 数据类型 | TTL | 说明 |
|-------------|---------|-----|------|
| `qh:question:like:{question_id}` | Set | - | 题目点赞用户集合（用于快速判断是否已点赞） |
| `qh:question:favorite:{user_id}` | Set | - | 用户收藏题目集合（用于快速判断是否已收藏） |
| `qh:question:hot` | Sorted Set | 1h | 热门题目排行（score=view_count+like_count*3+answer_count*2） |

### 8.3 缓存更新策略

```
练习完成后缓存更新流程:
    │
    ├── 1. 更新 practice_sessions 表 (正确数、正确率、状态)
    │
    ├── 2. 失效用户统计缓存
    │   └── Redis DEL qh:practice:stats:{user_id}
    │
    ├── 3. 更新每日练习缓存
    │   └── Redis HINCRBY qh:practice:daily:{user_id}:{date} total_count / correct_count
    │
    ├── 4. 更新排行榜 (如使用 Redis Sorted Set)
    │   └── Redis ZINCRBY qh:practice:leaderboard:{rank_type} score user_id
    │
    ├── 5. 更新题目统计
    │   └── Redis HINCRBY qh:question:hot view_count/answer_count
    │
    └── 6. 失效闯关进度缓存 (闯关模式时)
        └── Redis DEL qh:practice:challenge:progress:{user_id}
```

## 九、安全考虑

### 9.1 接口安全

| 措施 | 说明 |
|------|------|
| JWT 认证 | 所有练习接口需 JWT 认证，通过 AuthMiddleware 校验 |
| 用户隔离 | 所有操作校验 session.UserID == 当前用户 ID，防止越权操作 |
| 接口限流 | 答题提交接口限流 60 次/分钟/用户，防止刷题作弊 |
| 参数校验 | DTO binding tags 严格校验参数类型和范围 |

### 9.2 数据安全

| 措施 | 说明 |
|------|------|
| 软删除 | practice_sessions 使用 deleted_at 软删除 |
| 幂等提交 | 同一 session + question_id 的答题记录使用 UPSERT，仅保留最新答案 |
| 会话超时 | 超过 24 小时未操作的 session 自动结束，防止无限占用 |
| 防重复点赞 | question_likes 表 (user_id, question_id) 唯一索引 |
| 防重复收藏 | question_favorites 表 (user_id, question_id) 唯一索引 |

### 9.3 业务安全

| 措施 | 说明 |
|------|------|
| 闯关前置检查 | 闯关前验证前置关卡是否已通关 |
| 模拟考试次数限制 | 检查 max_attempts 限制，防止无限重考 |
| 练习计划状态检查 | 执行计划前检查 status == 1 (进行中) |
| 打卡防重复 | 同一天只允许打卡一次，通过数据库唯一索引保证 |
| 连续打卡计算 | 基于前一天打卡记录计算连续天数，防止人为篡改 |

### 9.4 性能安全

| 措施 | 说明 |
|------|------|
| 分页查询 | 练习历史、排行榜等列表接口强制分页，单页最大 100 条 |
| 统计缓存 | 统计数据缓存 5 分钟，避免高频聚合查询 |
| 批量操作 | 答题记录使用批量 INSERT，减少数据库交互 |
| 索引覆盖 | 所有高频查询字段均建立索引 |
