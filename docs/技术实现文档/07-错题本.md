## 一、模块概述

错题本系统是题小助平台的核心学习辅助模块，旨在帮助用户系统化管理和复习做错的题目，通过科学的记忆算法提升学习效率。模块涵盖以下核心功能：

- **错题自动收集**：练习答错和考试交卷后客观题答错时自动写入错题本，按 `user_id + question_id` 去重，累计错误次数
- **来源追踪**：记录错题来源（1=练习 / 2=考试）及来源记录 ID，支持追溯原始答题场景
- **错题复习流程**：基于艾宾浩斯遗忘曲线的间隔重复算法，重做后标记掌握/未掌握，连续答对 N 次自动移除
- **错题统计分析**：错题总量、按分类/知识点/题型/难度分布、趋势图、薄弱知识点排行、正确率分析
- **错题导出**：支持 CSV/Excel 格式导出，包含题目、答案、解析、错误次数等完整信息
- **错题分析与推荐**：薄弱区域识别、学习建议、同类题目推荐、错题预测

**错题收集触发流程**：

```
练习答错 / 考试交卷
    │
    ▼
updateWrongQuestion(userID, questionID, source, sourceID, answer)
    │
    ├── 查询 wrongRepo.FindByUserAndQuestion(userID, questionID)
    │       │
    │       ├── 已存在 → WrongCount++, LastAnswer 更新, Mastered 重置
    │       │
    │       └── 不存在 → 创建新 WrongQuestion 记录
    │
    ▼
错题本更新完成
```

**错题复习流程**：

```
用户点击「重做」
    │
    ▼
提交答案 (ReviewWrongQuestion)
    │
    ▼
对比标准答案 (Question.Answer == UserAnswer)
    │
    ├── 正确 → CorrectStreak++
    │       │
    │       ├── CorrectStreak >= 阈值 → Mastered = true
    │       └── 计算下次复习时间 (艾宾浩斯遗忘曲线)
    │
    └── 错误 → CorrectStreak = 0, WrongCount++, Mastered = false
            │
            └── 计算下次复习时间 (20分钟后)
    │
    ▼
创建复习记录 (WrongQuestionReview)
更新错题状态 (WrongQuestion)
    │
    ▼
返回结果
```

## 七、核心实现

### 7.1 错题自动收集

**触发时机**：
1. 练习模式答错时（`practice_service.go` 第 101-103 行）
2. 考试交卷后客观题答错时（待集成到 `exam_submit_job.go`）

**去重策略**：按 `user_id + question_id` 唯一标识一条错题记录。已存在的错题仅更新 `wrong_count` 和 `last_answer`，不重复创建。

**核心函数** (`practice_service.go` 第 226-248 行)：

```go
func updateWrongQuestion(userID, questionID, source, sourceID uint, answer string) {
    existing, err := wrongRepo.FindByUserAndQuestion(userID, questionID)
    if err == nil {
        // 已存在，更新错误次数
        existing.WrongCount++
        existing.LastAnswer = answer
        existing.Mastered = false  // 重置掌握状态
        wrongRepo.Update(existing)
        return
    }

    // 不存在，创建新记录
    wrong := &model.WrongQuestion{
        UserID:     userID,
        QuestionID: questionID,
        Source:     int8(source),
        SourceID:   sourceID,
        WrongCount: 1,
        LastAnswer: answer,
        Mastered:   false,
    }
    wrongRepo.Create(wrong)
}
```

**考试交卷集成方案**（待实现）：

在 `exam_submit_job.go` 的 `Handle` 方法中，自动评分完成后遍历答案记录，对答错的客观题调用错题收集：

```go
// 在自动评分完成后
```

### 7.3 错题复习流程

**艾宾浩斯遗忘曲线算法**：

系统使用 9 级间隔的艾宾浩斯遗忘曲线算法控制复习节奏：

```go
// 艾宾浩斯遗忘曲线间隔（单位：分钟）
var ebbinghausIntervals = []int{
    1,        // 20分钟后（答错重置）
    60,       // 1小时后（第1次答对）
    720,      // 12小时后（第2次答对）
    1440,     // 1天后（第3次答对）
    2880,     // 2天后（第4次答对）
    4320,     // 3天后（第5次答对）
    10080,    // 7天后（第6次答对）
    30240,    // 21天后（第7次答对）
    43200,    // 30天后（第8次答对）
}
```

**复习处理逻辑** (`processReview` 函数)：

```go
func processReview(wrong *model.WrongQuestion, req *dto.ReviewWrongAdvancedRequest) error {
    now := time.Now()

    // 创建复习记录
    review := &model.WrongQuestionReview{
        WrongQuestionID: wrong.ID,
        UserID:          wrong.UserID,
        IsCorrect:       req.IsCorrect,
        Answer:          req.Answer,
        Duration:        req.Duration,
        ReviewType:      req.ReviewType,
    }

    if req.IsCorrect {
        wrong.CorrectStreak++
        // 连续答对次数达到间隔总数时标记为已掌握
        wrong.Mastered = wrong.CorrectStreak >= len(ebbinghausIntervals)
    } else {
        wrong.CorrectStreak = 0
        wrong.Mastered = false
        wrong.WrongCount++
    }

    wrong.LastAnswer = req.Answer
    wrong.LastReviewAt = &now
    wrong.ReviewCount++

    // 计算下次复习时间
    nextReview := calculateNextReview(wrong.CorrectStreak)
    wrong.NextReviewAt = &nextReview
    review.NextReviewAt = &nextReview

    // 持久化
    wrongRepo.Update(wrong)
    wrongRepo.CreateReview(review)
    return nil
}
```

**下次复习时间计算**：

```go
func calculateNextReview(correctStreak int) time.Time {
    if correctStreak <= 0 {
        // 答错了，20分钟后重新复习
        return time.Now().Add(time.Duration(ebbinghausIntervals[0]) * time.Minute)
    }
    index := correctStreak - 1
    if index >= len(ebbinghausIntervals) {
        index = len(ebbinghausIntervals) - 1
    }
    interval := ebbinghausIntervals[index]
    return time.Now().Add(time.Duration(interval) * time.Minute)
}
```

**掌握标记阈值**：
- 默认连续答对 9 次（覆盖所有遗忘曲线间隔）自动标记为已掌握
- 用户可在个人设置中自定义阈值（1-10 次），通过 `retryEbbinghaus` 函数支持指数退避策略

**薄弱区域识别**：

```go
// 识别规则：

func IdentifyWeakAreas(analysis []dto.WrongCategoryAnalysisInfo) []WeakArea {
    var weakAreas []WeakArea

                CategoryID:   cat.CategoryID,
                CategoryName: cat.CategoryName,
                Severity:     calculateSeverity(cat),
                Suggestion:   generateSuggestion(cat),
            })
        }

}
```

## 八、安全考虑

### 8.1 数据隔离

- 所有错题操作均校验 `user_id`，确保用户只能访问自己的错题数据
- 标签、备注、附件、收藏等扩展数据同样绑定 `user_id`，防止越权访问

```go
// 典型的权限校验模式
if wrong.UserID != userID {

}
```

### 8.2 输入校验

- 错题搜索关键词使用参数化查询，防止 SQL 注入
- 备注内容通过敏感词过滤中间件（`middleware/sensitive.go`）审核
- 附件上传校验文件类型和大小，防止恶意文件上传
- 导出数量上限 10000 条，防止资源滥用

### 8.3 限流保护

- 错题列表/搜索接口通过 `middleware/rate_limit.go` 限流
- 导出接口单独限流，防止频繁导出消耗服务器资源
- 批量操作限制单次最多 100 条

### 8.4 软删除

- 错题删除使用 GORM 软删除（`deleted_at` 字段），数据可恢复
- 关联数据（标签关联、备注、附件、收藏、复习记录）跟随主记录逻辑删除

```go
// Service 层错误处理
func GetWrongQuestion(id, userID uint) (*dto.WrongQuestionInfo, error) {
    wrong, err := wrongRepo.FindByID(id)
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {

        }
        return nil, fmt.Errorf("查询错题失败: %w", err)

    }

    info := toWrongQuestionInfo(wrong)
    return &info, nil
}

    }
```
