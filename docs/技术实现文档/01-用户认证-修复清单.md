## P0 紧急 — 安全漏洞 / 核心功能缺失

### T01 登录支持邮箱/手机号查找

- **问题**：`auth_service.go` 仅调用 `FindByUsername`，不支持邮箱和手机号登录
- **文档**：功能需求 1.2 — 7 种登录方式
- **修复**：
  1. `repository/user/user.go` 新增 `FindByEmail(email)` 和 `FindByPhone(phone)`
  2. `service/auth/auth_service.go` Login 方法改为：先按 username 查，失败按 email 查，再失败按 phone 查
  3. 或使用 `FindByIdentifier(identifier)` 统一查询
- **涉及文件**：`internal/repository/user/user.go`, `internal/service/auth/auth_service.go`
- **验收**：使用用户名、邮箱、手机号均可登录成功

### T02 密码重置检查历史密码

- **问题**：`ResetPassword` 未调用 `GetRecentPasswords` 检查最近 3 次密码
- **文档**：功能需求 1.4 — 新密码不能与最近 3 次相同
- **修复**：`service/auth/auth_service.go` ResetPassword 方法中，更新密码前调用：
  ```go
  recentPasswords, _ := GetRecentPasswords(userID, 3)
  for _, oldHash := range recentPasswords {
      if encrypt.CheckPassword(newPassword, oldHash) == nil {
          return errors.New("A0001", "新密码不能与最近3次密码相同")
      }
  }
  ```
- **涉及文件**：`internal/service/auth/auth_service.go` (约 line 299-341)
- **验收**：重置密码时使用最近 3 次内的旧密码应被拒绝

### T03 验证码切换为 Redis 存储

- **问题**：当前使用 `base64Captcha.DefaultMemStore`（内存），多实例部署验证码不共享
- **文档**：技术实现 — Redis 缓存设计
- **修复**：
  1. `pkg/captcha/captcha.go` 中使用 `NewRedisStore()` 替代 `DefaultMemStore`
  2. 确保 `VerifyCaptchaFromRedis` 被调用而非 `base64Captcha.DefaultMemStore.Verify`
  3. 验证后自动删除（单次有效）
- **涉及文件**：`pkg/captcha/captcha.go`, `internal/service/auth/auth_service.go`
- **验收**：多实例部署下验证码可正常生成和验证

### T04 错误码使用场景化编码

- **问题**：`response.Error()` 始终返回 `"A0001"`，前端无法区分错误类型
- **文档**：API 概述 — 错误码规范
- **修复**：
  1. `pkg/response/response.go` 的 `Error()` 方法接受 code 参数，或使用 `ErrorWithCode()`
  2. `internal/service/auth/auth_service.go` 中所有错误返回使用具体错误码：
     - 用户名已存在 → `"A0200"`
     - 邮箱已存在 → `"A0201"`
     - 用户名或密码错误 → `"A0203"`
     - 账户被锁定 → `"A0204"`
     - 验证码错误 → `"A0205"`
     - Token 过期 → `"A0003"`
     - Token 无效 → `"A0004"`
  3. 所有 Controller 调用 `response.ErrorWithCode(c, httpCode, errCode, msg)`
- **涉及文件**：`pkg/response/response.go`, `internal/service/auth/*.go`, `internal/controller/auth/*.go`
- **验收**：不同错误场景返回不同错误码

### T05 敏感词中间件注册到路由

- **问题**：`SensitiveFilterMiddleware` 存在但 `router.go` 未挂载
- **文档**：技术实现 — 中间件配置
- **修复**：`internal/router/router.go` 在需要敏感词过滤的路由组中添加中间件：
  ```go
  authGroup.Use(middleware.SensitiveFilterMiddleware())
  ```
- **涉及文件**：`internal/router/router.go`
- **验收**：注册时包含敏感词的昵称被拦截

### T06 邮件发送服务实现

- **问题**：`SendEmailCode` 为空桩，注册邮箱验证和密码找回邮件均无法发送
- **文档**：功能需求 1.8 — 邮件验证码限制
- **修复**：
  1. `pkg/email/` 新建邮件发送包（使用 gomail 或 SMTP）
  2. `config/config.yaml` 添加邮件配置（SMTP host/port/user/pass/from）
  3. `service/auth/auth_service.go` 实现 `SendEmailCode`：
     - 生成 6 位随机验证码
     - 存入 Redis（TTL 5 分钟）
     - 检查 60 秒冷却（`qh:email:limit:{email}`）
     - 检查每日 10 次限制（`qh:email:daily:{email}`）
     - 发送邮件
  4. 注册流程中要求邮箱验证码
  5. 密码找回流程中发送重置邮件
- **涉及文件**：`pkg/email/` (新建), `internal/service/auth/auth_service.go`, `internal/controller/auth/captcha.go`, `config/config.yaml`
- **验收**：注册时发送验证码邮件，密码找回时发送重置邮件

### T09 密码强度正则校验

- **问题**：仅校验长度，不要求字母+数字组合
- **文档**：功能需求 1.1 — 必须包含字母和数字
- **修复**：`dto/auth_dto.go` 添加自定义验证：
  ```go
  // pkg/validator/validator.go
  func ValidatePassword(fl validator.FieldLevel) bool {
      pwd := fl.Field().String()
      hasLetter := regexp.MustCompile(`[a-zA-Z]`).MatchString(pwd)
      hasNumber := regexp.MustCompile(`[0-9]`).MatchString(pwd)
      return hasLetter && hasNumber
  }
  ```
  RegisterRequest.Password 添加 `validate:"password"` 标签
- **涉及文件**：`pkg/validator/validator.go`, `internal/dto/auth_dto.go`
