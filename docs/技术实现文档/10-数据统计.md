## 一、模块概述

数据统计模块是题小助平台的业务智能核心，提供全方位的数据采集、聚合分析与可视化能力，涵盖以下功能域：

- **仪表盘概览**：实时 KPI 展示（总用户数、总题目数、总考试数、今日活跃用户），支持数据趋势图表和时间段对比
- **用户统计**：注册趋势、活跃用户（DAU/MAU）、用户留存率、流失率、用户画像、角色分布、用户分群与路径分析
- **题库统计**：题目增长趋势、题型分布、难度分布、分类分布、热门题目排行、资源使用统计
- **考试统计**：参与率、成绩分布（平均分/最高分/最低分/中位数/分数段）、及格率、排名变化、成绩趋势
- **答题统计**：按分类/知识点的正确率、答题时长分析、完成率、改进趋势
- **班级统计**：班级概览、成员学习进度、班级成绩分析、班级间对比、创作者统计、活跃度
- **系统统计**：API 响应时间、错误率、资源使用、存储统计
- **数据订阅**：日报/周报/月报邮件推送，订阅 CRUD，定时发送
- **数据预警**：预警规则配置、实时监控、预警记录管理
- **数据导出**：Excel/CSV/PDF 导出，异步任务处理，导出记录管理
- **数据分享**：报告快照生成，Token 分享链接，访问计数

**统计分层策略**：

| 层级 | 存储 | 数据类型 | 示例指标 |
|------|------|----------|----------|
| 实时层 | Redis HyperLogLog / 计数器 | 秒级/分钟级 | DAU、在线人数、实时答题量 |
| 近线层 | MySQL (daily_statistics) | 每日聚合 | 用户增长、答题量、正确率、保留 90 天明细 |
| 离线层 | MySQL (monthly 聚合) | 每月聚合 | 留存率、转化漏斗、用户画像、保留 12 个月 |

**统计数据查询流程**：

```
客户端请求统计数据
    │
    ▼
Router (路由匹配: GET /api/v1/statistics/overview)
    │
    ▼
Middleware 链
    ├── CORS 中间件
    ├── RateLimit 中间件
    ├── Auth 中间件 (JWT 解析 + 黑名单检查)
    └── RBAC 中间件 (管理端接口需要 admin 角色)
    │
    ▼
Controller (StatisticsController.GetOverview)
    ├── 绑定 StatisticsQueryRequest DTO (start_date, end_date, granularity)
    ├── 参数校验
    └── 调用 Service 层
    │
    ▼
Service (statistics.GetOverview)
    ├── 1. 查询 Redis 缓存 (qh:stats:dashboard)
    │   ├── 缓存命中 → 直接返回
    │   └── 缓存未命中 → 继续
    ├── 2. 实时指标: Redis HyperLogLog (DAU/MAU)
    ├── 3. 历史指标: Repository 查询 statistics_snapshots 表
    ├── 4. 趋势计算: 对比上一周期数据，计算增长/下降比例
    ├── 5. 写入 Redis 缓存 (TTL 300s)
    └── 6. 返回 OverviewInfo DTO
```

**数据导出异步流程**：

```
客户端提交导出请求
    │
    ▼
Controller (StatisticsController.ExportData)
    ├── 参数校验
    └── 调用 Service 层
    │
    ▼
Service (statistics.CreateExportTask)
    ├── 1. 创建 export_records 记录 (status=1 处理中)
    ├── 2. 发送消息到 pkg/mq 队列
    └── 3. 返回 export_id
    │
    ▼
Job Worker (statistics.ExportJob)
    ├── 1. 更新 export_records status=1
    ├── 2. 查询统计数据 (根据 export_type)
    ├── 3. 使用 pkg/excel 生成 Excel/CSV 文件
    ├── 4. 保存文件到本地/MinIO
    ├── 5. 更新 export_records (status=2 已完成, file_path, file_size)
    └── 6. 异常时更新 status=3 失败 + error_msg
```

- `internal/controller/statistics/share_controller.go` — 数据分享控制器：CreateShare、GetShare
- `internal/controller/statistics/admin_statistics_controller.go` — 管理端统计控制器：GetDashboard、ExportDashboard、CompareData、GetCompareMetrics、GetRetention、GetRetentionTrend、GetChurn、GetChurnTrend

**业务逻辑层 (Service)**：

- `internal/service/statistics/dashboard_service.go` — 仪表盘业务：聚合实时指标（Redis HyperLogLog 计算 DAU/MAU、在线人数、实时答题量）和历史指标（statistics_snapshots），计算趋势对比，缓存结果
- `internal/service/statistics/user_stats_service.go` — 用户统计业务：注册趋势查询、活跃用户统计、留存率计算（register_date + retention_day 联合查询）、流失率计算、用户分群 CRUD（含动态规则引擎）、用户事件采集与分析
- `internal/service/statistics/question_stats_service.go` — 题库统计业务：题目增长趋势、题型/难度/分类分布（GROUP BY 聚合）、热门题目排行、资源使用统计
- `internal/service/statistics/exam_stats_service.go` — 考试统计业务：参与率计算、成绩分布（PERCENTILE 近似算法）、及格率、排名变化、成绩预测（线性回归）、成绩预警
- `internal/service/statistics/class_stats_service.go` — 班级统计业务：班级概览、成员学习进度、班级成绩分析、班级间对比、班级报告生成、班级预警
- `internal/service/statistics/practice_stats_service.go` — 答题统计业务：按分类/知识点正确率、答题时长分析、完成率、改进趋势、题目难度/区分度分析
- `internal/service/statistics/subscription_service.go` — 数据订阅业务：订阅 CRUD、报告生成（daily/weekly/monthly）、邮件发送（调用通知系统）
- `internal/service/statistics/share_service.go` — 数据分享业务：生成分享 Token、报告快照持久化、访问计数、过期检查
- `internal/service/statistics/export_service.go` — 数据导出业务：创建导出任务、生成 Excel/CSV 文件（pkg/excel）、文件过期清理
- `internal/service/statistics/comparison_service.go` — 数据对比业务：不同时间段指标对比、对比快照生成与查询

## 七、核心实现

### 7.1 仪表盘

**设计目标**：提供平台全局数据概览，实时 KPI 来自 Redis，历史趋势来自 MySQL。

**实时 KPI 获取**：

```
实时指标通过 Redis 实现，避免每次请求扫描数据库：

DAU (日活跃用户):
  ├── 用户登录时: PFADD qh:stats:dau:{date} userID
  └── 查询时: PFCOUNT qh:stats:dau:{date} → 返回近似去重计数

MAU (月活跃用户):
  ├── 用户登录时: PFADD qh:stats:mau:{month} userID
  └── 查询时: PFCOUNT qh:stats:mau:{month}

在线人数:
  ├── WebSocket 连接时: INCR  qh:stats:online
  ├── WebSocket 断开时: DECR  qh:stats:online
  └── 查询时: GET qh:stats:online

今日考试次数:
  ├── 考试提交时: INCR  qh:stats:exam_count:{date}
  └── 查询时: GET qh:stats:exam_count:{date}
```

**历史趋势查询**：

### 7.2 用户统计

**注册趋势**：

```go
// 按日/周/月聚合注册用户数
func GetUserRegistrationTrend(granularity string, startDate, endDate time.Time) ([]dto.DailyStatItem, error) {
    dateFormat := getDateTruncFormat(granularity) // DAY/week/month

    var results []dto.DailyStatItem
    err := database.DB.Model(&model.User{}).
        Select(fmt.Sprintf("DATE_FORMAT(created_at, '%s') as date, COUNT(*) as count", dateFormat)).
        Where("created_at BETWEEN ? AND ?", startDate, endDate).
        Group("date").
        Order("date ASC").
        Scan(&results).Error
    return results, err
}
```

**用户留存率**：

```
留存率计算逻辑 (定时任务每日凌晨执行):

1. 查询 register_date = D 的注册用户数 → register_count
2. 查询这些用户在 D+N 天是否有登录记录:
   SELECT COUNT(DISTINCT user_id)
   FROM login_logs
   WHERE user_id IN (SELECT id FROM users WHERE DATE(created_at) = D)
   AND DATE(login_time) = D + N

3. retention_rate = retention_count / register_count
4. 写入 user_retention 表

留存热力图数据结构:
  rows: register_date (注册日期)
  cols: retention_day (1/3/7/14/30)
  cells: retention_rate (百分比)
```

**题目难度与区分度分析**：

```
题目难度系数 (P 值):
  P = 答对该题的人数 / 参与作答的总人数
  范围: [0, 1]
  P > 0.7 → 容易题
  0.3 <= P <= 0.7 → 中等题
  P < 0.3 → 难题

题目区分度 (D 值):
  1. 按总分排序，取前 27% 为高分组，后 27% 为低分组
  2. D = (高分组答对率 - 低分组答对率)
  3. D > 0.4 → 区分度优秀
  4. 0.3 <= D <= 0.4 → 区分度良好
  5. 0.2 <= D < 0.3 → 区分度一般
  6. D < 0.2 → 区分度差

实现 (Repository 层):
  1. 统计每道题的总作答人数和答对人数
  2. 计算 P 值
  3. 按用户总分排序，取高低分组
  4. 计算 D 值
  5. 结果缓存到 Redis (TTL 3600s)
```

```go
    // 分数段分布 (10分一段)
    var segments []dto.ScoreSegment
    database.DB.Raw(`
        SELECT
            FLOOR(score / 10) * 10 as score_range,
            COUNT(*) as count,
            ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM exam_records WHERE exam_id = ? AND status = 2), 2) as percentage
        FROM exam_records
        WHERE exam_id = ? AND status = 2
        GROUP BY score_range
        ORDER BY score_range
    `, examID, examID).Scan(&segments)
    result.Segments = segments

    // 及格率
    var passCount int64
    database.DB.Model(&model.ExamRecord{}).
        Where("exam_id = ? AND status = 2 AND score >= 60", examID).
        Count(&passCount)
    result.PassRate = float64(passCount) / float64(result.TotalCount) * 100

    return &result, nil
}
```

**成绩预测**：

```
基于线性回归的成绩预测:

1. 获取用户最近 N 次考试成绩序列: [(t1, s1), (t2, s2), ..., (tn, sn)]
2. 使用最小二乘法拟合线性模型: score = a * time + b
3. 预测下一次考试成绩: predicted_score = a * (n+1) + b
4. 计算置信区间 (基于标准差)

实现:
  func PredictScore(userID uint) (*dto.ScorePrediction, error) {
      records, _ := examRecordRepo.FindRecentByUserID(userID, 10)
      if len(records) < 3 {
          return nil, errors.New("数据不足，至少需要3次考试记录")
      }

      // 最小二乘法
      var sumX, sumY, sumXY, sumX2 float64
      for i, r := range records {
          x := float64(i + 1)
          y := r.Score
          sumX += x
          sumY += y
          sumXY += x * y
          sumX2 += x * x
      }
      n := float64(len(records))
      a := (n*sumXY - sumX*sumY) / (n*sumX2 - sumX*sumX)
      b := (sumY - a*sumX) / n

      predicted := a*(n+1) + b
      // 限制在 0-100 范围
      predicted = math.Max(0, math.Min(100, predicted))

      return &dto.ScorePrediction{PredictedScore: predicted}, nil
  }
```


```
4. 避免 N+1 查询:
   ├── 使用 Preload 预加载关联数据
   └── 使用 Raw SQL 进行复杂聚合
```

### 10.3 导出优化

```
异步导出流程:

1. 前端提交导出请求 → 后端创建 export_record (status=1)
2. 发送消息到 pkg/mq 队列
3. Job Worker 异步处理:
   ├── 流式查询数据 (FindInBatches, batch_size=1000)
   ├── 使用 excelize 流式写入 Excel
   ├── 保存文件到本地/MinIO
   └── 更新 export_record (status=2, file_path)
4. 前端轮询导出状态 → 完成后下载

防重复导出:
  ├── 同一用户同一类型导出任务进行中时，返回已有任务
  └── 使用 Redis 分布式锁: SETNX qh:stats:export:{userId}:{type}
```
