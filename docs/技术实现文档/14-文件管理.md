## 1. 模块概述

文件管理模块是题小助平台的基础设施层，为题库管理、考试管理、班级管理、用户管理等业务模块提供统一的文件上传、存储、访问和生命周期管理能力。

### 1.1 核心能力

| 能力 | 说明 |
|------|------|
| 多类型文件上传 | 图片(5MB)、文档(20MB)、音频(50MB)、视频(200MB)、压缩包(50MB) |
| 多存储后端 | 本地文件系统、MinIO、阿里云 OSS，通过配置切换 |
| 引用计数 | 文件被业务数据引用时计数递增，解除引用时递减，防止误删 |
| 孤立文件清理 | 定时任务清理 ref_count=0 且超过 7 天的文件 |
| 缩略图生成 | 100x100、300x300、800x800 三种预设尺寸，7 天缓存 |
| 签名 URL | 私有文件 1 小时 HMAC 签名访问，考试场景自动续期 |
| 文件去重 | MD5 校验，相同文件只存储一份物理副本 |
| 安全校验 | 扩展名白名单 + MIME 类型 + Magic Number + 双扩展名防护 |
| 病毒扫描 | ClamAV 集成，扫描失败拒绝上传 |
| 水印功能 | 用户 ID + 时间戳不可见数字水印 |

### 1.2 业务场景

文件管理服务于以下业务场景：

| 场景 | 文件类型 | 目录前缀 | 关联模块 |
|------|----------|----------|----------|
| 题目附件 | 图片、音频、视频 | `question/` | 题库管理 |
| 试卷导出 | Word、PDF | `exam/` | 考试管理 |
| 班级封面 | 图片 | `class/` | 班级管理 |
| 用户头像 | 图片 | `avatar/` | 用户管理 |
| Excel 导入/导出 | Excel | `temp/` | 题库管理 |
| 错题导出 | Word、PDF | `export/` | 错题本 |

## 7. 核心实现

### 7.1 文件上传流程

文件上传是整个模块的核心流程，经过多层校验和处理后才持久化存储。

```
客户端 Multipart 上传
        │
        ▼
  ┌─ Controller 层 ─┐
  │ 解析 FormFile    │
  │ 提取 user_id     │
  └────────┬─────────┘
           │
           ▼
  ┌─ Service: UploadFile ─────────────────────────────────────┐
  │                                                           │
  │  Step 1: 文件名校验                                       │
  │    - 长度 <= 255 字符                                     │
  │    - 禁止特殊字符: < > : " / \ | ? *                      │
  │    - 禁止双扩展名: report.jpg.exe                         │
  │                                                           │
  │  Step 2: 扩展名白名单校验                                 │
  │    - 图片: jpg/jpeg/png/gif/webp                         │
  │    - 文档: pdf/doc/docx/xls/xlsx/ppt/pptx               │
  │    - 音频: mp3/wav                                       │
  │    - 视频: mp4                                           │
  │    - 压缩包: zip/rar                                     │
  │                                                           │
  │  Step 3: 文件大小限制                                     │
  │    - 图片 <= 5MB, 文档 <= 20MB, 音频 <= 50MB             │
  │    - 视频 <= 200MB, 压缩包 <= 50MB                       │
  │                                                           │
  │  Step 4: Magic Number 校验                               │
  │    - 读取文件头 512 字节                                  │
  │    - 比对已知格式签名 (PNG: 89504E47, JPEG: FFD8FF...)   │
  │    - 与声明的 MIME 类型交叉验证                           │
  │                                                           │
  │  Step 5: 病毒扫描 (ClamAV)                               │
  │    - 将文件流发送到 clamd daemon                         │
  │    - 扫描结果为 INFECTED → 拒绝上传                      │
  │    - 扫描超时或连接失败 → 拒绝上传                       │
  │                                                           │
  │  Step 6: MD5 去重检查                                     │
  │    - 流式计算 MD5 (io.MultiWriter 写入磁盘 + hash)       │
  │    - 查询 fileRepo.FindFileByMD5(md5)                    │
  │    - 已存在 → 删除临时文件，返回已有记录                  │
  │                                                           │
  │  Step 7: UUID 重命名 + 存储                              │
  │    - newFileName = uuid.New() + ext                      │
  │    - 按业务类型确定目录: uploads/{business_type}/         │
  │    - 调用 StorageProvider.Upload() 持久化                 │
  │                                                           │
  │  Step 8: 写入数据库                                       │
  │    - 创建 File 记录 (model.File)                         │
  │    - 初始状态: virus_scan_status="pending"               │
  │    - reference_count = 0                                 │
  │                                                           │
  │  Step 9: 返回结果                                         │
  │    - 返回文件 ID、URL、大小等信息                         │
  │                                                           │
  └───────────────────────────────────────────────────────────┘
```

**关键实现细节**：

```go
// 伪代码：核心上传流程
func UploadFile(uploaderID uint, fileName string, fileSize int64,
    fileType string, reader io.Reader) (*model.File, error) {

    // 1. 校验文件名
    if err := validateFileName(fileName); err != nil {
        return nil, err
    }

    // 2. 扩展名白名单
    ext := strings.ToLower(filepath.Ext(fileName))
    if !allowedExtensions[ext] {
        return nil, fmt.Errorf("不支持的文件类型: %s", ext)
    }

    // 3. 大小限制
    limit := getFileSizeLimit(ext)
    if fileSize > limit {
        return nil, fmt.Errorf("文件大小超过限制")
    }

    // 4. Magic Number 校验
    buf := make([]byte, 512)
    n, _ := io.ReadFull(reader, buf)
    if !validateMagicNumber(buf[:n], ext) {
        return nil, fmt.Errorf("文件内容与扩展名不匹配")
    }
    reader = io.MultiReader(bytes.NewReader(buf[:n]), reader)

    // 5. 病毒扫描
    if err := scanVirus(reader); err != nil {
        return nil, fmt.Errorf("文件安全检查未通过")
    }

    // 6. MD5 去重 + 存储
    newFileName := uuid.New().String() + ext
    hash := md5.New()
    teeReader := io.TeeReader(reader, hash)
    storagePath := fmt.Sprintf("%s/%s", businessType, newFileName)

    url, err := storageProvider.Upload(ctx, teeReader, storagePath)
    if err != nil {
        return nil, err
    }
    md5Hash := hex.EncodeToString(hash.Sum(nil))

    // 7. 去重检查
    if existing, _ := fileRepo.FindFileByMD5(md5Hash); existing != nil {
        storageProvider.Delete(ctx, storagePath)
        return existing, nil
    }

    // 8. 写入数据库
    file := &model.File{
        Name: newFileName, Original: fileName,
        Path: storagePath, URL: url,
        Size: fileSize, Type: fileType, Extension: ext,
        MD5: md5Hash, StorageType: currentStorageType,
        UploaderID: uploaderID, Status: "active",
        VirusScanStatus: "clean",
    }
    return file, fileRepo.CreateFile(file)
}
```

**图片上传额外流程** (`UploadImage`)：
- 图片解码验证 (`image.Decode`)
- 超过 1920px 宽度时等比缩放
- 自动生成三种尺寸缩略图
- 异步触发水印处理（通过 MQ 消息队列）

**批量上传** (`BatchUpload`)：
- 限制最多 20 个文件
- 逐个调用 `UploadFile`
- 收集成功/失败结果，不因单个失败中断全部

### 7.2 存储抽象层

存储抽象层通过接口统一不同存储后端的操作，支持配置驱动切换。

**接口定义**：

```go
// pkg/upload/provider.go
type StorageProvider interface {
    // Upload 上传文件，返回访问 URL
    Upload(ctx context.Context, reader io.Reader, key string) (string, error)
    // Delete 删除文件
    Delete(ctx context.Context, key string) error
    // GetURL 获取文件访问 URL
    GetURL(ctx context.Context, key string) (string, error)
    // GetSignedURL 获取签名 URL（私有文件访问）
    GetSignedURL(ctx context.Context, key string, expiry time.Duration) (string, error)
    // GetReader 获取文件读取器
    GetReader(ctx context.Context, key string) (io.ReadCloser, error)
    // Exists 检查文件是否存在
    Exists(ctx context.Context, key string) (bool, error)
}
```

**三种实现**：

| 实现 | 文件 | 说明 |
|------|------|------|
| `LocalStorage` | `pkg/upload/local.go` | 本地文件系统，开发环境默认使用 |
| `MinIOStorage` | `pkg/upload/minio.go` | MinIO 对象存储，兼容 S3 协议 |

### 7.3 引用计数

引用计数机制确保文件不会在仍有业务引用时被误删。

**计数规则**：
- **递增**：业务模块关联文件时（如题目添加附件），调用 `AddReference` 创建 `file_references` 记录，同时 `files.reference_count + 1`
- **递减**：业务模块解除关联时（如题目删除附件），调用 `DeleteReference` 删除引用记录，同时 `files.reference_count - 1`（下限为 0）
- **软删除不减计数**：题目软删除时，附件引用计数不变，附件保留
- **物理删除减计数**：题目物理删除时，附件引用计数减 1

**Repository 层实现**：

```go
// 原子递增（防并发）
func IncrementReferenceCount(fileID uint) error {
    return database.DB.Model(&model.File{}).
        Where("id = ?", fileID).
        UpdateColumn("reference_count",
            database.DB.Raw("reference_count + 1")).Error
}

// 原子递减（下限为 0）
func DecrementReferenceCount(fileID uint) error {
    return database.DB.Model(&model.File{}).
        Where("id = ? AND reference_count > 0", fileID).
        UpdateColumn("reference_count",
            database.DB.Raw("reference_count - 1")).Error
}
```

**引用关系查询**：`ListFileReferences(fileID)` 返回某文件被哪些业务数据引用，用于管理端展示引用详情和删除前确认提示。

**删除保护**：`DeleteFile` 方法在删除前检查 `reference_count`，大于 0 时返回错误提示 `"文件正在被 N 处引用，无法删除"`。

### 7.4 孤立文件清理

孤立文件指 `reference_count = 0` 且超过 7 天未被引用的文件。

**两种触发方式**：

1. **定时任务** (`task/file_cleanup.go`)：每日凌晨 2:00 自动执行
2. **手动触发**：管理员通过 `/api/v1/admin/file/cleanup/orphan` 接口执行

**清理流程**：

```
Step 1: 查询孤立文件
  SELECT * FROM files
  WHERE reference_count = 0
    AND status = 'active'
    AND created_at < NOW() - INTERVAL 7 DAY

Step 2: 逐个二次确认引用计数
  SELECT COUNT(*) FROM file_references WHERE file_id = ?

Step 3: 删除物理文件
  - 删除原始文件: storageProvider.Delete(ctx, file.Path)
  - 删除缩略图文件

Step 4: 更新数据库状态
  UPDATE files SET status = 'deleted' WHERE id = ?

Step 5: 记录清理日志
  INSERT INTO file_cleanup_logs (cleaned_count, freed_space, file_ids, status, ...)
```

**临时文件清理**：`path LIKE '%/temp/%'` 且创建时间超过 24 小时的文件直接物理删除。

**软删除文件清理**：`status = 'deleted'` 且 `deleted_at` 超过 30 天的记录彻底清除。

**清理日志结构**（`file_cleanup_logs`）：记录每次清理的批次号、清理数量、释放空间、文件 ID 列表、执行状态和耗时。

### 7.5 缩略图生成

缩略图仅支持图片文件（jpg/jpeg/png/gif/webp）。

**预设尺寸**：

| 尺寸标识 | 宽度 | 高度 | 用途 |
|----------|------|------|------|
| small | 100px | 100px | 列表缩略图 |
| medium | 300px | 300px | 卡片展示 |
| large | 800px | 800px | 预览大图 |

**生成流程**：

1. 图片上传时，异步发布 `image_process` 消息到 MQ
2. `ImageProcessJob` 消费消息，使用 Go 标准库 `image` + `golang.org/x/image` 生成三种尺寸
3. 缩略图存储在 `uploads/thumbnails/` 目录，命名为 `{uuid}_thumb_{size}.{ext}`
4. 缩略图 ID 记录在 `files.thumbnail_ids` 字段（JSON 数组）
5. 缩略图文件设置 7 天 HTTP 缓存头：`Cache-Control: public, max-age=604800`

**获取缩略图**：

```
GET /api/v1/file/:id/thumbnail/:size
  → Service 层校验文件类型是否为图片
  → 根据 size 参数查找对应缩略图
  → 缩略图不存在时回退返回原图
  → 设置 Content-Type 和 Cache-Control 头
  → 返回文件流
```

**宽度限制**：上传图片宽度超过 1920px 时，等比缩放到 1920px 后再存储原始文件，减少存储消耗。

### 7.6 签名 URL

私有文件通过 HMAC 签名 URL 访问，防止未授权访问。

**签名机制**：

```
签名 URL 格式:
  /api/v1/file/{id}/download?expires={timestamp}&sign={hmac_sha256}

签名计算:
  stringToSign = "{file_id}:{user_id}:{expires}"
  sign = HMAC-SHA256(stringToSign, server_secret)
```

**签名参数**：

| 参数 | 说明 |
|------|------|
| file_id | 文件 ID，绑定特定文件 |
| user_id | 请求用户 ID，绑定特定用户（不可转发） |
| expires | Unix 时间戳，签名过期时间 |
| sign | HMAC-SHA256 签名值 |

**有效期规则**：
- 默认有效期：1 小时（3600 秒）
- 考试场景：前端在签名 URL 过期前 5 分钟自动请求续期
- 续期方式：前端定时器检测，调用 `/api/v1/file/:id/signed-url` 获取新 URL

**验证流程**：

```
收到请求 → 提取 expires 和 sign 参数
  → 检查 expires 是否已过期
  → 重新计算 HMAC 并比对 sign
  → 验证 file_id 与请求路径匹配
  → 验证 user_id 与当前 JWT 用户匹配
  → 任一验证失败返回 403
```

### 7.7 文件去重

基于 MD5 哈希的文件去重，避免相同文件重复存储。

**实现策略**：

1. 上传时通过 `io.MultiWriter` 同时写入磁盘和计算 MD5
2. 写入完成后查询 `fileRepo.FindFileByMD5(md5Hash)`
3. 若已存在相同 MD5 的文件：
   - 删除刚写入的物理文件
   - 返回已有文件的记录（共享存储）
   - 日志记录去重事件
4. 若不存在，正常创建新记录

**注意事项**：
- MD5 冲突概率极低（2^128），在题库场景下可忽略
- 去重后多个 `files` 记录可指向同一物理文件，删除时需确认无其他记录引用
- 大文件 MD5 计算使用流式处理，避免一次性加载到内存

### 7.8 安全校验

多层安全校验确保上传文件的安全性。

**校验层次**：

```
Layer 1: 文件名校验
  - 长度 <= 255 字符
  - 禁止特殊字符: < > : " / \ | ? *
  - 禁止双扩展名检测 (正则: \.\w+\.\w+$)
  - 禁止路径遍历字符: .. 、 ./ 、 .\

Layer 2: 扩展名白名单
  - 仅允许预定义的扩展名集合
  - 扩展名统一转小写比较

Layer 3: MIME 类型校验
  - 检查 Content-Type header
  - 与扩展名交叉验证

Layer 4: Magic Number 校验
  - 读取文件头 512 字节
  - 比对已知格式的文件头签名:
    PNG:  89 50 4E 47
    JPEG: FF D8 FF
    GIF:  47 49 46 38
    PDF:  25 50 44 46
    ZIP:  50 4B 03 04
    RAR:  52 61 72 21
    MP4:  00 00 00 xx 66 74 79 70
  - Magic Number 与扩展名不匹配 → 拒绝

Layer 5: 图片解码验证
  - 仅限图片文件
  - 调用 image.Decode 尝试完整解码
  - 解码失败 → 拒绝（防止畸形图片攻击）
```

**双扩展名防护**：

```go
func hasDoubleExtension(filename string) bool {
    parts := strings.Split(filename, ".")
    if len(parts) < 3 {
        return false
    }
    // 检查倒数第二个扩展名是否也在危险列表中
    dangerous := map[string]bool{
        ".exe": true, ".bat": true, ".cmd": true,
        ".sh": true, ".ps1": true, ".vbs": true,
        ".js": true, ".php": true, ".asp": true,
    }
    secondExt := strings.ToLower("." + parts[len(parts)-2])
    return dangerous[secondExt]
}
```

### 7.10 水印功能

水印功能为图片添加不可见数字水印，用于版权追溯。

**水印类型**：

| 类型 | 说明 |
|------|------|
| text | 文字水印，内容包含用户 ID + 时间戳 |
| image | 图片水印，使用配置的水印图片 |

**不可见数字水印实现**：

```
水印内容: "UID:{user_id}|TS:{timestamp}|SIG:{hmac}"

嵌入方式 (LSB - Least Significant Bit):
  1. 将水印文本编码为 UTF-8 字节流
  2. 将字节流转换为二进制位序列
  3. 对图片像素的最低有效位进行替换
  4. R/G/B 三个通道各嵌入 1 bit，每个像素嵌入 3 bit
  5. 嵌入位置使用密钥派生的伪随机序列打乱（增强鲁棒性）

提取方式:
  1. 读取像素 LSB
  2. 按相同伪随机序列重组
  3. 解码为水印文本
```

**可见水印叠加**（管理员可选）：

- 位置：top-left / top-right / bottom-left / bottom-right / center
- 透明度：0.0 ~ 1.0，推荐 0.3 ~ 0.5
- 字体大小：默认 24px
- 字体颜色：默认 #FFFFFF

**处理时机**：
- 图片上传时异步处理（通过 `image_process` MQ 消息）
- 处理完成后更新 `files.watermark_applied = true`
- 原始文件保留，水印版本单独存储

**水印预览**：管理员可通过 `/api/v1/file/watermark/preview` 预览水印效果，返回带水印的示例图片。

## 12. 错误处理

### 12.1 错误码定义

| 错误码 | HTTP 状态 | 说明 |
|--------|-----------|------|
| F0001 | 400 | 文件类型不支持 |
| F0002 | 400 | 文件大小超过限制 |
| F0003 | 400 | 文件名不合法 |
| F0004 | 400 | 双扩展名检测拦截 |
| F0005 | 400 | Magic Number 校验失败 |
| F0006 | 400 | 病毒扫描未通过 |
| F0007 | 404 | 文件不存在 |
| F0008 | 403 | 无权操作此文件 |
| F0009 | 409 | 文件正在被引用，无法删除 |
| F0010 | 429 | 每日上传次数已达上限 |
| F0011 | 500 | 存储服务异常 |
| F0012 | 500 | 病毒扫描服务不可用 |

### 12.2 错误处理策略

**上传失败**：
- 校验失败：返回具体错误信息，不创建任何记录
- 存储失败：回滚已写入的部分数据，返回通用错误
- 数据库失败：删除已存储的物理文件，返回通用错误

**删除失败**：
- 物理文件删除失败：记录日志，不影响数据库状态更新
- 数据库更新失败：记录日志，返回错误

**清理任务失败**：
- 单个文件清理失败：跳过，继续处理下一个，记录错误日志
- 整体任务异常：更新清理日志状态为 `failed`，记录错误信息
