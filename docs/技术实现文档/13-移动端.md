## 1. 模块概述

移动端是题小助 V2.0 面向终端用户（学生、教师、创作者）的核心客户端，基于 UniApp 框架实现一套代码多端运行（H5、微信小程序、iOS/Android App）。模块覆盖以下核心功能域：

| 功能域 | 说明 | 对应后端模块 |
|--------|------|-------------|
| 首页 | 数据概览、快捷入口、推荐内容 | 数据统计、推荐系统 |
| 题库 | 分类浏览、题目列表、搜索、收藏 | 题库管理、分类管理 |
| 考试 | 考试列表、在线答题、成绩查看 | 考试管理、在线答题 |
| 练习 | 刷题练习、练习统计 | 刷题练习系统 |
| 班级 | 班级管理、作业、公告、考试 | 班级管理系统 |
| 个人中心 | 个人信息、角色申请、学习统计、错题本、收藏 | 用户管理、错题本系统 |
| 通知 | WebSocket 实时通知、消息中心 | 通知系统 |

移动端采用 5 Tab 导航结构，42 个页面分布在 13 个功能目录下，通过 Pinia 进行状态管理，通过封装的 `uni.request` 与后端 API 交互。

### 3.4 入口与初始化

应用入口为 `main.ts`，通过 `createSSRApp` 创建 Vue 应用并注册 Pinia：

```typescript
// main.ts
import { createSSRApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

export function createApp() {
  const app = createSSRApp(App)
  const pinia = createPinia()
  app.use(pinia)
  return { app, pinia }
}
```

`App.vue` 通过 `onLaunch` / `onShow` / `onHide` 生命周期钩子处理应用级事件。全局样式在 `App.vue` 的 `<style>` 块中定义，包括基础页面背景、通用按钮和卡片样式。

### 5.2 Token 刷新机制

**核心文件**：`api/request.ts`、`utils/auth.ts`、`api/auth.ts`

当前 `request.ts` 实现了基础的请求封装，包含 Token 注入和 401 检测。V2.0 需要升级为完整的 Refresh Token 静默刷新机制。

**现有实现**：

```typescript
// api/request.ts - 当前实现
const BASE_URL = import.meta.env.VITE_API_BASE_URL || '/api/v1'

export const request = <T = any>(options: RequestOptions): Promise<ApiResponse<T>> => {
  return new Promise((resolve, reject) => {
    const token = uni.getStorageSync('token')
    uni.request({
      url: BASE_URL + options.url,
      header: {
        'Content-Type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : '',
        ...options.header
      },
      success: (res) => {
        const data = res.data as ApiResponse<T>
        if (data.code === '00000') {
          resolve(data)
        } else if (data.code === 'A0003' || data.code === 'A0004') {
          // token 过期或无效，跳转登录
          uni.removeStorageSync('token')
          uni.navigateTo({ url: '/pages/login/index' })
          reject(new Error('登录已过期'))
        }
      }
    })
  })
}
```

**V2.0 升级方案**：

```typescript
// api/request.ts - V2.0 完整实现
import { getToken, setToken, getRefreshToken, setRefreshToken, clearAuth } from '@/utils/auth'
import { refreshToken as refreshTokenApi } from '@/api/auth'

const BASE_URL = import.meta.env.VITE_API_BASE_URL || '/api/v1'

// Token 刷新状态
let isRefreshing = false
let refreshSubscribers: ((token: string) => void)[] = []

// 将等待刷新的请求加入队列
function subscribeTokenRefresh(cb: (token: string) => void) {
  refreshSubscribers.push(cb)
}

// 刷新完成后执行队列中的请求
function onTokenRefreshed(newToken: string) {
  refreshSubscribers.forEach(cb => cb(newToken))
  refreshSubscribers = []
}

export const request = <T = any>(options: RequestOptions): Promise<ApiResponse<T>> => {
  return new Promise((resolve, reject) => {
    const token = getToken()

    uni.request({
      url: BASE_URL + options.url,
      method: options.method || 'GET',
      data: options.data,
      header: {
        'Content-Type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : '',
        ...options.header
      },
      success: async (res: any) => {
        const data = res.data as ApiResponse<T>

        if (data.code === '00000') {
          resolve(data)
        } else if (data.code === 'A0003') {
          // Access Token 过期，尝试刷新
          const refreshToken = getRefreshToken()
          if (!refreshToken) {
            clearAuth()
            uni.reLaunch({ url: '/pages/login/index' })
            reject(new Error('登录已过期'))
            return
          }

          if (!isRefreshing) {
            isRefreshing = true
            try {
              const refreshRes = await refreshTokenApi()
              const newToken = refreshRes.data.accessToken
              const newRefreshToken = refreshRes.data.refreshToken
              setToken(newToken)
              setRefreshToken(newRefreshToken)
              onTokenRefreshed(newToken)

              // 重试当前请求
              const retryRes = await retryRequest(options, newToken)
              resolve(retryRes)
            } catch (e) {
              clearAuth()
              uni.reLaunch({ url: '/pages/login/index' })
              reject(new Error('刷新 Token 失败'))
            } finally {
              isRefreshing = false
              refreshSubscribers = []
            }
          } else {
            // 正在刷新，将请求加入队列
            subscribeTokenRefresh(async (newToken: string) => {
              const retryRes = await retryRequest(options, newToken)
              resolve(retryRes)
            })
          }
        } else if (data.code === 'A0004') {
          // Refresh Token 也无效
          clearAuth()
          uni.reLaunch({ url: '/pages/login/index' })
          reject(new Error('登录已过期'))
        } else {
          uni.showToast({ title: data.msg || '请求失败', icon: 'none' })
          reject(new Error(data.msg))
        }
      },
      fail: (err: any) => {
        uni.showToast({ title: '网络错误', icon: 'none' })
        reject(err)
      }
    })
  })
}

// 重试请求
function retryRequest(options: RequestOptions, newToken: string): Promise<ApiResponse> {
  return new Promise((resolve, reject) => {
    uni.request({
      url: BASE_URL + options.url,
      method: options.method || 'GET',
      data: options.data,
      header: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${newToken}`,
        ...options.header
      },
      success: (res: any) => {
        const data = res.data
        if (data.code === '00000') {
          resolve(data)
        } else {
          reject(new Error(data.msg))
        }
      },
      fail: reject
    })
  })
}
```

### 5.3 WebSocket 实时通知

**连接配置**：

| 参数 | 值 |
|------|-----|
| 连接地址 | `ws://{host}/ws?token={accessToken}` |
| 心跳间隔 | 30 秒 |
| 心跳消息 | `{ "type": "ping" }` |
| 重连策略 | 指数退避：1s → 2s → 4s → 8s → 16s → 30s（最大） |
| 通知类型 | 考试提醒、成绩发布、作业通知、系统公告 |

**实现方案**：

```typescript
// utils/websocket.ts
import { getToken } from '@/utils/auth'

class WebSocketClient {
  private socketTask: UniApp.SocketTask | null = null
  private heartbeatTimer: ReturnType<typeof setInterval> | null = null
  private reconnectTimer: ReturnType<typeof setTimeout> | null = null
  private reconnectAttempts = 0
  private maxReconnectDelay = 30000
  private baseReconnectDelay = 1000
  private isManualClose = false
  private messageHandlers: Map<string, (data: any) => void> = new Map()

  connect() {
    const token = getToken()
    if (!token) return

    const wsUrl = `${this.getWsBaseUrl()}/ws?token=${token}`

    this.socketTask = uni.connectSocket({
      url: wsUrl,
      success: () => console.log('[WS] 连接中...')
    })

    this.socketTask.onOpen(() => {
      console.log('[WS] 连接成功')
      this.reconnectAttempts = 0
      this.startHeartbeat()
    })

    this.socketTask.onMessage((res) => {
      this.handleMessage(res.data)
    })

    this.socketTask.onClose(() => {
      console.log('[WS] 连接关闭')
      this.stopHeartbeat()
      if (!this.isManualClose) {
        this.scheduleReconnect()
      }
    })

    this.socketTask.onError((err) => {
      console.error('[WS] 连接错误', err)
    })
  }

  private getWsBaseUrl(): string {
    const apiUrl = import.meta.env.VITE_API_BASE_URL || '/api/v1'
    const wsProtocol = apiUrl.startsWith('https') ? 'wss' : 'ws'
    const baseUrl = apiUrl.replace(/^https?:\/\//, '').replace(/\/api\/v1$/, '')
    return `${wsProtocol}://${baseUrl}`
  }

  private startHeartbeat() {
    this.heartbeatTimer = setInterval(() => {
      this.send({ type: 'ping' })
    }, 30000)
  }

  private stopHeartbeat() {
    if (this.heartbeatTimer) {
      clearInterval(this.heartbeatTimer)
      this.heartbeatTimer = null
    }
  }

  private scheduleReconnect() {
    const delay = Math.min(
      this.baseReconnectDelay * Math.pow(2, this.reconnectAttempts),
      this.maxReconnectDelay
    )
    this.reconnectAttempts++

    console.log(`[WS] ${delay}ms 后重连 (第 ${this.reconnectAttempts} 次)`)

    this.reconnectTimer = setTimeout(() => {
      this.connect()
    }, delay)
  }

  private handleMessage(rawData: string | ArrayBuffer) {
    try {
      const data = JSON.parse(rawData as string)
      if (data.type === 'pong') return

      // 分发消息到注册的处理器
      const handler = this.messageHandlers.get(data.type)
      if (handler) {
        handler(data)
      }

      // 通用消息处理
      this.onNotification(data)
    } catch (e) {
      console.error('[WS] 消息解析失败', e)
    }
  }

  private onNotification(data: any) {
    // 更新未读计数
    uni.$emit('notification', data)

    // 显示 Toast 提示
    if (data.title) {
      uni.showToast({
        title: data.title,
        icon: 'none',
        duration: 3000
      })
    }
  }

  send(data: any) {
    this.socketTask?.send({ data: JSON.stringify(data) })
  }

  on(type: string, handler: (data: any) => void) {
    this.messageHandlers.set(type, handler)
  }

  off(type: string) {
    this.messageHandlers.delete(type)
  }

  close() {
    this.isManualClose = true
    this.stopHeartbeat()
    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer)
    }
    this.socketTask?.close({})
  }
}

export const wsClient = new WebSocketClient()
```

### 5.5 考试时间控制

**核心文件**：`hooks/useExam.ts`、`pages/exam/answer.vue`、`components/Timer/index.vue`

考试时间控制采用服务端权威时间方案，客户端仅做展示，实际超时由后端 Redis 延迟队列控制。

**useExam Hook**：

```typescript
// hooks/useExam.ts
import { ref, computed, onBeforeUnmount } from 'vue'

export const useExam = () => {
  const duration = ref(0)
  const remaining = ref(0)
  const isRunning = ref(false)
  let timer: ReturnType<typeof setInterval> | null = null
  let syncTimer: ReturnType<typeof setInterval> | null = null
  let serverTimestamp = 0    // 服务端时间戳
  let localTimestamp = 0     // 本地时间戳（校准点）

  const formattedTime = computed(() => {
    const hours = Math.floor(remaining.value / 3600)
    const minutes = Math.floor((remaining.value % 3600) / 60)
    const seconds = remaining.value % 60
    if (hours > 0) {
      return `${pad(hours)}:${pad(minutes)}:${pad(seconds)}`
    }
    return `${pad(minutes)}:${pad(seconds)}`
  })

  const isUrgent = computed(() => remaining.value < 300 && remaining.value > 0)

  function pad(n: number): string {
    return n.toString().padStart(2, '0')
  }

  // 使用服务端时间戳初始化
  const start = (seconds: number, serverTs?: number) => {
    duration.value = seconds
    remaining.value = seconds
    isRunning.value = true

    if (serverTs) {
      serverTimestamp = serverTs
      localTimestamp = Date.now()
    }

    timer = setInterval(() => {
      if (remaining.value > 0) {
        remaining.value--
      } else {
        stop()
      }
    }, 1000)

    // 每 60 秒与服务端同步时间
    syncTimer = setInterval(() => {
      syncWithServer()
    }, 60000)
  }

  // 与服务端同步时间
  const syncWithServer = async () => {
    try {
      const res = await getServerTime() // 调用后端获取服务端时间戳
      const serverNow = res.data.timestamp
      const localNow = Date.now()
      const drift = Math.abs((localNow - localTimestamp) - (serverNow - serverTimestamp))

      // 误差超过 30 秒时校准
      if (drift > 30) {
        const elapsed = Math.floor((serverNow - serverTimestamp) / 1000)
        remaining.value = Math.max(0, duration.value - elapsed)
        serverTimestamp = serverNow
        localTimestamp = localNow
        console.log(`[Timer] 时间校准，误差 ${drift} 秒`)
      }
    } catch (e) {
      console.error('[Timer] 时间同步失败', e)
    }
  }

  const stop = () => {
    if (timer) { clearInterval(timer); timer = null }
    if (syncTimer) { clearInterval(syncTimer); syncTimer = null }
    isRunning.value = false
  }

  const getElapsedTime = () => duration.value - remaining.value
  const getRemainingTime = () => remaining.value

  onBeforeUnmount(() => { stop() })

  return {
    duration, remaining, isRunning,
    formattedTime, isUrgent,
    start, stop, getElapsedTime, getRemainingTime
  }
}
```

### 6.2 请求封装规范

`request.ts` 封装了 `uni.request`，提供以下能力：

| 能力 | 说明 |
|------|------|
| 基础 URL | 通过 `VITE_API_BASE_URL` 环境变量配置，H5 模式使用相对路径走 Vite 代理 |

| 错误处理 | 网络错误显示 Toast，业务错误通过 reject 传播 |

## 8. 平台差异处理

### 8.1 条件编译语法

UniApp 使用预处理指令实现平台差异代码：

```typescript
// 仅在 H5 平台编译
// #ifdef H5
console.log('Running on H5')
// #endif

// 仅在微信小程序编译
// #ifdef MP-WEIXIN
wx.login({ ... })
// #endif

// 仅在 App 平台编译
// #ifdef APP-PLUS
plus.barcode.scan({ ... })
// #endif

// 非 H5 平台编译
// #ifndef H5
console.log('Not running on H5')
// #endif
```

### 8.2 登录差异

| 平台 | 登录方式 | 实现 |
|------|---------|------|
| H5 | 账号密码登录 | `POST /auth/login` |
| 微信小程序 | 微信静默登录 | `wx.login()` → 获取 code → `POST /auth/wx-login` |
| App | 账号密码 + 第三方登录 | `POST /auth/login` + OAuth |

### 8.3 分享差异

| 平台 | 分享方式 | 实现 |
|------|---------|------|
| H5 | 复制链接 | `uni.setClipboardData` |
| 微信小程序 | 微信分享 | `onShareAppMessage` / `onShareTimeline` |
| App | 系统分享 | `uni.share` |

### 8.4 推送差异

| 平台 | 推送方式 | 实现 |
|------|---------|------|
| H5 | 不支持 | WebSocket 通知替代 |
| 微信小程序 | 订阅消息 | `wx.requestSubscribeMessage` |
| App | 推送通知 | `uni.push` + 厂商推送通道 |

### 8.5 文件上传差异

| 平台 | 选择方式 | 实现 |
|------|---------|------|
| H5 | `input[type=file]` | DOM 操作创建 input 元素 |
| 微信小程序 | `uni.chooseMessageFile` | 微信原生文件选择 |
| App | `uni.chooseFile` | uni-app 封装的文件选择 |

### 8.6 扫码差异

| 平台 | 扫码方式 | 实现 |
|------|---------|------|
| H5 | 不支持 | 提示用户使用小程序或 App |
| 微信小程序 | `uni.scanCode` | 微信原生扫码 |
| App | `uni.scanCode` | 使用 `plus.barcode` 原生扫码 |

### 9.4 接口缓存

频繁请求且数据变化不频繁的接口结果进行本地缓存：

```typescript
// 缓存分类树（变化频率低）
const CACHE_KEY = 'category_tree_cache'
const CACHE_TTL = 3600000 // 1 小时

export const getCategoryTree = async () => {
  const cached = uni.getStorageSync(CACHE_KEY)
  if (cached) {
    const { data, timestamp } = JSON.parse(cached)
    if (Date.now() - timestamp < CACHE_TTL) {
      return { code: '00000', msg: 'success', data }
    }
  }
  const res = await request({ url: '/categories/tree' })
  uni.setStorageSync(CACHE_KEY, JSON.stringify({
    data: res.data,
    timestamp: Date.now()
  }))
  return res
}
```
