## 一、模块概述

内容创作系统是题小助 V2.0 的核心扩展模块，为创作者提供从入驻、创作、审核到内容分发的完整闭环。涵盖以下核心功能：

- **创作者入驻**：用户申请成为创作者，管理员审核资质，签署创作者协议，积分等级体系驱动创作者成长
- **创作者等级与积分**：五级等级体系（初级/中级/高级/资深/专家），积分获取与消耗，每月自动评估升降级
- **创作者作品集**：创建作品集，添加题目/试卷作品，支持公开/私有可见性，浏览量与点赞统计
- **内容版本管理**：每次保存自动生成版本快照，支持版本列表查看、版本详情对比、回滚到任意历史版本，最多保留 50 个版本
- **内容标签**：标签 CRUD，内容打标签/移除标签，热门标签排行，标签使用次数统计
- **内容收藏**：收藏夹分组管理，收藏/取消收藏，按内容类型和收藏夹筛选
- **内容预览与搜索**：按内容类型预览，综合搜索（题目/试卷/文章），搜索建议、热门搜索、搜索历史
- **内容审核流程**：提交审核、多步审核流程、审核通过/驳回/撤回，审核通知与意见回复，管理员批量审核

本模块复用现有的分层架构（Controller → Service → Repository → Model），通过包级函数调用，依赖全局 `database.DB` 进行数据访问。

**请求处理流程**：

```
客户端请求
    │
    ▼
Router (路由匹配)
    │
    ▼
Middleware 链
    ├── CORS (跨域)
    ├── RateLimit (限流)
    ├── JWT Auth (从 Header 提取 token，验证签名，注入 user_id)
    ├── RBAC Permission (检查角色: creator / admin)
    ├── SensitiveFilter (POST/PUT 请求体敏感词检测)
    └── OperationLog (记录操作日志)
    │
    ▼
ContentController
    ├── 参数绑定 (ShouldBindJSON / ShouldBindQuery)
    ├── 参数校验 (binding tag)
    └── 调用 content service 包级函数
    │
    ▼
content service
    ├── 业务逻辑校验 (权限、状态、重复检测)
    ├── 调用 content repository 包级函数
    ├── 调用 user repository 获取用户信息
    ├── Model ↔ DTO 转换
    └── 日志记录 (logger.Infof)
    │
    ▼
content repository
    ├── GORM 查询构建 (database.DB.Where/Joins/Order)
    ├── 分页查询 (Count + Offset + Limit)
    └── CRUD 操作 (Create/Save/Delete/First/Find)
    │
    ▼
MySQL (content_reviews, creator_levels, creator_points, ...)
```

## 五、数据库表

内容创作模块共涉及 18 张数据表，详细字段定义见 [数据库设计文档](../数据库设计文档/09-内容创作.md)。以下为表关系概览：

### 5.1 创作者体系表

```
users (已有)
  │
  ├── 1:N ──→ creator_points (积分表, user_id 唯一索引)
  │              │
  │              └── N:1 ──→ creator_levels (等级表)
  │
  ├── 1:N ──→ creator_point_logs (积分变动记录, user_id 索引)
  │
  ├── 1:N ──→ creator_agreement_signs (协议签署记录)
  │              │
  │              └── N:1 ──→ creator_agreements (协议表)
  │
  └── 1:N ──→ creator_portfolios (作品集)
                 │
                 └── 1:N ──→ creator_portfolio_items (作品集内容关联)
```

### 5.2 内容管理表

```
content_versions (内容版本表)
  └── 联合索引: (content_type, content_id, version)

content_tags (标签表)
  │
  └── 1:N ──→ content_tag_relations (标签关联)
                 └── 联合唯一索引: (tag_id, content_type, content_id)

content_favorites (收藏表)
  └── 联合唯一索引: (user_id, content_type, content_id, folder_name)

content_search_logs (搜索历史表)
  └── 索引: (user_id, created_at)
```

### 5.3 审核流程表

```
review_workflows (审核流程模板)
  │
  └── 1:N ──→ review_instances (审核实例)
                 │
                 ├── 1:N ──→ review_step_logs (审核步骤记录)
                 │
                 ├── 1:N ──→ review_notifications (审核通知)
                 │
                 └── 1:N ──→ review_replies (审核意见回复)

content_reviews (内容审核表, 简化版审核记录)
```

### 5.4 关键索引设计

| 表名 | 索引 | 类型 | 用途 |
|------|------|------|------|
| `creator_points` | `user_id` | UNIQUE | 每用户一条积分记录 |
| `creator_levels` | `level` | UNIQUE | 等级值唯一 |
| `creator_agreement_signs` | `(agreement_id, user_id)` | UNIQUE | 防止重复签署 |
| `creator_portfolio_items` | `(portfolio_id, content_type, content_id)` | UNIQUE | 防止重复添加 |
| `content_tag_relations` | `(tag_id, content_type, content_id)` | UNIQUE | 防止重复关联 |
| `content_favorites` | `(user_id, content_type, content_id, folder_name)` | UNIQUE | 防止重复收藏 |
| `content_versions` | `(content_type, content_id)` | INDEX | 版本列表查询 |
| `review_instances` | `(status, content_type)` | INDEX | 审核列表筛选 |
| `content_search_logs` | `(user_id, created_at)` | INDEX | 搜索历史查询 |

### 7.1 创作者入驻

#### 7.1.1 入驻流程

```
用户申请 → 检查是否已是创作者 → 检查是否有待审核申请 → 创建申请记录
    │
    ▼
管理员审核 (admin 接口)
    │
    ├── 通过 → 分配 creator 角色 → 初始化积分记录 → 通知用户
    │
    └── 拒绝 → 记录拒绝原因 → 通知用户
```

#### 7.1.2 角色检查实现

创作者入驻依赖 RBAC 角色体系。申请时检查用户是否已拥有 `creator` 角色：

```go
// service/content/content_service.go - ApplyCreator

func ApplyCreator(userID uint, req *dto.ApplyCreatorRequest) error {
    // 1. 查询用户及其角色
    user, err := userRepo.FindByID(userID)

    // 2. 检查是否已有 creator 角色
    for _, role := range user.Roles {
        if role.Code == "creator" {
            return errors.New("您已经是创作者")
        }
    }

    // 3. 查找 creator 角色 ID
    role, err := findRoleByCode("creator")

    // 4. 检查是否已有待审核申请 (status=0)
    _, err = contentRepo.FindApplicationByUserAndRole(userID, role.ID)
    if err == nil {
        return errors.New("您已有待审核的申请")
    }

    // 5. 创建申请记录
    app := &model.RoleApplication{
        UserID: userID,
        RoleID: role.ID,
        Reason: req.Reason,
        Status: 0, // 待审核
    }
    return contentRepo.CreateApplication(app)
}
```

#### 7.1.3 创作者等级体系

等级由积分驱动，存储在 `creator_levels` 表中。积分通过以下行为获得：

| 行为 | 积分 | 说明 |
|------|------|------|
| 发布公共题目 | +10 | 审核通过后获得 |
| 题目被使用 | +2 | 被其他用户练习/考试使用 |
| 发布公共试卷 | +20 | 审核通过后获得 |
| 收到点赞 | +1 | 其他用户点赞作品集 |
| 连续登录 | +5/天 | 最多累计 30 天 |

等级评估通过定时任务 `internal/task/creator_level_eval.go`（待新建）每月初执行：

```go
// internal/task/creator_level_eval.go (待新建)

// CreatorLevelEvalTask 每月初评估创作者等级
func CreatorLevelEvalTask() {
    // 1. 获取所有创作者积分记录
    points, _ := contentRepo.ListAllCreatorPoints()

    for _, point := range points {
        // 2. 根据总积分查找对应等级
        level, err := contentRepo.FindCreatorLevel(point.TotalPoints)
        if err != nil {
            continue
        }

        // 3. 等级变化时更新并记录
        if level.Level != point.Level {
            oldLevel := point.Level
            point.Level = level.Level
            contentRepo.UpdateCreatorPoint(point)
            logger.Infof("用户 %d 等级从 %d 升/降为 %d", point.UserID, oldLevel, level.Level)
        }
    }
}
```

#### 7.1.4 创作者协议签署

### 7.4 内容审核

#### 7.4.1 审核流程概览

内容审核复用现有的 `review_instances` + `review_step_logs` 模式，与题目审核流程一致：

```
创作者提交内容审核
    │
    ▼
创建 ReviewInstance (status=0, current_step=1)
    │
    ▼
查找适用的 ReviewWorkflow (根据 content_type)
    │
    ▼
创建初始 ReviewStepLog (step_name="待审核", status=0)
    │
    ▼
发送审核通知给审核人
    │
    ▼
管理员审核
    │
    ├── 通过 (ApproveReview)
    │   ├── 更新 ReviewInstance.status = 1
    │   ├── 创建 ReviewStepLog (action="pass")
    │   ├── 更新内容状态为已发布
    │   ├── 奖励创作者积分
    │   └── 发送通知给创作者
    │
    ├── 驳回 (RejectReview)
    │   ├── 更新 ReviewInstance.status = 2
    │   ├── 创建 ReviewStepLog (action="reject")
    │   ├── 内容退回为草稿
    │   └── 发送通知给创作者（含驳回原因）
    │
    └── 需修改 (Revision)
        ├── 更新 ReviewStepLog (action="return")
        └── 发送通知给创作者
```

#### 7.4.2 审核提交实现

```go
// service/content/content_service.go - SubmitReview (待新增)

func SubmitReview(userID uint, req *dto.SubmitReviewRequest) error {
    // 1. 验证内容所有权
    // 根据 content_type 验证内容是否存在且属于当前用户

    // 2. 检查是否已有进行中的审核
    existing, _ := contentRepo.FindPendingReview(req.ContentType, req.ContentID)
    if existing != nil {
        return errors.New("该内容已有进行中的审核")
    }

    // 3. 查找适用的审核流程
    workflow, err := contentRepo.FindDefaultWorkflow(req.ContentType)
    if err != nil {
        return errors.New("未找到适用的审核流程")
    }

    // 4. 创建审核实例
    instance := &model.ReviewInstance{
        WorkflowID:  workflow.ID,
        ContentType: req.ContentType,
        ContentID:   req.ContentID,
        CurrentStep: 1,
        Status:      0, // 进行中
        CreatorID:   userID,
    }
    contentRepo.CreateReviewInstance(instance)

    // 5. 创建初始步骤记录
    stepLog := &model.ReviewStepLog{
        InstanceID: instance.ID,
        StepIndex:  1,
        StepName:   "待审核",
        Action:     "pending",
        OperatedAt: time.Now(),
    }
    contentRepo.CreateReviewStepLog(stepLog)

    // 6. 更新内容状态为审核中
    updateContentStatus(req.ContentType, req.ContentID, 2) // 审核中

    // 7. 发送审核通知给管理员
    sendReviewNotification(instance)

    return nil
}
```

func GenerateShareLink(userID uint, contentType string, contentID uint, req *dto.ShareRequest) (*dto.ShareInfo, error) {
    // 1. 生成唯一 token
    token := generateShareToken()

    shareKey := fmt.Sprintf("share:%s", token)
    shareData := map[string]interface{}{

    shareURL := fmt.Sprintf("%s/share/%s", config.Cfg.Server.BaseURL, token)
    return &dto.ShareInfo{
        Token:     token,
        URL:       shareURL,

    }, nil
}

## 八、安全考虑

### 8.1 输入校验

| 场景 | 措施 |
|------|------|
| DTO 参数校验 | 使用 `binding` tag 进行必填、长度、格式校验 |
| 富文本内容 | 敏感词过滤中间件（`middleware/sensitive.go`）拦截 POST/PUT 请求 |
| 文件上传 | 文件类型白名单、文件大小限制（10MB）、文件名清洗 |
| SQL 注入 | GORM 参数化查询，禁止字符串拼接 SQL |
| XSS | 富文本 HTML 存储前进行 sanitize，前端使用 `v-html` 时需谨慎 |

### 8.2 权限控制

| 场景 | 措施 |
|------|------|
| 创作者接口 | JWT 认证 + 检查 `creator` 角色 |
| 管理员接口 | JWT 认证 + `AdminOnly()` 中间件 |
| 资源所有权 | Service 层校验 `resource.UserID == currentUserID` |
| 审核权限 | 仅管理员可执行审核通过/驳回操作 |
| 作品集可见性 | 私有作品集仅创建者可见，公开作品集所有人可见 |

### 8.3 数据安全

| 场景 | 措施 |
|------|------|
| 分享链接 | Redis 存储，设置过期时间，token 使用 `crypto/rand` 生成 |
| 软删除 | 所有可删除资源使用 `gorm.DeletedAt` 软删除 |
| 版本快照 | JSON 快照存储，回滚不覆盖原数据，创建新版本 |
| 积分操作 | 事务保护，防止并发导致积分错误 |
| 协议签署 | 联合唯一索引防止重复签署 |

### 8.4 限流

| 接口 | 限流策略 |
|------|----------|
| 搜索接口 | 10 次/分钟/用户 |
| 审核提交 | 5 次/小时/用户 |
| 分享生成 | 20 次/天/用户 |
| 笔记创建 | 50 次/天/用户 |

## 九、错误处理

### 9.1 错误码定义

复用 `pkg/errors/` 中的 `AppError` 体系：

| 错误码 | HTTP 状态 | 场景 |
|--------|-----------|------|
| `A0001` | 400 | 参数错误（缺少必填字段、格式不正确） |
| `A0002` | 401 | 未认证（JWT 缺失或过期） |
| `A0003` | 403 | 无权限（非创作者访问创作者接口） |
| `A0004` | 404 | 资源不存在（作品集、版本、审核记录） |
| `A2001` | 400 | 业务错误：已申请创作者 |
| `A2002` | 400 | 业务错误：协议已签署 |
| `A2003` | 400 | 业务错误：内容已收藏 |
| `A2004` | 400 | 业务错误：审核状态异常 |
| `A2005` | 400 | 业务错误：版本不存在 |
| `A2006` | 400 | 业务错误：分享链接过期 |
| `A2007` | 400 | 业务错误：已有进行中的审核 |
| `A2008` | 400 | 业务错误：内容包含敏感词 |

### 9.2 统一响应格式

所有接口通过 `pkg/response/` 返回统一 JSON 信封：

```json
// 成功响应
{"code": "00000", "msg": "success", "data": {...}}

// 分页响应
{"code": "00000", "msg": "success", "data": {"list": [...], "total": 100, "page": 1, "page_size": 10}}

// 错误响应
{"code": "A2001", "msg": "您已经是创作者", "data": null}
```

### 9.3 Service 层错误处理模式

Service 层使用 `fmt.Errorf` 包装底层错误，Controller 层统一返回 HTTP 响应：

```go
// Controller 层
if err := content.ApplyCreator(userID, &req); err != nil {
    response.Error(c, http.StatusInternalServerError, err.Error())
    return
}
response.SuccessWithMessage(c, "申请提交成功", nil)

// Service 层
func ApplyCreator(userID uint, req *dto.ApplyCreatorRequest) error {
    user, err := userRepo.FindByID(userID)
    if err != nil {
        return fmt.Errorf("查询用户失败: %w", err)
    }
    // ...
}
```
