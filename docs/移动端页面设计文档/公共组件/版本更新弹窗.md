# 版本更新弹窗（Update Modal）

> **组件路径**：`components/UpdateModal/index.vue`  
> **类型**：居中弹窗（Modal）  
> **触发时机**：App 启动时检查版本、设置页点击"检查更新"

---

## 弹窗结构

### 非强制更新

```
┌─────────────────────────────┐
│    🆕 发现新版本 v2.1.0      │
│                             │
│    更新内容：                 │
│    · 新增错题导出功能         │
│    · 优化练习答题体验         │
│    · 修复已知问题             │
│                             │
│    [稍后提醒]  [立即更新]     │
└─────────────────────────────┘
```

### 强制更新

```
┌─────────────────────────────┐
│    ⚠️ 重要更新               │
│                             │
│    当前版本过旧，必须更新      │
│    才能继续使用               │
│                             │
│    更新内容：                 │
│    · 安全性升级               │
│    · 数据格式变更             │
│                             │
│       [ 立即更新 ]           │  ← 仅一个按钮
└─────────────────────────────┘
```

## 功能说明

| 更新类型 | 关闭按钮 | 稍后提醒 | 说明 |
|---------|---------|---------|------|
| 非强制更新 | ✅ 可关闭 | ✅ 可稍后 | 用户可选择不更新 |
| 强制更新 | ❌ 不可关闭 | ❌ 不可稍后 | 必须更新才能使用 |

### 版本检查逻辑

1. App 启动 → 调用 `GET /api/v1/app/version?platform=xxx&version=xxx`
2. 后端返回：`{ hasUpdate, isForce, latestVersion, downloadUrl, changelog }`
3. 有更新 → 弹出更新弹窗
4. 无更新 → 静默跳过

### 平台差异

| 平台 | 下载方式 | 说明 |
|------|---------|------|
| H5 | 跳转应用商店 | 提示用户手动更新 |
| 微信小程序 | `wx.updateManager` | 微信自动下载重启 |
| APP | 下载 APK/IPA | 显示下载进度条 |

---

## 交互逻辑

1. 检查到新版本 → 弹出更新弹窗
2. 非强制更新：点击"立即更新"→ 开始下载；点击"稍后提醒"→ 关闭，下次启动再提醒
3. 强制更新：仅"立即更新"按钮，不可关闭弹窗
4. 下载中：显示进度条，可点击"后台下载"关闭弹窗继续下载
5. 下载完成 → 自动安装（APP）或提示重启（小程序）
