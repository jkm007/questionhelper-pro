# 注册

> **路径**：`pages/login/register`
> **页面背景**：`#ffffff`

---

## 页面结构

### 导航栏
- 标题：**"注册"**
- 返回按钮

### 注册表单

- **用户名输入框**
  - placeholder：请输入用户名（4-20 位字母或数字）
  - 必填
  - 键盘类型：`type="text"`
  - 输入限制：仅允许字母和数字，长度 4-20

- **昵称输入框**
  - placeholder：请输入昵称（选填，2-20 位）
  - 选填，不填则默认使用用户名

- **邮箱输入框**
  - placeholder：请输入邮箱
  - 必填
  - 键盘类型：`type="text"`

- **邮箱验证码**
  - 输入框 + 发送按钮
  - 输入框 placeholder：请输入邮箱验证码
  - 发送按钮：点击后调用发送邮箱验证码 API，启动 60 秒倒计时（倒计时期间灰色禁用，文字变为 "XXs 后重发"）

- **图形验证码**
  - 验证码图片 + 输入框
  - 点击图片可刷新验证码
  - 输入框 placeholder：请输入验证码
  - 验证码图片由后端 `/api/v1/auth/captcha` 接口返回（captchaId + captchaBase64）

- **密码输入框**
  - emoji 图标显示/隐藏切换：🙈 / 👁️
  - placeholder：请输入密码（6-20 位，需包含字母和数字）

- **确认密码**
  - emoji 图标显示/隐藏切换：🙈 / 👁️
  - placeholder：请再次输入密码

- **手机号输入框**
  - placeholder：请输入手机号（选填）
  - 选填
  - 键盘类型：`type="number"`

- **用户协议**
  - 复选框 + 文字："我已阅读并同意《用户协议》和《隐私政策》"
  - "用户协议"和"隐私政策"为可点击链接（`#4A90D9`）

### 注册按钮
- `#4A90D9` 全宽圆角按钮
- 文字：**"注册"**

### 底部链接
- **"已有账号？去登录"**（24rpx，`#4A90D9`）

## 交互逻辑

1. 表单验证：
   - 用户名：4-20 位，仅字母和数字，必填
   - 昵称：选填，若填写则 2-20 位
   - 邮箱：合法邮箱格式，必填
   - 邮箱验证码：6 位数字，必填
   - 图形验证码：非空，必填
   - 密码：6-20 位，需包含字母和数字
   - 确认密码：必须与密码一致
   - 手机号：选填，若填写则 11 位数字
   - 必须勾选用户协议
2. 页面加载时 → 调用图形验证码接口获取 captchaId 和 captchaBase64 并展示
3. 点击图形验证码图片 → 重新获取验证码
4. 点击"发送验证码"→ 先校验邮箱格式，再调用发送邮箱验证码 API（`/api/v1/auth/email/send`），启动 60 秒倒计时
5. 点击"注册"按钮 → 校验全部表单字段 → 调用注册 API
6. 注册成功 → Toast 提示"注册成功" + `redirectTo` 登录页（注册后自动登录，token 存储后也可直接跳转首页）
7. 注册失败 → Toast 显示错误信息（用户名已存在、邮箱已被注册、验证码错误等）
8. 点击协议链接 → `navigateTo` 协议页

## API 接口

### 1. 获取图形验证码

- **请求方式**：`GET`
- **路径**：`/api/v1/auth/captcha`
- **响应**：

```json
{
  "code": "00000",
  "msg": "success",
  "data": {
    "captchaId": "xxx",
    "captchaBase64": "data:image/png;base64,..."
  }
}
```

### 2. 发送邮箱验证码

- **请求方式**：`POST`
- **路径**：`/api/v1/auth/email/send`
- **请求体**：

```json
{
  "email": "user@example.com"
}
```

### 3. 用户注册

- **请求方式**：`POST`
- **路径**：`/api/v1/auth/register`
- **请求体**：

```json
{
  "username": "testuser",
  "password": "abc123456",
  "confirmPassword": "abc123456",
  "nickname": "测试用户",
  "phone": "13800138000",
  "email": "user@example.com",
  "emailCode": "123456",
  "captchaId": "captcha-xxx",
  "captchaCode": "abcd",
  "agreement": true,
  "source": "h5"
}
```

- **请求参数说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，4-20 位字母或数字 |
| password | string | 是 | 密码，6-20 位，需包含字母和数字 |
| confirmPassword | string | 是 | 确认密码，必须与 password 一致 |
| nickname | string | 否 | 昵称，2-20 位，不填则默认使用用户名 |
| phone | string | 否 | 手机号 |
| email | string | 是 | 邮箱地址 |
| emailCode | string | 是 | 邮箱验证码，6 位数字 |
| captchaId | string | 是 | 图形验证码 ID（获取验证码接口返回） |
| captchaCode | string | 是 | 图形验证码值（用户输入） |
| agreement | boolean | 是 | 是否同意用户协议 |
| source | string | 否 | 注册来源：h5 / miniapp / app / web，移动端传 `"h5"` |

- **响应**（注册成功后自动登录，返回 Token）：

```json
{
  "code": "00000",
  "msg": "注册成功",
  "data": {
    "accessToken": "eyJhbG...",
    "refreshToken": "eyJhbG...",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "user": {
      "id": 1,
      "username": "testuser",
      "nickname": "测试用户",
      "createdAt": "2026-06-02T10:00:00Z"
    }
  }
}
```
