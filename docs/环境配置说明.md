## 二、远程服务连接 (SSH 隧道)

### 2.1 架构说明

开发环境下，MySQL、Redis、MinIO 均部署在远程服务器上，本地开发通过 SSH 隧道进行安全连接：

```
本地开发机                          远程服务器 (182.92.4.169)
┌─────────────┐                   ┌─────────────────────┐
│ localhost    │   SSH 隧道        │                     │
│   :3306  ────┼──────────────────►│  MySQL   :3306      │
│   :6379  ────┼──────────────────►│  Redis   :6379      │
│   :9000  ────┼──────────────────►│  MinIO   :9000      │
│   :9001  ────┼──────────────────►│  MinIO   :9001 (UI) │
└─────────────┘                   └─────────────────────┘
```

隧道建立后，本地访问 `localhost:3306` 等同于访问远程服务器的对应端口，应用配置无需修改。

### 2.2 创建 SSH 隧道

```bash
sshpass -p 'iqh.88' ssh -o StrictHostKeyChecking=no -p 6000 -f -N \
  -L 3306:localhost:3306 \
  -L 6379:localhost:6379 \
  -L 9000:localhost:9000 \
  -L 9001:localhost:9001 \
  root@182.92.4.169
```

**参数说明**：

| 参数 | 说明 |
|------|------|
| `sshpass -p 'iqh.88'` | 自动输入 SSH 密码 |
| `-o StrictHostKeyChecking=no` | 跳过主机密钥确认 (首次连接不提示) |
| `-p 6000` | SSH 端口为 6000 (非默认 22) |
| `-f` | SSH 连接建立后转入后台运行 |
| `-N` | 不执行远程命令，仅做端口转发 |
| `-L 3306:localhost:3306` | 本地 3306 → 远程 3306 (MySQL) |
| `-L 6379:localhost:6379` | 本地 6379 → 远程 6379 (Redis) |
| `-L 9000:localhost:9000` | 本地 9000 → 远程 9000 (MinIO API) |
| `-L 9001:localhost:9001` | 本地 9001 → 远程 9001 (MinIO 控制台) |

### 2.3 验证隧道连接

```bash
# 验证 MySQL
mysql -h 127.0.0.1 -P 3306 -u root -p -e "SELECT 1;"

# 验证 Redis
redis-cli -h 127.0.0.1 -p 6379 ping

# 验证 MinIO
curl -s http://localhost:9000/minio/health/live
```

### 2.4 关闭 SSH 隧道

```bash
# 查找隧道进程
ps aux | grep "ssh.*182.92.4.169"

# 杀掉隧道进程
kill $(ps aux | grep "ssh.*182.92.4.169" | grep -v grep | awk '{print $2}')
```

或者使用一条命令：

```bash
pkill -f "ssh.*182.92.4.169"
```

### 2.5 远程服务器信息

| 服务 | 远程地址 | 端口 | 用户名 | 说明 |
|------|----------|------|--------|------|
| SSH | 182.92.4.169 | 6000 | root | SSH 登录 |
| MySQL | localhost (远程) | 3306 | root | 数据库 |
| Redis | localhost (远程) | 6379 | (无密码) | 缓存 |
| MinIO API | localhost (远程) | 9000 | | 对象存储 API |
| MinIO Console | localhost (远程) | 9001 | | MinIO 管理界面 |

> **注意**：远程服务均绑定在 `localhost`，只能通过 SSH 隧道访问，不暴露公网端口。

### 2.6 开发环境启动流程


## 三、后端配置 (config.yaml)

配置文件路径：`questionhelper-server/config/config.yaml`

> **前提**：确保 SSH 隧道已建立 (参见 [第二节](#二远程服务连接-ssh-隧道))，本地 3306/6379/9000 端口已映射到远程服务器。

### 完整配置示例

```yaml
server:
  port: 8080
  mode: debug

mysql:
  host: 127.0.0.1    # 通过 SSH 隧道连接远程 MySQL
  port: 3306
  user: root
  password: "root123456"
  dbname: questionhelper

redis:
  host: 127.0.0.1    # 通过 SSH 隧道连接远程 Redis
  port: 6379
  password: ""
  db: 0

jwt:
  secret: "questionhelper-secret-key-2024"
  expire: 7200
  refresh_expire: 604800

log:
  level: info
  format: json

oss:
  type: local                        # local=本地存储, minio=MinIO对象存储
  endpoint: ""                       # MinIO 时填写 "http://127.0.0.1:9000"
  access_key: ""
  secret_key: ""
  bucket: ""
  cdn: ""
```

> **注意**：默认使用 `local` 本地存储。如需使用 MinIO，将 `type` 改为 `minio` 并填写对应的 endpoint、access_key、secret_key 和 bucket。

### 配置项详细说明

#### server (服务器配置)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| server.port | int | 8080 | HTTP 服务监听端口 |
| server.mode | string | debug | 运行模式: `debug` / `release` / `test` |

- `debug` 模式：详细日志、热重载、CORS 宽松策略
- `release` 模式：精简日志、Gin 性能优化、严格 CORS

#### mysql (数据库配置)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| mysql.host | string | 127.0.0.1 | MySQL 主机地址 |
| mysql.port | int | 3306 | MySQL 端口 |
| mysql.user | string | root | 数据库用户名 |
| mysql.password | string | root123456 | 数据库密码 |
| mysql.dbname | string | questionhelper | 数据库名称 |

**数据库初始化**：
```sql
CREATE DATABASE questionhelper
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;
```

**字符集要求**：必须使用 `utf8mb4` 以支持 emoji 和特殊字符。

#### redis (缓存配置)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| redis.host | string | 127.0.0.1 | Redis 主机地址 |
| redis.port | int | 6379 | Redis 端口 |
| redis.password | string | (空) | Redis 密码 (生产环境必须设置) |
| redis.db | int | 0 | Redis 数据库编号 (0-15) |

**Redis 用途**：
- JWT Token 黑名单
- 验证码存储
- 会话缓存
- 频率限制计数器

#### jwt (JWT 认证配置)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| jwt.secret | string | questionhelper-secret-key-2024 | JWT 签名密钥 |
| jwt.expire | int | 7200 | Access Token 有效期 (秒) |
| jwt.refresh_expire | int | 604800 | Refresh Token 有效期 (秒) |

**默认值说明**：
- Access Token: 7200 秒 = 2 小时
- Refresh Token: 604800 秒 = 7 天

**安全要求**：
- 生产环境必须更换 `jwt.secret` 为随机强密钥 (至少 32 位)
- 密钥泄露后需立即更换并清除所有现有 Token

#### log (日志配置)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| log.level | string | info | 日志级别: `debug` / `info` / `warn` / `error` |
| log.format | string | json | 日志格式: `json` / `console` |

**日志级别说明**：
- `debug`: 最详细，包含所有调试信息 (仅开发环境)
- `info`: 常规运行信息 (推荐生产环境)
- `warn`: 警告信息
- `error`: 仅错误信息

#### oss (文件存储配置)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| oss.type | string | local | 存储类型: `local` / `minio` / `s3` |
| oss.endpoint | string | (空) | 对象存储端点 (minio/s3 时必填) |
| oss.access_key | string | (空) | Access Key |
| oss.secret_key | string | (空) | Secret Key |
| oss.bucket | string | (空) | 存储桶名称 |
| oss.cdn | string | (空) | CDN 加速域名 (可选) |

**存储类型说明**：
- `local`: 本地文件系统，文件存储在 `uploads/` 目录
- `minio`: MinIO 对象存储 (S3 兼容)
- `s3`: AWS S3 或兼容服务

## 四、管理后台环境变量

配置文件路径：`questionhelper-admin/.env.development` 和 `questionhelper-admin/.env.production`

### 环境变量列表

| 变量名 | 说明 |
|--------|------|
| VITE_APP_BASE_API | API 请求基础路径 (开发: `/dev-api`, 生产: `/prod-api`) |
| VITE_APP_TITLE | 应用标题 |
| VITE_MOCK_DEV_SERVER | 是否启用 Mock 服务器 |
| VITE_APP_TENANT_ENABLED | 是否启用多租户 |

### 开发环境配置 (.env.development)

```env
VITE_APP_BASE_API=/dev-api
VITE_APP_TITLE=QuestionHelper管理后台
VITE_MOCK_DEV_SERVER=false
VITE_APP_TENANT_ENABLED=false
```

### 生产环境配置 (.env.production)

```env
VITE_APP_BASE_API=/prod-api
VITE_APP_TITLE=题小助
VITE_APP_TENANT_ENABLED=false
```

### API 代理配置

开发环境下，Vite 会将 `/dev-api` 前缀的请求代理到后端服务（实际前缀取自 `.env.development` 中的 `VITE_APP_BASE_API`）：

```typescript
// vite.config.ts 中的代理配置（动态读取环境变量）
proxy: {
  '/dev-api': {
    target: 'http://localhost:8080',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/dev-api/, '')
  }
}
```

## 六、Docker 配置

### docker-compose.yml 服务说明

```yaml
services:
  mysql:
    image: mysql:8.0
    ports: ["3306:3306"]
    environment:
      MYSQL_ROOT_PASSWORD: root123456
      MYSQL_DATABASE: questionhelper
    volumes:
      - mysql_data:/var/lib/mysql
      - ./deploy/mysql/init.sql:/docker-entrypoint-initdb.d/init.sql

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
    volumes:
      - redis_data:/data

  server:
    build: ./questionhelper-server
    ports: ["8080:8080"]
    environment:
      GIN_MODE: release
    depends_on: [mysql, redis]
    volumes:
      - ./questionhelper-server/config:/app/config
      - ./uploads:/app/uploads

  admin:
    build: ./questionhelper-admin
    ports: ["3000:80"]
    depends_on: [server]
```

### 环境变量覆盖

可通过 `.env` 文件覆盖 Docker Compose 中的默认值：

```env
# .env
MYSQL_ROOT_PASSWORD=your_secure_password
MYSQL_DATABASE=questionhelper
REDIS_PASSWORD=your_redis_password
```

| MySQL | 3306 | 3306 | 数据库 |
| Redis | 6379 | 6379 | 缓存 |
| Server | 8080 | 8080 | API 服务 |
| Admin | 80 | 3000 | 管理后台 |

## 七、第三方服务配置

### 6.1 MySQL 8.0

#### 安装

```bash
# macOS
brew install mysql@8.0

# Ubuntu
sudo apt install mysql-server-8.0

# Docker
docker run -d --name mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root123456 \
  -e MYSQL_DATABASE=questionhelper \
```

#### 连接配置

```bash
# 命令行连接
mysql -h 127.0.0.1 -P 3306 -u root -p

# 创建数据库
CREATE DATABASE questionhelper DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

#### 生产环境建议

- 设置强密码
- 创建专用数据库用户 (非 root)
- 开启 binlog 用于备份
- 配置慢查询日志

### 6.2 Redis 7

#### 安装

```bash
# macOS
brew install redis

# Ubuntu
sudo apt install redis-server

# Docker
docker run -d --name redis \
  -p 6379:6379 \
  redis:7-alpine
```

#### 连接配置

```bash
# 命令行连接
redis-cli -h 127.0.0.1 -p 6379

# 设置密码 (redis.conf)

```

#### 生产环境建议

- 设置密码
- 配置 maxmemory 和淘汰策略
- 开启 AOF 持久化
- 使用 Redis Sentinel 或 Cluster 实现高可用

### 6.3 MinIO

#### 开发环境

MinIO 部署在远程服务器，通过 SSH 隧道访问。确保隧道已建立：

```bash
sshpass -p 'iqh.88' ssh -o StrictHostKeyChecking=no -p 6000 -f -N \
  -L 9000:localhost:9000 \
  -L 9001:localhost:9001 \
  root@182.92.4.169
```

- **MinIO API**: http://localhost:9000
- **MinIO 控制台**: http://localhost:9001 (浏览器访问管理界面)

#### 本地部署 (可选)

如需在本地运行 MinIO：

```bash
docker run -d --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  minio/minio server /data --console-address ":9001"
```

#### 配置

```yaml
# config.yaml
oss:
  type: minio
  endpoint: "http://127.0.0.1:9000"  # SSH 隧道映射
  access_key: "minioadmin"
  secret_key: "minioadmin"
  bucket: "questionhelper"
```

## 附录：常见问题

### Q: 数据库连接失败

检查项：
1. MySQL 服务是否启动
2. config.yaml 中的 host/port/user/password 是否正确
3. 数据库 `questionhelper` 是否已创建
4. MySQL 用户是否有远程连接权限

### Q: Redis 连接失败

检查项：
1. Redis 服务是否启动
2. config.yaml 中的 host/port/password 是否正确
3. Redis 是否配置了 bind 地址限制

### Q: 前端无法访问后端 API

检查项：
1. 后端服务是否在运行 (http://localhost:8080/health)
2. `.env` 中的 `VITE_APP_API_URL` 是否正确
3. 浏览器控制台是否有 CORS 错误
4. 开发服务器的代理配置是否正确

### Q: 文件上传失败

检查项：
1. `uploads/` 目录是否存在且有写入权限
2. 如果使用 MinIO，检查 OSS 配置是否正确
3. 文件大小是否超过限制
