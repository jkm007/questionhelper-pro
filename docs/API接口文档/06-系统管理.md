# 06 系统管理 API

本文档覆盖字典管理、菜单管理、公告管理、系统配置、标签管理、实名认证审核、角色申请管理、缓存管理、功能开关、存储配置等系统级接口。

## 1. 字典类型管理

字典类型（dict-types）用于定义全局 key-value 字典的分类，前端通过 `DictSelect` / `DictTag` 组件消费字典数据。

### 用户端接口 (需登录)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/dict/types` | 获取字典类型列表（仅启用状态） |

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/dict/types` | 获取全部字典类型列表 |
| GET | `/api/v1/admin/dict/types/:id` | 获取字典类型详情 |
| POST | `/api/v1/admin/dict/types` | 创建字典类型 |
| PUT | `/api/v1/admin/dict/types/:id` | 更新字典类型 |
| DELETE | `/api/v1/admin/dict/types/:id` | 删除字典类型 |

**POST /api/v1/admin/dict/types**

```json
// 请求
{
  "type_code": "user_status",
  "type_name": "用户状态",
  "status": 1,
  "remark": "用户账号状态字典"
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**GET /api/v1/admin/dict/types** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keyword` | string | 否 | 搜索关键词，匹配 type_code / type_name |
| `status` | int | 否 | 状态筛选: 0=禁用, 1=启用 |

**响应字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 主键 |
| `type_code` | string | 字典类型编码（唯一） |
| `type_name` | string | 字典类型名称 |
| `status` | int8 | 状态: 0=禁用, 1=启用 |
| `remark` | string | 备注 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

## 2. 字典数据管理

字典数据（dict-data）按字典类型存储具体的选项列表，支持 Redis HASH 缓存（Key 格式 `dict:{type_code}`，TTL 24 小时）。

### 用户端接口 (需登录)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/dict/data` | 按类型编码获取字典数据列表 |

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/dict/data` | 获取字典数据列表（支持分页） |
| GET | `/api/v1/admin/dict/data/:id` | 获取字典数据详情 |
| POST | `/api/v1/admin/dict/data` | 创建字典数据 |
| PUT | `/api/v1/admin/dict/data/:id` | 更新字典数据 |
| DELETE | `/api/v1/admin/dict/data/:id` | 删除字典数据 |

**GET /api/v1/dict/data** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type_code` | string | 是 | 字典类型编码 |

**GET /api/v1/admin/dict/data** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type_code` | string | 否 | 字典类型编码 |
| `status` | int | 否 | 状态筛选 |
| `keyword` | string | 否 | 搜索关键词 |
| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页条数，默认 10 |

**POST /api/v1/admin/dict/data**

```json
// 请求
{
  "type_code": "user_status",
  "data_label": "启用",
  "data_value": "1",
  "sort": 1,
  "status": 1,
  "css_class": "el-tag--success",
  "remark": "正常启用状态"
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**响应字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 主键 |
| `type_code` | string | 所属字典类型编码 |
| `data_label` | string | 显示标签 |
| `data_value` | string | 存储值 |
| `sort` | int | 排序 |
| `status` | int8 | 状态: 0=禁用, 1=启用 |
| `css_class` | string | 前端样式类名 |
| `remark` | string | 备注 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

**错误码**

| 错误码 | HTTP | 说明 |
|--------|------|------|
| `A3010` | 400 | 字典类型编码已存在 |
| `A3011` | 400 | 字典数据值在该类型下已存在 |

## 3. 菜单管理

菜单采用树形结构，通过 `parent_id` 关联父子关系。类型分为目录（type=1）、菜单（type=2）、按钮（type=3）。

### 用户端接口 (需登录)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/menus/routes` | 获取当前用户路由（前端动态路由格式） |
| GET | `/api/v1/menu/user` | 获取当前用户菜单列表 |
| GET | `/api/v1/menu/buttons` | 获取当前用户按钮权限标识 |

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/menus` | 获取所有菜单（平铺列表） |
| GET | `/api/v1/admin/menus/tree` | 获取菜单树形结构 |
| GET | `/api/v1/admin/menus/:id` | 获取菜单详情 |
| POST | `/api/v1/admin/menus` | 创建菜单 |
| PUT | `/api/v1/admin/menus/:id` | 更新菜单 |
| DELETE | `/api/v1/admin/menus/:id` | 删除菜单 |

**POST /api/v1/admin/menus**

```json
// 请求
{
  "parent_id": null,
  "name": "SystemManage",
  "path": "/system",
  "component": "Layout",
  "redirect": "/system/user",
  "title": "系统管理",
  "icon": "setting",
  "hidden": false,
  "type": 1,
  "permission": "",
  "sort": 100
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**PUT /api/v1/admin/menus/:id**

```json
// 请求
{
  "title": "系统管理",
  "icon": "setting",
  "hidden": false,
  "sort": 100,
  "status": 1
}
```

**响应字段（MenuInfo）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 主键 |
| `parent_id` | uint | 父菜单 ID，null 表示顶级 |
| `name` | string | 路由名称（唯一标识） |
| `path` | string | 路由路径 |
| `component` | string | 前端组件路径 |
| `redirect` | string | 重定向路径 |
| `title` | string | 菜单标题（显示名称） |
| `icon` | string | 菜单图标 |
| `hidden` | bool | 是否隐藏 |
| `type` | int8 | 类型: 1=目录, 2=菜单, 3=按钮 |
| `permission` | string | 权限标识（如 `user:create`） |
| `sort` | int | 排序值 |
| `status` | int8 | 状态: 0=禁用, 1=启用 |
| `children` | array | 子菜单列表（树形接口返回） |

**错误码**

| 错误码 | HTTP | 说明 |
|--------|------|------|
| `A3003` | 400 | 存在子菜单，无法删除 |

## 5. 系统配置管理

系统配置采用通用 key-value 模型（`system_settings` 表），按业务模块分组查询，支持 Redis 缓存（Key 格式 `setting:{key}`，TTL 1 小时）。

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/settings` | 获取全部系统设置 |
| PUT | `/api/v1/admin/settings` | 更新系统设置（批量） |
| GET | `/api/v1/admin/settings/class` | 获取班级相关设置 |
| PUT | `/api/v1/admin/settings/class` | 更新班级相关设置 |
| GET | `/api/v1/admin/settings/resource` | 获取资源相关设置 |
| PUT | `/api/v1/admin/settings/resource` | 更新资源相关设置 |
| GET | `/api/v1/admin/security` | 获取安全配置 |
| PUT | `/api/v1/admin/security` | 更新安全配置 |

```json
{
```

|------|------|------|
| GET | `/api/v1/admin/realnames` | 获取实名认证审核列表 |
| PUT | `/api/v1/admin/realnames/:id/review` | 审核实名认证 |

**POST /api/v1/user/realname**

```json
// 请求
{
  "real_name": "张三",
  "id_card": "110101199001011234"
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
```

**PUT /api/v1/admin/realnames/:id/review**

```json
// 请求
{
  "status": 1,
  "reason": ""
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**GET /api/v1/user/realname 响应字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 主键 |

## 9. 缓存管理

缓存管理接口用于查看 Redis 缓存运行状态和按需清除缓存。

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/cache/stats` | 获取缓存统计信息 |
| POST | `/api/v1/admin/cache/clear` | 清除缓存 |

**GET /api/v1/admin/cache/stats 响应**

```json
{
  "code": "00000",
  "msg": "success",
  "data": {
    "total_keys": 1234,
    "used_memory": "12.5MB",
    "used_memory_peak": "25.0MB",
    "hit_rate": 0.95,
    "cache_details": [
      {
        "cache_key": "menu:tree",
        "ttl": 86400,
        "description": "完整菜单树缓存",
        "is_enabled": true,
        "last_cleared": "2026-05-31T10:00:00Z"
      }
    ]
  }
}
```

**POST /api/v1/admin/cache/clear**

```json
// 请求（支持空 body 清除全部缓存）
{
  "pattern": "dict:*"
}

// 响应
{
  "code": "00000",
  "msg": "缓存清除成功",
  "data": null
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pattern` | string | 否 | Redis Key 匹配模式（支持通配符），为空则清除全部 |

**缓存统计响应字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `total_keys` | int64 | 缓存 Key 总数 |
| `used_memory` | string | 当前内存使用量 |
| `used_memory_peak` | string | 内存使用峰值 |
| `hit_rate` | float64 | 缓存命中率 |
| `cache_details` | array | 各类缓存详情列表 |

**缓存详情项字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `cache_key` | string | 缓存 Key 前缀/名称 |
| `ttl` | int | 过期时间（秒） |
| `description` | string | 缓存用途描述 |
| `is_enabled` | bool | 是否启用 |
| `last_cleared` | datetime | 最近清除时间 |

**常用缓存 Key 说明**

| 缓存用途 | Key 格式 | 数据结构 | 过期时间 |
|----------|----------|----------|----------|
| 用户菜单 | `menu:user:{userID}` | STRING (JSON) | 24 小时 |
| 角色菜单 | `menu:role:{roleID}` | STRING (JSON) | 24 小时 |
| 菜单树 | `menu:tree` | STRING (JSON) | 24 小时 |
| 系统配置 | `setting:{key}` | STRING | 1 小时 |
| 功能开关 | `feature:{key}` | STRING | 1 小时 |
| 字典数据 | `dict:{type_code}` | HASH | 24 小时 |
| 用户权限 | `user:permissions:{userID}` | SET | 5 分钟 |

## 11. 存储配置管理

存储配置用于管理文件存储后端（本地 / MinIO / OSS / S3），支持多配置并存，指定一个默认配置。

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/storage` | 获取存储配置列表 |
| POST | `/api/v1/admin/storage` | 创建存储配置 |
| PUT | `/api/v1/admin/storage/:id` | 更新存储配置 |

**POST /api/v1/admin/storage**

```json
// 请求
{
  "name": "MinIO 生产环境",
  "type": "minio",
  "endpoint": "http://minio.example.com:9000",
  "bucket": "questionhelper",
  "access_key": "AKIAIOSFODNN7EXAMPLE",
  "secret_key": "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY",
  "region": "us-east-1",
  "base_url": "https://cdn.example.com",
  "is_default": true,
  "is_active": true
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**PUT /api/v1/admin/storage/:id**

```json
// 请求
{
  "name": "MinIO 生产环境",
  "type": "minio",
  "endpoint": "http://minio.example.com:9000",
  "bucket": "questionhelper-prod",
  "access_key": "AKIAIOSFODNN7EXAMPLE",

  "region": "us-east-1",
  "base_url": "https://cdn.example.com",
  "is_default": true,
  "is_active": true
}
```

**响应字段（StorageConfig）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 主键 |
| `name` | string | 配置名称 |
| `type` | string | 存储类型: local / minio / oss / s3 |
| `endpoint` | string | 服务端点 URL |

| `access_key` | string | 访问密钥 ID |

| `base_url` | string | 基础访问 URL / CDN 域名 |
| `is_default` | bool | 是否默认存储 |
| `is_active` | bool | 是否启用 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

**存储类型说明**

| 类型 | 说明 |
|------|------|
| `local` | 本地文件系统存储 |

| `oss` | 阿里云 OSS |
| `s3` | AWS S3 |
