# 09 内容创作 API

## 用户端接口 (需登录)

### 创作者申请

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/user/apply/creator` | 申请成为创作者 |

---

#### POST `/api/v1/user/apply/creator`

普通用户提交创作者申请，需提供实名信息和申请理由。申请后需等待管理员审核。

**认证**: 需要 JWT Bearer Token

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `reason` | string | 是 | 申请理由，最长 500 字符 |
| `real_name` | string | 是 | 真实姓名，最长 50 字符 |
| `id_card` | string | 是 | 身份证号 |
| `phone` | string | 是 | 联系电话 |
| `email` | string | 否 | 邮箱地址 |
| `description` | string | 否 | 个人简介/擅长领域，最长 1000 字符 |

**请求示例**

```json
{
  "reason": "擅长数学教学，有5年出题经验",
  "real_name": "张三",
  "id_card": "110101199001011234",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "description": "高级数学教师，长期从事高考数学题目研究"
}
```

**响应示例**

```json
{
  "code": "00000",
  "msg": "申请提交成功"
}
```

**错误代码**

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `A0001` | 400 | 请求参数错误（缺少必填项或格式不符） |
| `B0001` | 500 | 服务器内部错误（如重复申请） |

#### GET `/api/v1/creator/profile`

获取当前用户的创作者资料信息，包括等级、积分、是否已签署协议等。

**认证**: 需要 JWT Bearer Token

**响应示例**

```json
{
  "code": "00000",
  "msg": "success",
  "data": {
    "user_id": 1001,
    "username": "creator01",
    "nickname": "创作达人",
    "avatar": "https://example.com/avatar.jpg",
    "bio": "专注教育内容创作",
    "level": 3,
    "level_name": "金牌创作者",
    "total_points": 2500,
    "is_creator": true,
    "agreement": true,
    "created_at": "2026-01-15T10:00:00Z"
  }
}
```

**响应参数**

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | int | 用户 ID |
| `username` | string | 用户名 |
| `nickname` | string | 昵称 |
| `avatar` | string | 头像 URL |
| `bio` | string | 个人简介 |
| `level` | int | 创作者等级 |
| `level_name` | string | 等级名称 |
| `total_points` | int | 累计积分 |
| `is_creator` | boolean | 是否为创作者 |
| `agreement` | boolean | 是否已签署创作者协议 |
| `created_at` | string | 创建时间 (ISO 8601) |

**错误代码**

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `B0001` | 500 | 服务器内部错误 |

#### GET `/api/v1/creator/points/logs`

分页获取当前用户的积分变动记录，支持按变动类型筛选。

**认证**: 需要 JWT Bearer Token

**查询参数**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `page` | int | 否 | 1 | 页码 |
| `page_size` | int | 否 | 10 | 每页数量，最大 100 |
| `change_type` | string | 否 | - | 变动类型筛选 |

**响应示例**

```json
{
  "code": "00000",
  "msg": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "change_type": "earn",
        "points": 100,
        "balance": 2500,
        "reason": "发布优质题目获得奖励",
        "created_at": "2026-06-01T10:00:00Z"
      },
      {
        "id": 2,
        "change_type": "redeem",
        "points": -200,
        "balance": 2400,
        "reason": "兑换会员权益",
        "created_at": "2026-05-28T14:30:00Z"
      }
    ],
    "total": 50,
    "page": 1,
    "page_size": 10
  }
}
```

**响应参数**

| 字段 | 类型 | 说明 |
|------|------|------|
| `list[].id` | int | 记录 ID |
| `list[].change_type` | string | 变动类型: `earn`(获取)、`redeem`(兑换)、`deduct`(扣除) |
| `list[].points` | int | 积分变动值（正数为增加，负数为减少） |
| `list[].balance` | int | 变动后积分余额 |
| `list[].reason` | string | 变动原因 |
| `list[].created_at` | string | 变动时间 (ISO 8601) |
| `total` | int | 总记录数 |
| `page` | int | 当前页码 |
| `page_size` | int | 每页数量 |

**错误代码**

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `A0001` | 400 | 请求参数错误 |
| `B0001` | 500 | 服务器内部错误 |
