## 1. 用户端评论接口

需登录（JWT Bearer Token）。

### 接口总览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/comments` | 评论列表 |
| POST | `/api/v1/comments` | 发表评论 |
| PUT | `/api/v1/comments/:id` | 编辑评论 |

### GET /api/v1/comments

**权限**: 需登录

| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页条数，默认 10，最大 100 |
| `status` | int8 | 否 | 状态筛选：`0`=隐藏, `1`=正常, `2`=举报 |
| `keyword` | string | 否 | 搜索关键词 |

**响应示例**:

### POST /api/v1/comments

发表评论，支持回复（嵌套）、图片、@用户。

**权限**: 需登录

**请求体**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `target_type` | int8 | 是 | 目标类型：`1`=题目, `2`=考试, `3`=班级 |
| `target_id` | uint | 是 | 目标ID |
| `content` | string | 是 | 评论内容，最长 500 字符 |
| `parent_id` | uint | 否 | 父评论ID（回复时填写） |
| `images` | string | 否 | 图片URL列表，JSON 数组字符串 |
| `mentions` | string | 否 | @的用户ID列表，JSON 数组字符串 |
| `reply_to_user_id` | uint | 否 | 回复的目标用户ID |

**请求示例**:

```json
{
  "target_type": 1,
  "target_id": 42,
  "content": "@张三 你说的对，这道题确实应该这样解",
  "parent_id": 1,
  "images": "[\"https://example.com/img/solution.jpg\"]",
  "mentions": "[5]",
  "reply_to_user_id": 5
}
```

**响应**:

```json
{
  "code": "00000",
  "msg": "评论成功",
  "data": null
}
```

### GET /api/v1/admin/comments

获取全部评论列表，支持多维度筛选。与用户端评论列表不同，此接口可查看所有目标下的全部评论。

**权限**: 管理员

**查询参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页条数，默认 10，最大 100 |
| `target_type` | int8 | 否 | 目标类型筛选：`1`=题目, `2`=考试, `3`=班级 |
| `status` | int8 | 否 | 状态筛选：`0`=隐藏, `1`=正常, `2`=举报 |
| `keyword` | string | 否 | 搜索关键词 |
| `user_id` | uint | 否 | 按用户ID筛选 |
| `start_date` | time | 否 | 起始时间（ISO 8601 格式） |
| `end_date` | time | 否 | 结束时间（ISO 8601 格式） |

**响应**: 同用户端评论列表响应格式（分页结构）。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|


```json
{
```
```json
  "user_id": 20,
  "target_type": 1,
  "target_id": 42,

}
```


## 附录: 枚举与常量

### 评论目标类型 (target_type)

| 值 | 说明 |
|----|------|
| `1` | 题目 |
| `2` | 考试 |
| `3` | 班级 |

### 评论状态 (status)

| 值 | 说明 |
|----|------|
| `0` | 隐藏（审核未通过或管理员隐藏） |
| `1` | 正常 |
| `2` | 被举报 |

### 举报原因类型 (reason_type)

| 值 | 说明 |
|----|------|
| `spam` | 垃圾信息 |
| `abuse` | 辱骂 |
| `infringement` | 侵权 |
| `other` | 其他 |

### 举报处理状态 (status)

| 值 | 说明 |
|----|------|
| `0` | 待处理 |
| `1` | 已处理（举报成立） |
| `2` | 已驳回（举报不成立） |

### 黑名单目标类型 (target_type)

| 值 | 说明 |
|----|------|
| `0` | 全局（在所有目标下禁止评论） |
| `1` | 题目 |
| `2` | 考试 |
| `3` | 班级 |

### 审核规则类型 (rule_type)

| 值 | 说明 | pattern 示例 |
|----|------|--------------|
| `keyword` | 关键词匹配 | `代考,包过,加微信`（逗号分隔） |
| `regex` | 正则表达式 | `1[3-9]\\d{9}`（匹配手机号） |
| `length` | 长度限制 | `500`（超过500字符触发） |
| `repeat` | 重复检测 | `3`（连续重复3次以上触发） |

### 审核规则动作 (action)

| 值 | 说明 |
|----|------|
| `1` | 标记待审（人工复核） |
| `2` | 自动隐藏 |
| `3` | 自动删除 |
| `4` | 拒绝发布（发布时拦截） |

### 审核规则状态 (status)

| 值 | 说明 |
|----|------|
| `0` | 禁用 |
| `1` | 启用 |
