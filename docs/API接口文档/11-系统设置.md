### 基本设置

---

#### GET `/api/v1/admin/settings`

获取系统全局设置，以键值对形式返回所有系统配置项。

**认证**: 需要 JWT Bearer Token + Admin 权限

**请求参数**

无。

**响应示例**

```json
{
  "code": "00000",
  "msg": "success",
  "data": {
    "site_name": "题小助",
    "site_description": "教育领域的题库管理与在线考试平台",
    "site_url": "https://example.com",
    "icp_number": "京ICP备XXXXXXXX号",
    "contact_email": "admin@example.com",
    "contact_phone": "400-000-0000"
  }
}
```

**响应参数**

响应 `data` 为键值对对象（`map[string]string`），字段数量和名称取决于系统配置项。每个键值对对应 `system_settings` 表中的一条记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| `{key}` | string | 系统配置键名 |
| `{key}` 的值 | string | 对应的配置值 |

**错误代码**

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `A0200` | 401 | 未认证（未提供或无效的 Token） |
| `A0301` | 403 | 无管理员权限 |
| `B0001` | 500 | 服务器内部错误 |

---

#### PUT `/api/v1/admin/settings`

更新系统全局设置，传入需要更新的键值对即可，未传入的键不受影响。

**认证**: 需要 JWT Bearer Token + Admin 权限

**请求参数**

请求体为键值对对象（`map[string]string`），键和值均为字符串。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `{key}` | string | 是 | 系统配置键名，至少传入一个键值对 |
| `{key}` 的值 | string | 是 | 对应的配置值 |

> 以上为动态键值对，常用配置项包括：`site_name`（站点名称）、`site_description`（站点描述）、`site_url`（站点 URL）、`icp_number`（ICP 备案号）、`contact_email`（联系邮箱）、`contact_phone`（联系电话）等。实际可设置的键由 `system_settings` 表中的记录决定。

**请求示例**

```json
{
  "site_name": "题小助考试平台",
  "site_description": "专业的在线考试平台"
}
```

**响应示例**

```json
{
  "code": "00000",
  "msg": "更新成功"
}
```

**响应参数**

无 `data` 返回。

**错误代码**

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| `A0001` | 400 | 请求参数错误（JSON 格式错误或类型不匹配） |
| `A0200` | 401 | 未认证（未提供或无效的 Token） |
| `A0301` | 403 | 无管理员权限 |
| `B0001` | 500 | 服务器内部错误 |
