# 08 审计管理 API

本文档覆盖评论审核、举报处理、评论黑名单、自动审核规则、审核统计与导出等审计相关接口。所有管理端接口需要管理员权限（JWT + admin 角色）。

---

## 1. 评论审核

评论审核用于管理员对用户发布的评论进行人工审核，支持单条审核、批量审核以及按条件筛选待审评论。

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/comments` | 获取评论列表（支持按状态、类型、关键词筛选） |
| PUT | `/api/v1/admin/comments/:id/status` | 审核单条评论（通过/隐藏） |
| PUT | `/api/v1/admin/comments/batch-audit` | 批量审核评论 |
| POST | `/api/v1/admin/comments/batch-delete` | 批量删除评论 |

**GET /api/v1/admin/comments** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `target_type` | int8 | 否 | 目标类型: 1=题目, 2=考试, 3=班级 |
| `status` | int8 | 否 | 状态: 0=隐藏, 1=正常, 2=举报 |
| `keyword` | string | 否 | 搜索关键词，匹配评论内容 |
| `user_id` | uint | 否 | 按评论作者筛选 |
| `start_date` | datetime | 否 | 起始时间 |
| `end_date` | datetime | 否 | 截止时间 |
| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页条数，默认 10 |

**PUT /api/v1/admin/comments/batch-audit**

```json
// 请求
{
  "ids": [101, 102, 103],
  "status": 1
}

// 响应
{
  "code": "00000",
  "msg": "批量审核成功",
  "data": null
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `ids` | []uint | 是 | 评论 ID 列表，最少 1 条 |
| `status` | int8 | 是 | 目标状态: 0=隐藏, 1=正常 |

**POST /api/v1/admin/comments/batch-delete**

```json
// 请求
{
  "ids": [101, 102, 103]
}

// 响应
{
  "code": "00000",
  "msg": "批量删除成功",
  "data": null
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `ids` | []uint | 是 | 评论 ID 列表，最少 1 条 |

**响应字段（CommentInfo）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 评论 ID |
| `target_type` | int8 | 目标类型: 1=题目, 2=考试, 3=班级 |
| `target_id` | uint | 目标 ID |
| `user_id` | uint | 评论作者 ID |
| `user_name` | string | 评论作者昵称 |
| `user_avatar` | string | 评论作者头像 |
| `content` | string | 评论内容 |
| `parent_id` | uint | 父评论 ID（null 表示顶级评论） |
| `images` | string | 图片 URL 列表（JSON 数组） |
| `mentions` | string | @的用户 ID 列表（JSON 数组） |
| `reply_to_user_id` | uint | 回复的目标用户 ID |
| `like_count` | int | 点赞数 |
| `is_pinned` | bool | 是否置顶 |
| `is_featured` | bool | 是否精选 |
| `is_official` | bool | 是否官方解答 |
| `status` | int8 | 状态: 0=隐藏, 1=正常, 2=举报 |
| `created_at` | datetime | 创建时间 |

## 2. 举报管理

举报管理用于管理员查看和处理用户提交的评论举报。当举报被判定为成立时，相关评论将自动隐藏。

### 用户端接口 (需登录)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/comments/:id/report` | 举报评论 |

**POST /api/v1/comments/:id/report**

```json
// 请求
{
  "reason": "该评论包含广告内容",
  "reason_type": "spam"
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `reason` | string | 是 | 举报原因描述，最多 200 字 |
| `reason_type` | string | 否 | 举报类型: `spam`=垃圾信息, `abuse`=辱骂, `infringement`=侵权, `other`=其他 |

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/comments/reports` | 获取举报列表 |
| PUT | `/api/v1/admin/comments/reports/:id` | 处理举报 |

**GET /api/v1/admin/comments/reports** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `status` | int8 | 否 | 状态: 0=待处理, 1=已处理（成立）, 2=已驳回 |
| `reason_type` | string | 否 | 举报类型筛选: spam / abuse / infringement / other |
| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页条数，默认 10 |

**PUT /api/v1/admin/comments/reports/:id**

```json
// 请求
{
  "status": 1,
  "handle_remark": "确认为广告内容，已隐藏该评论"
}

// 响应
{
  "code": "00000",
  "msg": "success",
  "data": null
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `status` | int8 | 是 | 处理结果: 1=成立（隐藏评论）, 2=驳回 |
| `handle_remark` | string | 否 | 处理备注，最多 500 字 |

**响应字段（ReportInfo）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 举报记录 ID |
| `comment_id` | uint | 被举报的评论 ID |
| `comment_content` | string | 评论内容摘要（超过 100 字截断） |
| `user_id` | uint | 举报人 ID |
| `user_name` | string | 举报人昵称 |
| `reason` | string | 举报原因 |
| `reason_type` | string | 举报类型: spam / abuse / infringement / other |
| `status` | int8 | 状态: 0=待处理, 1=已处理, 2=已驳回 |
| `handler_id` | uint | 处理人 ID |
| `handle_remark` | string | 处理备注 |
| `handled_at` | datetime | 处理时间 |
| `created_at` | datetime | 举报时间 |

## 3. 评论黑名单

评论黑名单用于限制特定用户在指定目标（题目、考试、班级）或全局范围内发表评论。支持定时拉黑，到期自动解除。

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/comments/blacklist` | 获取黑名单列表 |
| POST | `/api/v1/admin/comments/blacklist` | 添加黑名单 |
| DELETE | `/api/v1/admin/comments/blacklist/:id` | 移除黑名单 |

**GET /api/v1/admin/comments/blacklist** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user_id` | uint | 否 | 按用户筛选 |
| `target_type` | int8 | 否 | 目标类型: 0=全局, 1=题目, 2=考试, 3=班级 |
| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页条数，默认 10 |

**POST /api/v1/admin/comments/blacklist**

```json
// 请求
{
  "user_id": 42,
  "target_type": 0,
  "target_id": 0,
  "reason": "多次发布违规评论",
  "duration": 72
}

// 响应
{
  "code": "00000",
  "msg": "添加黑名单成功",
  "data": null
}
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user_id` | uint | 是 | 被拉黑用户 ID |
| `target_type` | int8 | 是 | 目标类型: 0=全局, 1=题目, 2=考试, 3=班级 |
| `target_id` | uint | 否 | 目标 ID（全局拉黑时填 0） |
| `reason` | string | 否 | 拉黑原因，最多 200 字 |
| `duration` | int | 否 | 拉黑时长（小时），0 或不填表示永久 |

**DELETE /api/v1/admin/comments/blacklist/:id**

```
// 响应
{
  "code": "00000",
  "msg": "移除黑名单成功",
  "data": null
}
```

**响应字段（BlacklistInfo）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 黑名单记录 ID |
| `user_id` | uint | 被拉黑用户 ID |
| `user_name` | string | 被拉黑用户昵称 |
| `user_avatar` | string | 被拉黑用户头像 |
| `target_type` | int8 | 目标类型: 0=全局, 1=题目, 2=考试, 3=班级 |
| `target_id` | uint | 目标 ID（全局时为 0） |
| `reason` | string | 拉黑原因 |
| `operator_id` | uint | 操作人 ID |
| `expires_at` | datetime | 过期时间（null 表示永久） |
| `created_at` | datetime | 创建时间 |

**目标类型说明**

| 值 | 说明 |
|----|------|
| `0` | 全局拉黑，用户无法在任何目标下发表评论 |
| `1` | 题目，仅限制在指定题目下评论 |
| `2` | 考试，仅限制在指定考试下评论 |
| `3` | 班级，仅限制在指定班级下评论 |

## 4. 审核规则管理

审核规则用于评论的自动审核，支持关键词匹配、正则表达式、长度限制、重复内容检测等多种规则类型。规则按优先级从高到低执行，命中后自动执行对应动作。

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/comments/audit-rules` | 获取审核规则列表 |
| POST | `/api/v1/admin/comments/audit-rules` | 创建审核规则 |
| PUT | `/api/v1/admin/comments/audit-rules/:id` | 更新审核规则 |
| DELETE | `/api/v1/admin/comments/audit-rules/:id` | 删除审核规则 |

**GET /api/v1/admin/comments/audit-rules**

```
// 响应
{
  "code": "00000",
  "msg": "success",
  "data": [
    {
      "id": 1,
      "name": "广告关键词过滤",
      "rule_type": "keyword",
      "pattern": "代考,包过,答案出售",
      "action": 3,
      "priority": 100,
      "description": "过滤广告类关键词，自动删除",
      "status": 1,
      "created_at": "2026-05-01 10:00:00",
      "updated_at": "2026-05-01 10:00:00"
    }
  ]
}
```

**POST /api/v1/admin/comments/audit-rules**

```json
// 请求
{
  "name": "辱骂性内容过滤",
  "rule_type": "regex",
  "pattern": "(?i)(傻|蠢|滚|废物)",
  "action": 2,
  "priority": 90,
  "description": "自动隐藏辱骂性评论"
}

// 响应
{
  "code": "00000",
  "msg": "创建成功",
  "data": null
}
```

**创建请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 规则名称，最多 100 字 |
| `rule_type` | string | 是 | 规则类型（见下方说明） |
| `pattern` | string | 是 | 匹配模式（关键词/正则/数值） |
| `action` | int8 | 是 | 处理动作（见下方说明） |
| `priority` | int | 否 | 优先级，数值越大越先执行 |
| `description` | string | 否 | 规则描述，最多 200 字 |

**PUT /api/v1/admin/comments/audit-rules/:id**

```json
// 请求
{
  "name": "辱骂性内容过滤（更新）",
  "action": 3,
  "status": 0
}

// 响应
{
  "code": "00000",
  "msg": "更新成功",
  "data": null
}
```

**更新请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 否 | 规则名称，最多 100 字 |
| `rule_type` | string | 否 | 规则类型 |
| `pattern` | string | 否 | 匹配模式 |
| `action` | int8 | 否 | 处理动作 |
| `priority` | int | 否 | 优先级 |
| `description` | string | 否 | 规则描述，最多 200 字 |
| `status` | int8 | 否 | 状态: 0=禁用, 1=启用 |

**DELETE /api/v1/admin/comments/audit-rules/:id**

```
// 响应
{
  "code": "00000",
  "msg": "删除成功",
  "data": null
}
```

**响应字段（AuditRuleInfo）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | uint | 规则 ID |
| `name` | string | 规则名称 |
| `rule_type` | string | 规则类型 |
| `pattern` | string | 匹配模式 |
| `action` | int8 | 处理动作 |
| `priority` | int | 优先级 |
| `description` | string | 规则描述 |
| `status` | int8 | 状态: 0=禁用, 1=启用 |
| `created_at` | string | 创建时间 |
| `updated_at` | string | 更新时间 |

**规则类型说明**

| 类型 | 说明 | pattern 示例 |
|------|------|-------------|
| `keyword` | 关键词匹配，多个关键词用逗号分隔 | `代考,包过,答案出售` |
| `regex` | 正则表达式匹配 | `(?i)(广告|推广)` |
| `length` | 评论长度检测（超过阈值触发） | `500` |
| `repeat` | 重复内容检测（重复字符/词语占比超过阈值） | `0.5` |

**处理动作说明**

| 动作值 | 说明 |
|--------|------|
| `1` | 标记待审 —— 将评论状态设为待审核，等待人工处理 |
| `2` | 自动隐藏 —— 将评论状态设为隐藏，用户不可见 |
| `3` | 自动删除 —— 软删除评论 |
| `4` | 拒绝发布 —— 直接拒绝评论发布请求，返回错误提示 |

## 5. 审核统计与导出

### 管理端接口 (需管理员权限)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/admin/comments/stats` | 获取评论审核统计 |
| GET | `/api/v1/admin/comments/export` | 导出评论为 Excel |

**GET /api/v1/admin/comments/stats**

```json
// 响应
{
  "code": "00000",
  "msg": "success",
  "data": {
    "total_count": 12580,
    "today_count": 45,
    "pending_count": 12,
    "hidden_count": 38,
    "report_count": 12,
    "daily_stats": [
      {"date": "2026-05-01", "count": 120},
      {"date": "2026-05-02", "count": 98}
    ],
    "top_target_stats": [
      {"target_type": 1, "target_id": 1001, "count": 56},
      {"target_type": 2, "target_id": 2001, "count": 43}
    ]
  }
}
```

**统计响应字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `total_count` | int64 | 评论总数 |
| `today_count` | int64 | 今日新增评论数 |
| `pending_count` | int64 | 待处理举报数 |
| `hidden_count` | int64 | 已隐藏评论数 |
| `report_count` | int64 | 举报总数 |
| `daily_stats` | array | 最近 30 天每日评论数 |
| `top_target_stats` | array | 评论最多的目标 TOP 10 |

**每日统计项（DailyCommentStat）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `date` | string | 日期（YYYY-MM-DD） |
| `count` | int64 | 当日评论数 |

**目标统计项（TargetCommentStat）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `target_type` | int8 | 目标类型: 1=题目, 2=考试, 3=班级 |
| `target_id` | uint | 目标 ID |
| `count` | int64 | 评论数 |

**GET /api/v1/admin/comments/export** 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `target_type` | int8 | 否 | 目标类型筛选 |
| `status` | int8 | 否 | 状态筛选 |
| `start_date` | datetime | 否 | 起始时间 |
| `end_date` | datetime | 否 | 截止时间 |
| `format` | string | 否 | 导出格式: csv, xlsx |
