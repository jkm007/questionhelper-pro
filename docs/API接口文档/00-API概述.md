## 请求格式

- **Content-Type**: `application/json`（常规请求）
- **Content-Type**: `multipart/form-data`（文件上传）
- 所有接口均返回 JSON 格式响应

## 通用响应结构

所有接口统一返回以下 JSON 信封格式：

```json
{
  "code": "00000",
  "msg": "success",
  "data": {}
}
```

| 字段    | 类型   | 说明                                  |
| ------- | ------ | ------------------------------------- |
| `code`  | string | 响应码，`"00000"` 表示成功           |
| `msg`   | string | 响应消息                              |
| `data`  | object | 响应数据，具体结构因接口而异          |

## 分页请求参数

支持分页的接口统一使用以下查询参数：

| 参数       | 类型   | 必填 | 默认值 | 说明               |
| ---------- | ------ | ---- | ------ | ------------------ |
| `page`     | int    | 否   | 1      | 页码，从 1 开始    |
| `page_size`| int    | 否   | 10     | 每页条数，最大 100 |

## 分页响应结构

分页接口在 `data` 中返回以下结构：

```json
{
  "code": "00000",
  "msg": "success",
  "data": {
    "list": [],
    "total": 0,
    "page": 1,
    "page_size": 10
  }
}
```

| 字段       | 类型   | 说明         |
| ---------- | ------ | ------------ |
| `list`     | array  | 当前页数据   |
| `total`    | int    | 总记录数     |
| `page`     | int    | 当前页码     |
| `page_size`| int    | 每页条数     |

## 认证与授权

### JWT Token 说明

系统采用 JWT（JSON Web Token）认证机制，包含两个 Token：

- **access_token**：访问令牌，用于请求接口时的身份验证
- **refresh_token**：刷新令牌，用于在 access_token 过期后获取新的 token 对

### 请求头

需要认证的接口必须在请求头中携带 access_token：

```
Authorization: Bearer <access_token>
```

### 权限级别

| 级别     | 说明                   | 请求头要求                    |
| -------- | ---------------------- | ----------------------------- |
| Public   | 公开接口，无需认证     | 无                            |
| Authorized | 需要用户登录         | `Authorization: Bearer <token>` |
| Admin    | 需要管理员权限         | `Authorization: Bearer <token>`（token 对应用户须具有 admin 角色） |

### Token 配置

| 配置项             | 值       | 说明             |
| ------------------ | -------- | ---------------- |
| access_token 有效期 | 7200s    | 2 小时           |
| refresh_token 有效期 | 604800s  | 7 天             |

## 通用约定

### HTTP 状态码

| 状态码 | 含义       | 说明                       |
| ------ | ---------- | -------------------------- |
| 200    | 成功       | 请求处理成功               |
| 400    | 参数错误   | 请求参数校验失败           |
| 401    | 未认证     | 未提供 Token 或 Token 无效 |
| 403    | 无权限     | 已认证但无权访问该资源     |
| 404    | 不存在     | 请求的资源不存在           |
| 409    | 冲突       | 资源状态冲突（如重复创建） |
| 422    | 语义错误   | 请求格式正确但语义有误     |
| 429    | 频率限制   | 请求过于频繁，触发限流     |
| 500    | 服务器错误 | 服务端内部异常             |
